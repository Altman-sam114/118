# v0.5（CI 恢复 Native 资产）

## 版本号

v0.5

## 版本分配依据

v0.4 已建立 `main` 直推和云端结果包流程。首次云端 run `28669505695` 显示 CI checkout 缺少被 `.gitignore` 排除的 `LocalDiffusionNative.xcframework`，导致 native preflight 和 Xcode build 失败。本轮是 v0 治理和 CI 骨架的小版本修复，因此使用 `v0.5`。

## 背景

`LocalDiffusionNative.xcframework` 是 170MB 级生成物，不适合普通 git 跟踪，且当前 `.gitignore` 明确排除 `LocalDiffusion/Frameworks/*.xcframework/`。云端要完成 native preflight 和 iPhoneOS build，必须在 workflow 中恢复该资产。

## 目标

- 将本地 `LocalDiffusionNative.xcframework` 打包为 `LocalDiffusionNative.xcframework.zip` 并上传到 GitHub Release `native-backend-current`。
- 修改 `.github/workflows/ci-results.yml`，在验证前下载并解压该 Release asset。
- 将恢复日志纳入 CI 结果包。
- 同步 `md/test/test.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、README 和 `update_log.md`。
- 重新 push `main`，等待 Actions，下载并核对新结果包。

## 非目标

- 不把 170MB native XCFramework 提交进 git。
- 不修改 Swift 业务源码、Xcode project、native bridge ABI 或 stable-diffusion.cpp 包装代码。
- 不引入 PR、候选分支或多分支发布流。

## 架构依据

- native backend 仍通过 `ImageGenerationBackend` 间接暴露给 UI。
- `Scripts/check-native-backend.sh` 仍是 native linkage 和 ABI 包装预检入口。
- 生成的 native binary 作为外部 CI 资产管理，源码和项目配置仍由 git 跟踪。

## 实现步骤

1. 下载并核对失败 run 的 CI 结果包，确认 manifest、JUnit、failure summary 和日志。
2. 修改 workflow，增加 Release asset 下载和解压步骤。
3. 将 `native-backend-restore.log` 纳入结果包和文档。
4. 打包并上传 `LocalDiffusionNative.xcframework.zip` 到 Release `native-backend-current`。
5. 运行本地轻量检查。
6. 提交 `v0.5` 并 push `origin/main`。
7. 等待新 run，下载 artifact，核对 manifest 的 branch、commit、run id、run attempt 和关键日志。

## 文档更新要求

- `md/test/test.md` 写清 Release asset 名称、恢复日志和缺失时的失败解释。
- `flow.md` 与 `flowchart.md` 展示 CI 恢复 native backend 资产。
- README 简短说明 CI native asset 来源。
- `update_log.md` 记录 v0.5 是 CI 资产恢复修复。

## 验收标准

- GitHub Release `native-backend-current` 有 `LocalDiffusionNative.xcframework.zip`。
- 新 run 的 manifest 指向 `main`、最新 commit、正确 run id 和 run attempt。
- `native-backend-restore.log` 显示 asset 已下载并解压。
- native preflight 通过。
- Xcode build 通过，或若失败，结果包能定位到新的真实原因。

## 风险和禁止项

- 禁止把 ignored XCFramework 强行塞进普通 git。
- 禁止用旧 artifact 或旧 run 冒充新 run。
- 禁止在 Release asset 缺失时声称云端验证通过。
- Release asset 刷新后必须重新跑云端 CI。
