# v0.1（建立多 Agent 迭代文档）

## 版本号

v0.1

## 版本分配依据

人工未指定版本号。本轮目标是建立项目长期协作和治理文档体系，不是实现业务功能，因此从 `v0.1` 项目治理阶段开始。

## 背景

当前项目是 Local Diffusion：一个 SwiftUI + SwiftData 的原生 iOS 本地图像生成 App，已具备模型管理、生成页、图库、提示词库和 stable-diffusion.cpp native backend 接入。项目需要一套长期迭代的 Agent A/B/C 协作系统，避免后续任务缺少上下文、测试规范和核心流程文档。

## 目标

- 创建或更新 `AGENTS.md`。
- 创建 `update_log.md`。
- 创建 `md/prompt/README.md` 和版本目录。
- 创建 `md/test/test.md`。
- 创建 `md/flow/flow.md`。
- 创建 `md/flow/flowchart.md`。
- 更新 README 的 Agent handoff 入口。

## 非目标

- 不开发新业务功能。
- 不修改 Swift 源码。
- 不重建 native backend。
- 不改变当前 UI 或推理行为。

## 当前架构依据

- App 入口：`LocalDiffusion/App/LocalDiffusionApp.swift`
- 导航：`LocalDiffusion/Views/RootContentView.swift`
- 生成：`LocalDiffusion/ViewModels/GenerationViewModel.swift`
- 推理边界：`LocalDiffusion/Inference/ImageGenerationBackend.swift`
- 数据模型：`LocalDiffusion/Models/AppModels.swift`
- 存储：`LocalDiffusion/Services/AppFileStore.swift`
- 下载：`LocalDiffusion/Services/HuggingFaceDownloadManager.swift`
- 验证脚本：`Scripts/check-native-backend.sh`、`Scripts/smoke-test-simulator.sh`

## 实现步骤

1. 读取 README、旧 `AGENTS.md`/`agent.md`、git 状态、脚本和关键源码。
2. 建立 `md/prompt`、`md/test`、`md/flow` 目录。
3. 将入口规则写入 `AGENTS.md`，保持精简。
4. 将历史、决策、遗留问题写入 `update_log.md`。
5. 将测试分层和命令写入 `md/test/test.md`。
6. 将真实核心架构和运行流程写入 `md/flow/flow.md`。
7. 先基于 `flow.md` 写 `md/flow/flowchart.md`，包含核心逻辑图和 Agent 迭代图。
8. 更新 README，指向新的文档体系。
9. 运行 `git diff --check`。

## 测试要求

文档-only 修改至少运行：

```bash
git diff --check
```

无需运行完整 iOS 构建；最终输出必须说明未跑业务测试的原因。

## 文档更新要求

- README 必须指向新 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/test/test.md`。
- `update_log.md` 必须记录本次 v0.1 文档体系建立。
- `md/flow/flowchart.md` 必须与 `md/flow/flow.md` 内容一致。

## 验收标准

- 所有要求的文件和目录存在。
- 文档内容适合当前 Local Diffusion 项目，不是空泛模板。
- Mermaid 图描述真实数据流、执行流和 Agent 迭代流。
- `git diff --check` 通过。

## 风险和禁止项

- 禁止编造当前项目没有的测试工具或 CI。
- 禁止把历史流水账堆进 `AGENTS.md`。
- 禁止改业务代码。
- 禁止把 mock backend 记为真实推理验收已完成。
