# v1.95 / Gallery 文件夹删除确认语义

## 版本号

v1.95

## 版本分配依据

- v1.94 已完成 Gallery image delete confirmation buttons 的当前图片上下文 VoiceOver 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续 Gallery 删除确认链路体验优化，聚焦 folder 删除确认弹窗，让 destructive / cancel 按钮保留待删除 folder 名称和确认/取消含义。

## 背景

Gallery folder 的 swipe action 和 context menu action 已经提供 `Delete folder: <folder name>` 语义，但 `.confirmationDialog("Delete folder?")` 内的两个按钮目前只有可见文案：

- destructive `Delete Folder`；
- cancel `Cancel`。

删除确认弹窗 message 已说明图片会留在图库并回到 All Images，但按钮本身没有 folder 名上下文，也没有明确确认删除 folder 或取消保留 folder 的辅助功能说明。

## 目标

- 为 folder 删除 confirmation dialog 的 destructive `Delete Folder` button 增加：
  - accessibility label：`Delete folder: <folder name>`；
  - accessibility value：待删除 folder 名称；
  - accessibility hint：说明只删除 folder，图片保留并回到 All Images。
- 为 cancel `Cancel` button 增加：
  - accessibility label：`Cancel deleting folder: <folder name>`；
  - accessibility value：待删除 folder 名称；
  - accessibility hint：说明保留 folder 并关闭确认。
- 可新增小型 private helper，用于从 `pendingFolderDeletion?.name` 生成稳定的 folder 名称和值。
- 保留 dialog title、message、按钮可见文案、destructive/cancel roles、`deleteFolder(id:)` 行为、`pendingFolderDeletion = nil` 清理和 `folderDeleteBinding` 不变。

## 非目标

- 不修改 `deleteFolder(id:)` 的数据行为：不得删除图片文件，不得删除 `GeneratedImage`，不得改变 `folderID = nil` 迁移逻辑。
- 不修改 `folderDeleteBinding`、`pendingFolderDeletion` 的展示/清理机制、swipe action、context menu action、folder rename、新建 folder 或 folder filter 行为。
- 不修改 Gallery image delete flow、detail delete confirmation、Tags、Folder picker、sorting、thumbnail loading、SwiftData schema、file storage、`GenerationViewModel`、native backend、bridge、workflow、Xcode project、StoreKit、paid gates、Mac Catalyst 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Gallery 页面位于 `LocalDiffusion/Views/Gallery/GalleryView.swift`，属于 UI 展示层。
- `FolderEditorState` 已保存待操作 folder 的 `id` 和 `name`，可直接用于 confirmation dialog button accessibility。
- `deleteFolder(id:)` 只处理 folder 删除与图片 folderID 迁移；本轮不进入数据层或文件存储边界。

## 实现步骤

1. 阅读 `.confirmationDialog("Delete folder?")`、`pendingFolderDeletion`、`folderDeleteBinding`、`deleteFolder(id:)` 和已有 folder delete swipe/context menu action。
2. 新增或复用 helper 获取待删除 folder 名称。
3. 在 destructive `Delete Folder` button 后追加 accessibility label/value/hint。
4. 在 cancel `Cancel` button 后追加 accessibility label/value/hint。
5. 核对没有改动 dialog title/message、button roles、visible copy、deleteFolder 行为、pending state 清理或 folder filter logic。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
