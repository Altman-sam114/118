# v1.26 / Add Model 键盘提交语义

## 版本号

v1.26

## 版本分配依据

- 当前体验优化阶段已完成到 v1.25。
- v1.24 优化了 Add Model 错误提示，v1.25 优化了 Models 存储摘要语义。
- Add Model 表单当前依赖 toolbar `Download` 和 `Parse Hugging Face URL` 按钮；文本输入没有 Return/外接键盘提交语义，确认按钮也缺少明确 ready/missing VoiceOver value。

## 背景

`AddModelView` 是 Models 页面新增 Hugging Face GGUF 模型的入口。当前表单包含：

- `huggingFaceURL`
- `name`
- `repository`
- `filename`
- `revision`
- `family`
- `Parse Hugging Face URL`
- toolbar `Download`

本轮只改善表单提交语义和辅助功能反馈，不改变 URL 解析、重复检测、文件存在检测、下载创建或 SwiftData 写入。

## 目标

- 为 Add Model 表单的关键文本输入增加合理的 submit label / onSubmit 行为：
  - Hugging Face URL 输入按 Return 时，在输入非空时执行现有 `applyHuggingFaceURL()`。
  - 文件名或 revision 输入按 Return 时，在 `canSubmit` 为 true 时执行现有新增模型流程。
- 将 toolbar `Download` 按钮 action 包装为安全提交 helper，避免禁用状态外的键盘提交绕过 `canSubmit`。
- 为 `Download` 按钮增加明确 accessibility label/value/hint，表达 ready 或 missing required fields。
- 可选为 `Parse Hugging Face URL` 按钮补充 accessibility value/hint，表达 URL 是否可解析。
- 保留现有视觉布局、按钮文案、错误文案、禁用逻辑、表单 section 和 Sci-Fi 风格。

## 非目标

- 不修改 `ParsedHuggingFaceFileURL`、`HuggingFaceURLBuilder`、`AppFileStore`、`LocalModel`、`HuggingFaceDownloadManager` 或 SwiftData schema。
- 不修改 URL 解析规则、错误文案、重复检测、文件存在检测、下载启动或 dismiss 行为。
- 不新增焦点状态机、表单重排、输入校验 UI、StoreKit、Mac Catalyst、native backend 或 CI workflow。
- 不修改 Untracked file import、Model row、Model detail、Storage Matrix 或其他页面。

## 当前架构依据

- `AddModelView` 属于 UI 展示层，只负责输入、反馈和调用 `onAdd(model)`。
- 真正下载仍由外层 `ModelLibraryView` 插入 SwiftData 并调用 `downloads.start(model)`。
- 本轮应复用已有 `applyHuggingFaceURL()`、`addModel()` 和 `canSubmit`，不得复制或重写业务校验。
