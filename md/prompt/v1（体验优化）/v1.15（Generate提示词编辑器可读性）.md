# v1.15（Generate提示词编辑器可读性）

## 版本号

v1.15

## 版本分配依据

v1.14 已优化 Prompt Template Editor 的 prompt 编辑体验，让保存模板时的 positive/negative prompt 输入区具备可见标题、placeholder、字符计数和 Dynamic Type-friendly 高度。当前 Generate 页主 prompt editor 仍存在两个小问题：header 在 accessibility Dynamic Type 下强制横排，字符计数使用过小的 `caption2`；清除按钮是 image-only `Button`，命中区不足 44pt，也没有真实文本 `Label`。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个高频但可独立验收的小步：提升 Generate 页 prompt 编辑器 header 与清除按钮可读性，不改变生成参数或生成流程。

## 背景

Generate 页是图片生成主入口，positive prompt 和 negative prompt 是最高频输入区。v1.9 已让 Generate 在 accessibility Dynamic Type 下回退单列，v1.13 已改善共享参数控件，v1.14 已改善模板编辑器。现在 `GenerationView.promptEditor(...)` 内部仍使用单个 `HStack` 承载标题、字符数和清除按钮，大字号或窄 iPad split/detail 场景下容易挤压；清除按钮虽然有 `accessibilityLabel`，但视觉 label 仍是裸 `Image(systemName:)`，不符合当前项目对 icon-only 控件的实现方式。

## 目标

- 只优化 `LocalDiffusion/Views/Generation/GenerationView.swift` 中的 `promptEditor(...)` UI 展示层。
- 将 prompt editor header 在 accessibility Dynamic Type 下改为更可读的纵向或分行布局，避免标题、字符计数和清除按钮强制挤在一行。
- 将字符计数从过小 `caption2` 调整到更可读的 Dynamic Type 文本样式，并保留 monospaced digit。
- 将清除按钮改为带真实文本的 `Label("Clear <title>", systemImage: "xmark.circle.fill")`，普通字号可继续 `.labelStyle(.iconOnly)` 保持紧凑视觉。
- 清除按钮补足至少 44x44 命中区。
- 清除按钮只在对应 prompt 非空时出现，点击后仍只清空对应 `Binding<String>`。
- 给 prompt `TextEditor` 补足明确 accessibility label/value/hint，区分 positive 与 negative prompt。
- placeholder 继续只在文本为空时显示，不写入参数值，并避免遮挡触摸或重复 VoiceOver 朗读。
- 保持 existing title、placeholder、focus、minHeight、accent、prompt/negativePrompt binding、generation gate、Save Template、Generate/Cancel 行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不修改 `GenerationParameters` 默认值、范围、归一化、prompt 字段语义、negative prompt 字段语义或 sampler/seed/size 行为。
- 不修改 `GenerationViewModel`、`ImageGenerationBackend`、SwiftData schema、文件存储、native bridge、Hugging Face 下载或 Gallery 逻辑。
- 不修改 `PromptTemplateEditor`、`ParameterEditor`、Prompt Library 分类/加载/保存逻辑。
- 不接入 StoreKit，不新增购买、恢复、receipt、subscription、product ID 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 Xcode project 平台设置、native XCFramework、CI workflow。
- 不重构 Generate 整页布局，不改变 compact / iPad regular / accessibility Dynamic Type 的页面选择逻辑。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `GenerationView` 属于 UI 展示层，负责收集 prompt、参数、模型选择和用户操作。
- `GenerationView.promptEditor(...)` 只编辑 `viewModel.parameters.prompt` 和 `viewModel.parameters.negativePrompt` 的 binding；本轮不得改变 binding 指向或保存/生成路径。
- `GenerationViewModel` 负责生成状态、校验和调用后端，本轮不得触碰。
- v1.9 已用 `dynamicTypeSize.isAccessibilitySize` 让 Generate 页面在大字号下回退单列，本轮可在局部 header 中复用同一环境值。
- v1.10-v1.14 已把 icon-only 控件改为带文本 `Label` 后再视觉 icon-only，本轮应沿用该模式。

## 实现步骤

1. 阅读必读文件、本提示词、`GenerationView.swift` 中 `promptsSection` 和 `promptEditor(...)`。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `promptEditor(...)` 内调整 header：
   - 普通 Dynamic Type 下保持紧凑横排。
   - accessibility Dynamic Type 下标题单独成行，字符计数与清除按钮可分行或纵向排列，避免三者强制挤压。
4. 将字符计数显示为更清晰的 `"<count> chars"` 或等价文本，使用不小于 `.caption` 的样式并保留 monospaced digit。
5. 将清除按钮改为 `Label("Clear \(title)", systemImage: "xmark.circle.fill")`：
   - 普通字号可 `.labelStyle(.iconOnly)`。
   - 按钮至少 `frame(minWidth: 44, minHeight: 44)`。
   - 继续使用 `.buttonStyle(.plain)` 或当前视觉兼容的样式。
   - 点击行为仍是 `text.wrappedValue = ""`。
6. 给 `TextEditor` 添加明确 accessibility：
   - label 使用字段 title。
   - value 为空时为 `Empty prompt`，非空时可读当前文本或说明内容存在。
   - hint 区分 positive prompt / negative prompt 的用途。
7. placeholder overlay 加上 `.allowsHitTesting(false)` 和 `.accessibilityHidden(true)`，避免挡住编辑或重复朗读。
8. 保持 `.focused($focusedPrompt, equals: field)`、`.frame(minHeight: minHeight)`、背景、描边和现有 prompt editor 调用参数不变。
9. 同步文档：
   - `README.md`：说明 Generate prompt editors 的 header/clear controls 更适合 Dynamic Type 和辅助功能。
   - `md/flow/flow.md`：更新 Generate UI 职责。
   - `md/flow/flowchart.md`：更新 Generate 节点说明。
   - `update_log.md`：追加 v1.15 记录。

- Generate 页 positive prompt 和 negative prompt header 在 accessibility Dynamic Type 下不再强制把 title、count、clear button 挤在一行。

- placeholder 不挡住编辑触摸，也不会被 VoiceOver 重复朗读。
- TextEditor 有明确 accessibility label/value/hint，区分 positive/negative prompt 用途。
- prompt focus、prompt/negativePrompt binding、Save Template、generation gate、Generate/Cancel、View in Gallery 行为保持不变。
- 不新增 StoreKit、Mac Catalyst、Xcode project、native backend、SwiftData schema、文件存储或 CI workflow 改动。
- 本地轻量检查和 Swift parse 通过；建议的 iPhoneOS build 通过或明确记录沙箱限制。
