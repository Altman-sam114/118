# v1.58 Prompt 搜索入口语义

## 版本号

v1.58

## 版本分配依据

v1.57 已完成 Plan summary row VoiceOver hints。本轮继续 Agent X 的体验优化循环，转向 Prompt Library 的 iPad / Mac 键盘与辅助功能细节，选择一个低风险、可验收的小改动。

## 背景

Prompt Library 已具备模板分类、搜索、空态、分类菜单、模板行按钮和指标 pill 的可访问语义。当前搜索入口使用 `.searchable(text:prompt:)`，筛选逻辑正确，但搜索框本身还可以补充更明确的键盘提交意图和 VoiceOver 上下文，帮助 iPad 外接键盘、Mac 指针/键盘环境和 VoiceOver 用户理解该字段会搜索哪些模板内容。

## 目标

- 为 Prompt Library 搜索入口增加搜索提交语义。
- 为搜索入口增加清晰 VoiceOver hint，说明会按模板名称、分类、positive prompt 和 negative prompt 过滤模板。
- 保持现有搜索字段视觉、占位文案、筛选算法、空态文案和模板列表行为不变。
- 同步更新当前实现文档与维护日志。

## 非目标

- 不修改 `filteredTemplates` 或 `matchesSearch(_:)` 的匹配规则。
- 不新增搜索 token、scope、排序、分组、highlight、debounce 或状态持久化。
- 不修改模板新增、编辑、删除、加载、分类重命名、分类清空或 SwiftData query。
- 不修改 SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。
- 不做 simulator VoiceOver 目检、Mac build、StoreKit 测试或真实 GGUF 生成。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，Prompt Library 位于 `LocalDiffusion/Views/Prompts/PromptLibraryView.swift`。
- 搜索输入只写入本地 `@State searchText`，再由 `filteredTemplates` 读取并调用 `PromptTemplate.matchesSearch(_:)`。
- 本轮只添加 SwiftUI modifier，不跨越 SwiftData、文件存储、推理后端或 native bridge 边界。

## 实现步骤

1. 在 `PromptLibraryView` 的 `.searchable(text:prompt:)` 后追加 `.submitLabel(.search)`。
2. 在同一搜索入口追加 `.accessibilityHint(...)`，说明搜索会过滤模板名称、分类、positive prompt 和 negative prompt。
3. 不改动 `searchText`、`filteredTemplates`、`matchesSearch(_:)`、列表空态或模板 mutation 逻辑。
4. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录 v1.58 的当前真实行为与风险边界。
