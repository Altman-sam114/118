# v1.119 / Generate 控制台状态汇总语义

## 版本号

- 版本：`v1.119`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.118 / Plan Platform Mac 发行流水线边界语义`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- 本轮从 Plan 的 Mac 发行边界转向 iPad/Generate 使用体验，选择 Generate iPad 右栏 console status 区域做一个小而可验收的 VoiceOver 汇总语义优化。

## 背景

Generate 在 iPad regular 下使用双栏创作台，右栏展示 Local Render Console、backend/model status pills、steps/canvas metrics、run gate 和 result。当前 `backendStatusPill` 与 `modelStatusPill` 各自可组合朗读，但 `consoleStatusPills` 本身没有一个汇总 backend readiness 与 selected model readiness 的 row-level VoiceOver 摘要。VoiceOver 用户在 iPad 右栏扫描时，需要更快理解当前 console 是否 ready、backend 是否 online、以及当前模型状态。

## 目标

1. 只增强 `GenerationView.consoleStatusPills` 的组合辅助功能语义。
2. 增加 `consoleStatusAccessibilityValue` 和 `consoleStatusAccessibilityHint`，汇总 backend online/offline、backend title/message、selected model 名称或 no model 状态。
3. 保持 status pills 可见文案、布局、颜色、图标、Dynamic Type 分支和生成逻辑不变。
4. 不改变 `GenerationGatePanel`、model picker、run button、progress、result 或参数控件行为。

## 非目标

- 不修改 backend selection、generation gate、model selection、prompt editing、parameter values、generation save flow、Gallery handoff 或 SwiftData。
- 不修改 `SciFiStatusPill` 共享组件。
- 不改变 iPad dual-column layout、compact layout、Dynamic Type layout 分支或视觉样式。
- 不启用 StoreKit、Mac Catalyst、native backend 配置、Xcode project、workflow 或测试规范。

## 当前架构依据

- `GenerationView` 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，属于 SwiftUI UI 展示层。
- `consoleStatusPills` 在 iPad regular 右栏与 compact/single-column console section 中复用。
- `viewModel.backendStatus` 提供 backend title/message/isReady。
- `selectedModel` 和 `readyModels` 提供当前 ready model 状态。
- 本轮只改辅助功能语义，不触碰状态层、数据层、文件存储、native backend 或 SwiftData schema。

## 实现步骤

1. 阅读 `GenerationView.consoleStatusPills`、`backendStatusPill`、`modelStatusPill`、`modelPickerAccessibilityValue` 和 `generationGate`。
2. 在 `consoleStatusPills` 的外层 stack 上添加：
   - `.accessibilityElement(children: .ignore)`
   - `.accessibilityLabel(Text("Render console status"))`
   - `.accessibilityValue(Text(consoleStatusAccessibilityValue))`
   - `.accessibilityHint(Text(consoleStatusAccessibilityHint))`
3. 新增 `consoleStatusAccessibilityValue`：
   - backend 部分区分 online/offline，并包含 `viewModel.backendStatus.title` 与 `viewModel.backendStatus.message`。
   - model 部分区分 selected model 与 no ready model selected，并包含 ready model count。
4. 新增 `consoleStatusAccessibilityHint`：
   - backend offline 时提示需要 backend available。
   - no selected model 时提示 Open Models 下载或导入 ready GGUF model。
   - ready 时提示 console is ready to use current prompts and parameters.
5. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
6. 检查 diff，确认只包含本轮相关源码、文档和本提示词。

## 风险和禁止项

- 风险：将 status pills 组合为一个 VoiceOver 元素会减少逐个 pill 的 swipe 停靠点；本轮选择更快的 console readiness summary，后续可再按实机 VoiceOver 调整。
- 风险：本轮不做 simulator 或真机 VoiceOver 朗读检查，实际朗读顺序仍需后续人工辅助功能 QA。
- 禁止修改生成门禁、模型选择、参数编辑、native backend、SwiftData schema、文件存储、Gallery handoff、workflow 或测试规范。
