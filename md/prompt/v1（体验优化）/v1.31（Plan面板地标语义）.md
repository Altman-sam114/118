# v1.31 / Plan 面板地标语义

## 版本号

v1.31

## 版本分配依据

- 当前体验优化阶段已完成到 v1.30。
- v1.18-v1.21 已补齐 Plan overview、状态行、Availability 行和说明行的 VoiceOver label/value/hint。
- iPad regular 的 Plan 页面使用自定义 `PlanPanel` 组成双栏阅读布局，panel 标题和 footer 仍只是普通 `Text`，不像 compact `Form` 的 `Section` header/footer 那样天然提供阅读结构。
- 本轮继续人工总目标中的 iPad、UI、设计和付费能力规划优化，选择一个可独立验收的小步：为 Plan 宽屏面板补充 heading/reading landmark 语义，不新增付费能力、不启用 Mac Catalyst。

## 背景

Plan 是当前本地计划、StoreKit 未配置、付费候选、entitlement rules、Availability 和 Mac readiness 的事实入口。compact 布局使用 `Form` section，系统可以为 VoiceOver 提供较清晰的分区结构；iPad regular 布局使用自定义 `PlanPanel`，标题视觉上像 section header，但辅助技术不一定能把它当作可导航 heading，footer 脱离 `Section` 后也缺少上下文 label。

本轮只增强 Plan 的宽屏阅读结构，让 VoiceOver 用户更容易在 iPad 双栏布局中按面板跳转和理解 footer 说明。

## 目标

- 只优化 `LocalDiffusion/Views/RootContentView.swift` 中 Plan 宽屏 panel 的辅助功能语义。
- `PlanPanel` 标题应暴露为 heading，便于 VoiceOver rotor 或等价阅读导航按面板跳转。
- `PlanPanel` footer 应保留现有视觉文本，同时提供带面板标题上下文的 accessibility label/value，例如 `Mac Readiness note` + footer 文本。
- 可为 `planOverview` 增加简短 hint，说明这是 Plan overview，不改变现有 label/value 摘要。
- 保持 Plan 现有视觉布局、暗色 Sci-Fi 风格、panel 间距、双栏/单列回退条件和内容顺序不变。
- 保持 Current Build、Platform Status、Mac Readiness、Capability Matrix、Entitlement Rules、Availability 的事实内容不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不新增或删除 capability、entitlement rule、availability item 或 Mac readiness item。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、native XCFramework、Xcode project、签名或分发配置。
- 不修改 Root 导航、compact `TabView`、iPad `NavigationSplitView`、Generate、Models、Gallery、Prompts、SwiftData schema、文件存储、native backend、CI workflow 或生成流程。
- 不做 simulator screenshot、VoiceOver 目检、真实 StoreKit 测试、Mac build 或真机 GGUF 生成验收。

## 当前架构依据

- `PlanView` 位于 `RootContentView.swift`，属于 SwiftUI UI 展示层，只陈述当前 Local plan、StoreKit 未配置、Mac 未启用和未来候选能力。
- `PlanPanel` 只用于 iPad regular 的自定义阅读布局；compact `Form` 仍使用系统 `Section`。
- `PlanStatusSummaryRow`、`PlanStatusRow`、`PlanNoteRow` 和 `planOverview` 已有明确 VoiceOver label/value/hint，本轮不重做这些行级语义。
- SwiftUI 可访问性要求：重要分区标题应能被辅助技术识别，说明性 footer 脱离系统 `Section` 后应补充上下文，避免用户只听到孤立说明文本。
