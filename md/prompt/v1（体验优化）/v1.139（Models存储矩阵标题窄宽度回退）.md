## 版本号

- 版本：`v1.139`

## 版本分配依据

- 当前最新正式版本为 `v1.138 / Models 未跟踪行窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- Models 列表 tracked/untracked 行已完成窄宽度回退；`StorageSummaryRow.header` 在普通 Dynamic Type 下仍固定横排 title + ready pill，是下一个小而可验收的 UI 韧性优化。

## 背景

`StorageSummaryRow.header` 在 accessibility Dynamic Type 下已纵向堆叠 titleBlock + readyPill，但普通 Dynamic Type 固定 `HStack`。Storage Matrix 标题与 ready pill 在窄 list / iPad split 下可能挤压。本轮用 `ViewThatFits(in: .horizontal)` 对齐既有模式。

## 目标

1. 只修改 `StorageSummaryRow.header` 的普通 Dynamic Type 分支。
2. `ViewThatFits(in: .horizontal)` 优先横排 titleBlock + readyPill，不足时回退纵向。
3. accessibility 纵向布局不变。
4. 保留 storage metrics、accessibility summary、ready pill 文案/颜色不变。

## 非目标

- 不修改 tracked/untracked rows、download manager、Add Model、detail sheet、SwiftData、native、StoreKit、Mac Catalyst、workflow。
- 禁止本地 Xcode / Simulator / 真机构建测试。

## 实现步骤

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        titleBlock
        Spacer(minLength: 12)
        readyPill
    }

    VStack(alignment: .leading, spacing: 8) {
        titleBlock
        readyPill
    }
}
```
