# v1.106 Models 未跟踪导入字段语义

## 版本号

- 版本：`v1.106`
- 阶段：`v1（体验优化）`
- 任务名：Models 未跟踪导入字段语义

## 版本分配依据

- 最新已完成版本为 `v1.105`，已补齐 Prompt Library 分类标题 heading 与当前可见模板数量语义。
- 本轮转向 Models 的未跟踪 GGUF 导入编辑器，继续做小步 UI/可访问性优化，让导入表单字段本身暴露明确 label、value 和下一步提示，递增为 `v1.106`。

## 背景

`UntrackedModelImportEditor` 负责把 Application Support 中未连接 SwiftData metadata 的 GGUF 文件导入 Models。当前 toolbar 的 Import / Cancel 已有 ready/name-required 语义，但 `Display name` TextField 和 `Family` Picker 主要依赖默认控件语义。VoiceOver 用户进入导入表单时，不容易直接知道显示名是导入前必填、当前草稿名称是什么，以及 family 会写入导入模型的 metadata。本轮只补字段级辅助功能语义，不改变导入逻辑、默认名称或 family 推断。

## 目标

1. 给 `UntrackedModelImportEditor` 的 `Display name` TextField 增加明确 accessibility label。
2. `Display name` value 区分空值和当前 trimmed name，空值读作 `No display name`。
3. `Display name` hint 说明导入前必填，并会作为 Models 列表显示名。
4. 给 `Family` Picker 增加明确 accessibility label、当前 family value 和 metadata 用途 hint。
5. 保留默认 name 推断、family 推断、Import disabled gate、`onImport(name, family)` 和 dismiss 行为不变。

## 非目标

- 不修改未跟踪文件扫描、导入逻辑、默认文件名转 display name、`ModelFamily.inferred(from:)` 或 `performUntrackedModelImport`。
- 不修改 `LocalModel` SwiftData schema、`AppFileStore`、`HuggingFaceDownloadManager`、文件存储或下载状态机。
- 不修改 Add Model 下载表单、Models list/detail、delete confirmation、toolbar menu 或 storage matrix。
- 不新增 StoreKit product、entitlement gate、Mac Catalyst 设置、native backend、Xcode project 或 workflow 变更。
- 不新增 simulator、真机 VoiceOver 或完整本地 build 要求。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，本轮只改 SwiftUI 表单语义。
- `UntrackedModelImportEditor` 是未跟踪 GGUF 文件导入的局部编辑器，适合承载字段级 label/value/hint。
- Import toolbar 已负责导入 ready/name-required 状态；字段本体补充当前值和用途语义即可，不需要改数据流。
- 测试策略按 `md/test/test.md`：SwiftUI 源码变更需跑本地轻量检查、普通 Swift parse、native bridge Swift parse，并通过 `main` push 触发 GitHub Actions 结果包。
