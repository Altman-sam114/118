# v1.68 / Generate 模型选择语义

## 版本号

v1.68

## 版本分配依据

- v1.67 已完成 Generate 空模型状态 `Open Models` 入口语义和 44pt 命中区，并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段补齐 Generate Model section 中 ready model 选择器的 VoiceOver 上下文。

## 背景

Generate Model section 在没有 ready model 时已经提供明确的 `Open Models` 下一步入口语义。存在 ready model 时，`Picker("Model", selection:)` 主要依赖系统默认朗读，VoiceOver 缺少当前选中模型和 ready model 数量上下文，也没有说明该选择器会选择用于下一次本地生成的 ready GGUF 模型。

## 目标

- 为 Generate Model section 的 ready model Picker 补充稳定 VoiceOver label。
- VoiceOver value 暴露当前选中模型名，并包含 ready model 数量。
- VoiceOver hint 说明该 Picker 用于选择下一次本地生成使用的 ready GGUF 模型。
- 保持现有默认选择、`selectedModelBinding` 和模型列表显示行为不变。

## 非目标

- 不修改 `selectedModel` fallback、`selectedModelBinding`、`viewModel.selectedModelID`、模型查询排序、模型下载/导入、生成门禁、生成/取消行为、Prompt/参数编辑、SwiftData、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 GitHub Actions workflow。
- 不修改空模型状态 `Open Models` 按钮、Run section、iPad 双栏布局或 Generate result handoff。
- 不新增购买、订阅、entitlement、价格、product ID 或 Mac 平台开关。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。
- Generate 页面通过 `@Query` 读取模型元数据，通过 `GenerationViewModel.selectedModelID` 保存当前选择；本轮只增强 Picker 语义，不改变数据流。
- 当前核心流程文档要求 Generate 的模型入口、运行门禁、Open Models/Edit Prompt 下一步入口、进度、取消按钮、结果预览和 Gallery handoff 提供明确可访问标签/值/提示。

## 实现步骤

1. 阅读 `GenerationView.swift` 中 `readyModels`、`selectedModel`、`selectedModelBinding` 和 `modelSection`。
2. 只修改 `readyModels` 非空分支中的 Model Picker 语义。
3. 给 Picker 增加稳定 accessibility label/value/hint。
4. 如需 helper，限制在 `GenerationView` 内，避免跨文件抽象。
5. 保留 `Picker("Model", selection: selectedModelBinding)`、`ForEach(readyModels)` 和 `Text(model.name).tag(Optional(model.id))` 行为不变。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 语义事实。
