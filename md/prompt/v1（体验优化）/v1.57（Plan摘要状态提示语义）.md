# v1.57 / Plan 摘要状态提示语义

## 版本号

v1.57

## 版本分配依据

- v1.53 至 v1.56 已为 Plan Availability、Mac readiness、Capability Matrix 和 Entitlement Rules rows 增加行级 VoiceOver hint。
- 本轮继续补齐 Plan 顶部 summary rows 的行级 hint，使 Current Build 与 Platform Status 的核心状态也具备明确提示语义。

## 背景

Plan 页的 `PlanStatusSummaryRow` 用于 Current Build 和 Platform Status 中的简短状态摘要：Plan、StoreKit products、iPhone / iPad、Mac Catalyst。当前 summary rows 已提供 accessibility label/value，但没有 hint 说明这些状态对当前 build 的含义。相比已有的 detailed rows，summary rows 是用户进入 Plan 页时最先接触的状态入口，适合补充简短解释。

## 目标

- 为 `PlanStatusSummaryRow` 增加可选 `accessibilityHint`。
- 为四个现有 summary row 填入 hint：
  - Plan：说明当前 build 使用 Local plan。
  - StoreKit products：说明当前没有配置 StoreKit products 或购买流程。
  - iPhone / iPad：说明当前 iOS/iPadOS target 可用。
  - Mac Catalyst：说明 Mac Catalyst 当前未启用。
- 保持所有可视文案、状态、图标、顺序、布局和业务行为不变。

## 非目标

- 不接入 StoreKit。
- 不新增 product IDs、purchase/restore/receipt/subscription、entitlement persistence 或 paid gates。
- 不启用 Mac Catalyst。
- 不修改 Xcode project、native XCFramework、native bridge、SwiftData schema、文件存储、推理流程或 CI workflow。
- 不修改 `PlanStatusRow`、Capability/Entitlement/Availability/Mac readiness 数据源或现有 detailed row hints。
- 不新增 simulator VoiceOver 自动化、StoreKit 测试、Mac build 或真实购买测试。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，Plan 页面在 `LocalDiffusion/Views/RootContentView.swift`。
- `PlanStatusSummaryRow` 是 Current Build 和 Platform Status 的共享 summary wrapper，已统一提供 label/value。
- 为 summary row 添加可选 hint 只影响调用方显式传入的四个 summary rows，不改变 detailed `PlanStatusRow` 的行为。

## 实现步骤

1. 在 `PlanStatusSummaryRow` 增加 `accessibilityHint: String? = nil` 或等价默认参数。
2. 在 summary row body 中仅当 hint 非空时追加 `.accessibilityHint(...)`，保持现有 label/value。
3. 为 Current Build 的 Plan、StoreKit products 和 Platform Status 的 iPhone / iPad、Mac Catalyst 四个 row 填入简短 hint。
4. 不改 title/status/systemImage、row 顺序、layout、StoreKit facts 或 Mac facts。
5. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
