# v1.11（Prompts 控件可读性）

## 版本号

v1.11

## 版本分配依据

v1.8、v1.9、v1.10 已分别优化 Plan、Generate、Models 的 Dynamic Type 和控件可访问性。当前 Prompt Library 仍有模板行 icon-only 按钮只用 `Image`，分类菜单触发器也只显示 `Image`；模板名称、prompt 文本、编辑/加载按钮和参数 pill 在大字号下集中在横向 `HStack` 中。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升 Prompts 控件语义和大字号布局，不改模板业务行为。

## 背景

Prompts 是保存、分类、编辑和加载提示词模板的入口。这里的 Load/Edit/Category actions 是高频操作，按钮必须有稳定文本 label，且在 accessibility Dynamic Type 下不应挤压模板名称、prompt 摘要和参数 pill。`swiftui-pro` accessibility 规则要求图标按钮必须包含文本 label，即使视觉上继续使用 icon-only 样式；Apple 最小点击区域应至少 44x44。

## 目标

- 在 `PromptLibraryView.swift` 中只优化 Prompts 页展示层。
- `PromptCategoryHeader` 的分类操作菜单触发器改为带文本的 `Label("Category Actions", systemImage: "ellipsis.circle")`，视觉上可继续 icon-only。
- `PromptTemplateRow` 的编辑和加载按钮改为带文本的 `Label("Edit Template", ...)` 和 `Label("Load Template", ...)`，不能只用 `Image` 加 `accessibilityLabel`。
- 行内小按钮和菜单触发器保留 Sci-Fi 风格，并补足至少 44x44 的可点击区域。
- `PromptCategoryHeader` 在 accessibility Dynamic Type 下将分类标题和菜单按钮纵向排列，普通字号保留横向结构。
- `PromptTemplateRow` 在 accessibility Dynamic Type 下将模板文案、操作按钮和参数 pill 纵向排列，普通字号保留当前紧凑结构。
- 保持模板加载、编辑、添加、删除、分类重命名、分类清空、搜索和保存行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不接入 StoreKit，不新增 product ID、价格、SKU、购买、恢复、receipt、subscription 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、native XCFramework、CI workflow 或 Xcode project。
- 不修改 `GenerationViewModel`、SwiftData schema、`PromptTemplate` 模型、文件存储、native bridge、Generate、Models、Gallery 或 Plan。
- 不改共享 `SciFiStatusPill`、`SciFiSecondaryButtonStyle` 的全局行为，避免扩大影响到其他页面。
- 不改变 `generation.load(template:)`、`editingTemplate`、`renameCategory`、`clearCategory`、template delete 或 editor save 的调用路径。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- Prompts 属于 UI 层，只负责展示、输入、导航和用户反馈。
- 模板数据由 SwiftData `PromptTemplate` 保存；本轮不得改 schema 或保存语义。
- Prompt 模板加载通过 `GenerationViewModel.load(template:)` 回填生成参数，本轮不得改变跨页回填行为。
- `GenerationView`、`PlanView` 和 `ModelLibraryView` 已使用 `dynamicTypeSize.isAccessibilitySize` 做大字号回退，本轮可在 Prompts 行内局部复用这个布局策略。
- SwiftUI 设计要求：支持 Dynamic Type；图标按钮必须包含文本 label；菜单触发器应使用 `Menu("...", systemImage:)` 或 `Label`；点击区域不小于 44x44。

## 实现步骤

1. 阅读必读文件、本提示词、`PromptLibraryView.swift`、共享 Sci-Fi 样式和 v1.8-v1.10 的 Dynamic Type 布局模式。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `PromptCategoryHeader` 和 `PromptTemplateRow` 需要处读取 `@Environment(\.dynamicTypeSize)`。
4. 将 `PromptCategoryHeader` 的 `Menu` label 从 `Image(systemName:)` 改为 `Label("Category Actions", systemImage: "ellipsis.circle")`，继续 `.labelStyle(.iconOnly)`，并设置最小 44x44 命中区。
5. `PromptCategoryHeader` 普通字号保持横排；accessibility Dynamic Type 下改为标题和菜单纵向堆叠，避免标题与菜单挤压。
6. 将 `PromptTemplateRow` 的编辑和加载按钮改为带文本 `Label`，继续 `.labelStyle(.iconOnly)`，并设置最小 44x44 命中区。
7. `PromptTemplateRow` 普通字号保持当前标题/摘要与按钮横排；accessibility Dynamic Type 下改为模板文案、操作按钮纵向排列。
8. 将参数 pill group 抽成局部 helper：普通字号横排；accessibility Dynamic Type 下纵向排列，避免 sampler/尺寸 pill 挤压。
9. 保持 `onEdit`、`onLoad`、`generation.load(template:)`、分类 rename/clear、delete、search 和 editor save 行为不变。
10. 同步文档：
    - `README.md`：说明 Prompt Library 控件有可访问文本 label，并在 accessibility Dynamic Type 下堆叠。
    - `md/flow/flow.md`：更新 Prompts UI 职责。
    - `md/flow/flowchart.md`：更新 Prompts 节点。
    - `update_log.md`：追加 v1.11 记录。
