# v1.73 / Mac 就绪 QA 状态

## 版本号

v1.73

## 版本分配依据

- v1.72 已完成 Plan compact footer note 上下文语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续优化 Mac readiness 表达，让 Mac 未来支持的 QA 阻塞项在状态 badge 上更准确地表达“仍需验证”，而不是泛化为 “Planned”。

## 背景

Plan 的 Mac Readiness checklist 用于说明当前 iOS target 不 shipping Mac/Catalyst app，Mac 支持仍需平台配置、native slice、窗口/sidebar/keyboard/pointer QA 和签名分发决策。其中 `Window and sidebar QA` 的 detail 已写明需要 smoke coverage，但状态 badge 当前显示 `Planned`，容易弱化它仍是待验证阻塞项。

## 目标

- 将 Mac Readiness 中 `.planned` 状态的可见 title 从 `Planned` 改为 `Needs QA`。
- 保持 `Window and sidebar QA` 的 title、detail、icon、accessibility hint 和数据结构不变。
- 保持 Mac Catalyst 未启用、Mac app not enabled、native backend slice 未配置等事实不变。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode target/platform、signing、native backend slice、XCFramework、workflow、SwiftData、文件存储或推理链路。
- 不修改 StoreKit product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不修改 Mac readiness item 数组结构、row layout、Plan 页面导航或可见文案中除该状态 title 以外的内容。
- 不新增截图验收、simulator smoke、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- `MacReadinessStatus` 只服务 Plan 页面 Mac readiness rows。
- `MacReadinessStatus.planned` 当前用于 `Window and sidebar QA` 这一项，状态 title 会进入 `PlanStatusBadge` 和 row accessibility value。
- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `MacReadinessStatus`、`macReadinessItems`、`MacReadinessRow` 和 `PlanStatusBadge`。
2. 只将 `MacReadinessStatus.title` 中 `.planned` 对应字符串从 `Planned` 改为 `Needs QA`。
3. 不改 enum case 名称、status color/icon、Mac readiness item detail、accessibility hint、Plan layout 或任何业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
