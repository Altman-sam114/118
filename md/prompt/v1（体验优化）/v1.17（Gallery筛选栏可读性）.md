# v1.17（Gallery筛选栏可读性）

## 版本号

v1.17

## 版本分配依据

v1.12 已优化 Gallery 图块可读性，v1.16 已优化 Gallery 图片详情页参数和操作区可读性。当前 Gallery 左侧筛选栏仍使用裸 `Label` 展示 All Images、Folders 和 Tags，iPad embedded wide filter rail 固定宽度，folder/tag 名称和当前筛选提示在 accessibility Dynamic Type 下容易横向挤压，筛选行也缺少明确的 VoiceOver label/value/hint。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升 Gallery 筛选栏可读性和辅助功能语义，不改变筛选、排序、folder/tag、图片网格、详情、复用或删除业务行为。

## 背景

Gallery 是生成结果管理与复用入口。图块与详情页已经完成 Dynamic Type-friendly 改造，但筛选栏仍是 Gallery 宽屏体验中的薄弱点。iPad regular 下 filter rail 长时间常驻，用户需要通过 folder/tag 快速定位结果；如果筛选行在大字号下被截断，或 VoiceOver 只读出一段裸文本，筛选入口会降低可用性。本轮应只改善 `filterList` 展示层和工具栏语义，让筛选入口更容易扫描和朗读。

## 目标

- 只优化 `LocalDiffusion/Views/Gallery/GalleryView.swift` 中 Gallery filter rail / filter list 的 UI 展示层。
- 将 All Images、Folder、Tag 筛选行从裸 `Label` 改为项目风格的可读筛选行或等价 helper。
- 所有筛选行保持真实文本 label 和 SF Symbol，不做纯图标按钮。
- 筛选行保持至少 44pt 触控高度。
- accessibility Dynamic Type 下 folder/tag 名称允许换行，减少横向挤压。
- iPad embedded wide filter rail 在 accessibility Dynamic Type 下小幅加宽，普通字号保持现有密度。
- 筛选行补足明确 accessibility label/value/hint，区分 all/folder/tag filter。
- 当前筛选 toolbar label 在 accessibility Dynamic Type 下可读，并提供明确辅助功能语义。
- Refresh Gallery、Sort、New Folder 维持原 toolbar placement 和行为，可补充明确 accessibility hint。
- 保持当前暗色 Sci-Fi list row 视觉和 `sciFiListRow()` 风格。
- 保持 filter selection、sort selection、folder rename/delete、tag filter、refresh reconcile 和 new folder 行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不修改 `GalleryFilter`、`GallerySort`、`visibleImages` 筛选和排序逻辑。
- 不修改 `GeneratedImage`、`GalleryFolder`、`LocalModel` 或任何 SwiftData schema。
- 不修改 folder rename/delete、tag parsing、folder assignment、image delete、reuse/regenerate/share、detail navigation 或 focused image 跳转行为。
- 不修改 Gallery image grid、image tile、image detail、Generate、Models、Prompts、Plan、Root 导航或 shared ParameterEditor。
- 不接入 StoreKit，不新增购买、恢复、receipt、subscription、product ID 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 Xcode project 平台设置、native XCFramework、CI workflow。
- 不把 `List(selection:)` 改为自定义全量筛选控件。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- Gallery 属于 UI 展示层，筛选状态由 `@State private var filter: GalleryFilter` 维护。
- `visibleImages` 根据 `filter` 和 `sort` 计算可见图片，本轮不得改变过滤条件、排序条件或比较逻辑。
- `filterList` 使用 `List(selection: filterBinding)` 和 `.tag(GalleryFilter...)` 实现选择，本轮必须保留 `.tag(...)` selection 语义。
- Folder 行同时带有 `swipeActions` 和 `contextMenu`，本轮新增 helper 必须是纯展示 view，不得嵌套 Button/Menu 造成交互冲突。
- v1.9-v1.16 已用 `dynamicTypeSize.isAccessibilitySize` 做局部大字号布局回退，本轮可沿用该策略。

## 实现步骤

1. 阅读必读文件、本提示词、`GalleryView.swift` 中 `filterList`、`embeddedWideLayout`、`selectedFilterLabel`、`imageGridToolbar`、`newFolderToolbarItem`、`visibleImages`、folder rename/delete 相关代码。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `GalleryView` 内新增只读统计 helper：
   - All Images 使用 `images.count`。
   - Folder 行可显示该 folder 下图片数量。
   - Tag 行可显示该 tag 对应图片数量。
   - 统计只用于展示和 accessibility value，不改变筛选结果。
4. 新增局部 `GalleryFilterRow` 或等价 helper：
   - 参数包含 title、systemImage、kind、count。
   - 普通字号下保持紧凑单行。
   - accessibility Dynamic Type 下允许 title 换行。
   - 使用 `Label` 保持系统图标和文本语义。
   - `.frame(maxWidth: .infinity, minHeight: 44, alignment: .leading)`。
   - `.accessibilityElement(children: .ignore)`。
   - `.accessibilityLabel(...)` 明确为 All Images filter / Folder filter / Tag filter。
   - `.accessibilityValue(...)` 包含名称和图片数量。
   - `.accessibilityHint(...)` 说明会筛选图库图片。
5. 用该 helper 替换 `filterList` 中 All Images、folder、tag 的裸 `Label`，但保留 `.tag(...)`、`.sciFiListRow()`、folder `swipeActions` 和 `contextMenu` 位置与动作。
6. `embeddedWideLayout` 中 filter rail 宽度在 `dynamicTypeSize.isAccessibilitySize` 下小幅加宽，普通字号保持 280 左右。
7. `selectedFilterLabel` 补足可读 line limit 和 accessibility label/value。
8. Refresh Gallery、Sort、New Folder 控件补足明确 accessibility hint，但不改变 placement、selection、action。
9. 同步文档：
   - `README.md`：说明 Gallery filter rail / folder/tag filters 支持 Dynamic Type-friendly 可读布局与辅助功能语义。
   - `md/flow/flow.md`：更新 Gallery UI 职责和用户入口。
   - `md/flow/flowchart.md`：更新 Gallery 节点说明。
   - `update_log.md`：追加 v1.17 记录。

- All Images、folder、tag filter row 不再只是裸 `Label`，具备可读 helper 或等价布局。

- `.tag(GalleryFilter...)` selection、folder swipe/context menu、folder rename/delete、tag filter、sort、refresh reconcile、新建 folder、图片网格和详情导航行为保持不变。
- 不新增 StoreKit、Mac Catalyst、Xcode project、native backend、SwiftData schema、文件存储或 CI workflow 改动。
- 本地轻量检查和 Swift parse 通过；建议的 iPhoneOS build 通过或明确记录沙箱限制。
