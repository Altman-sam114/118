# v1.84 / Plan Mac Readiness footer 文案

## 版本号

v1.84

## 版本分配依据

- v1.83 已完成 Plan Availability `Mac app` 可用性文案优化，并通过 `origin/main` 云端结果包验收。
- 本轮继续沿 Mac/iPad 体验透明度做小步优化，聚焦 Plan `Mac Readiness` footer 中仍残留的旧 `This iOS target...` 表述。

## 背景

Plan 的 Mac Readiness 区说明未来 Mac build 的阻塞项。v1.83 已把 Availability 中 `Mac app` row 改成正面说明当前 iPhone/iPad app 可用，同时 Mac/Catalyst app 未启用。但 Mac Readiness compact section footer 和 iPad regular panel footer 仍使用 `This iOS target does not currently ship a Mac or Catalyst app.`，语气与 v1.83 不一致，也没有正面说明当前 iPhone/iPad app 可用。

## 目标

- 将 Mac Readiness compact section footer 和 iPad regular panel footer 改为 `These are blockers for a future Mac build. The current iPhone and iPad app is available; a Mac/Catalyst app is not enabled.`。
- 保持 Mac Readiness 的 title、rows、status、icons、layout 和业务逻辑不变。
- 保持 Platform Status、Availability rows、Mac app Not enabled 状态和 Mac/Catalyst 未启用事实不变。
- 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode platform、target、scheme、signing、notarization、distribution setup 或 native Mac/Catalyst slice。
- 不修改 StoreKit、购买、entitlement persistence、paid gate、SwiftData schema、文件存储、workflow 或推理链路。
- 不修改 Mac Readiness rows、status token、icon、layout、Platform Status、Availability rows 或除 Mac Readiness footer 以外的可见与可访问文案。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- iPad regular 的 Mac Readiness footer 通过 `PlanPanel("Mac Readiness", footer: ...)` 渲染，并作为 `Mac Readiness note` 暴露给 VoiceOver。
- compact 的 Mac Readiness footer 通过 `compactSectionFooter("Mac Readiness", ...)` 渲染，并作为同名 section note 暴露给 VoiceOver。
- footer 是静态 UI 文案，不触碰状态层、SwiftData、文件存储、native backend 或 StoreKit。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `regularDetailColumn`、`macReadinessSection` 和 `compactSectionFooter`。
2. 只修改 Mac Readiness footer 的两处相同字符串。
3. 不改 Mac Readiness rows、Platform Status、Availability rows、Plan data structures 或业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
