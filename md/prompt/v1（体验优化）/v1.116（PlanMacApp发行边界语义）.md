# v1.116 / Plan Mac App 发行边界语义

## 版本号

- 版本：`v1.116`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.115 / Plan Purchase UI 恢复订阅入口边界语义`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- 本轮转向 Mac/iPad 方向，继续在 Plan 页面中提高平台状态透明度，只做可验收的 Availability 文案与 VoiceOver 语义优化，不启用 Mac Catalyst 或改变 Xcode 平台配置。

## 背景

Plan 页面 `Mac app` 行当前说明 iPhone/iPad app 可用、Mac/Catalyst app 未启用。为了避免用户误解当前 build 已经包含隐藏的 Mac 产物、Catalyst entitlement 或桌面发行路径，本轮需要让 Availability 的 `Mac app` 行明确排除 separate Mac binary、Catalyst entitlement set 和 desktop distribution channel。

## 目标

1. 只增强 `PlanView.availabilityItems` 中 `Mac app` 行的 `detail`。
2. 同步增强同一行的 `accessibilityHint`。
3. 明确当前没有 separate Mac binary、Catalyst entitlement set 或 desktop distribution channel。
4. 保持该行 title、status、icon、排序、row renderer 和所有业务逻辑不变。

## 非目标

- 不启用 Mac Catalyst、separate Mac target、separate Mac binary、Catalyst entitlement set、desktop distribution channel、signing、sandboxing、notarization 或 Mac release channel。
- 不修改 Plan overview、Current Build、Capability Matrix、Entitlement Rules、Availability 其他行、Mac Readiness、Platform Status 或 navigation。
- 不启用 StoreKit、purchase UI、restore flow、receipt validation、subscriptions、entitlement persistence、paid gate、feature flags 或 unlock logic。
- 不修改 SwiftData schema、native backend、模型下载、生成、Gallery、Prompt Library、Models、workflow 或测试规范。

## 当前架构依据

- `PlanView` 位于 `LocalDiffusion/Views/RootContentView.swift`，属于 SwiftUI UI 展示层。
- `PlanAvailabilityItem.detail` 会由 `AvailabilityRow` 传入 `PlanStatusRow` 并组成 row-level accessibility label。
- `PlanAvailabilityItem.accessibilityHint` 会追加为该行 VoiceOver hint。
- 本轮只改静态展示文案和辅助功能语义，不触碰状态层、数据层、文件存储、native backend、StoreKit 或平台配置。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `PlanView.availabilityItems`、`AvailabilityRow` 和 `PlanStatusRow`。
2. 只修改 `Mac app` item：
   - detail 继续说明 iPhone/iPad app 可用、Mac/Catalyst app 未启用，并补充没有 separate Mac binary、Catalyst entitlement set 或 desktop distribution channel。
   - accessibilityHint 同步说明 Mac support 仍需要 platform settings、native Mac/Catalyst backend slice、signing/distribution decisions、dedicated UI validation，并且当前没有上述 Mac/Catalyst 发行产物或权限集。
3. 不修改该 item 的 title、status、systemImage、数组顺序或 row renderer。
4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
5. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
