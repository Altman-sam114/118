# v1.131 / Gallery 宽屏筛选回退

## 版本号

- 版本：`v1.131`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.130 / Gallery 图块 Prompt 可读性`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.129-v1.130 已连续改善 Gallery 图块在 iPad 分栏和窄 tile 下的可读性；本轮继续优化 Gallery 在 iPad Root detail 的 embedded wide 布局，让 filter rail 在窄 regular 宽度下有可用回退。

## 背景

`GalleryView.embeddedWideLayout` 当前始终使用横向 `HStack`：左侧固定宽度 filter rail，右侧 image grid。该布局避免了 iPad Root detail 中的嵌套 split，但在 Stage Manager、Split View 或较窄 regular 宽度下，固定 filter rail 会继续占据横向空间并挤压 grid。Gallery 仍需要保留筛选能力，因此本轮不直接隐藏筛选栏，而是用 `ViewThatFits(in: .horizontal)` 在空间足够时保留横向 rail，空间不足时回退为顶部 filter shelf + 下方 grid。

## 目标

1. 只修改 `GalleryView` 的 embedded wide 内容布局。
2. 新增 `embeddedGalleryContent`，使用 `ViewThatFits(in: .horizontal)` 在两个候选之间选择：
   - `embeddedGallerySplitContent`：既有左侧 filter rail + divider + image grid 横向布局。
   - `embeddedGalleryStackedContent`：顶部可滚动 filter shelf + divider + image grid 纵向布局。
3. 横向候选通过 `embeddedWideMinimumWidth` 给出合理最小宽度，让窄 regular 宽度可选择 stacked fallback。
4. 保持 standalone Gallery 内部 split、filter row 内容、filter selection、Sort/Refresh/New Folder toolbar、image grid、detail navigation、SwiftData 和文件存储逻辑不变。

## 非目标

- 不修改 Gallery 图块 thumbnail、prompt、metadata、accessibility label/value/hint 或 grid column minimum。
- 不修改 Gallery detail、folder/tags、删除确认、Refresh Gallery、Sort menu、folder editor 或 image reconciliation。
- 不修改 Root navigation、TabView、NavigationSplitView、Generate、Models、Prompts、Plan、SwiftData schema、AppFileStore、GenerationViewModel、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GalleryView.swift` 位于 `LocalDiffusion/Views/Gallery/`，属于 SwiftUI UI 展示层。
- Root 在 iPad detail 中以 `.embeddedWide` 模式嵌入 Gallery，用单层 split 避免嵌套 NavigationSplitView。
- Gallery filter state、sort state、folder editor、detail path 和 generated image data flow 都在 `GalleryView` 内部；本轮只调整 embedded mode 的布局容器，不改变状态或数据。

## 实现步骤

1. 阅读 `embeddedWideLayout`、`filterList`、`imageGrid`、`filterRailWidth` 和 `imageGridMinimumWidth`。
2. 将 `embeddedWideLayout` 中原来的 `HStack(spacing: 0)` 提取为 `embeddedGalleryContent`。
3. 新增：

```swift
private var embeddedGalleryContent: some View {
    ViewThatFits(in: .horizontal) {
        embeddedGallerySplitContent
            .frame(minWidth: embeddedWideMinimumWidth)
        embeddedGalleryStackedContent
    }
}
```

4. `embeddedGallerySplitContent` 保留既有 filter rail、vertical divider 和 image grid。
5. `embeddedGalleryStackedContent` 使用 `VStack(spacing: 0)`，顶部显示 `filterList`，隐藏 scroll background，使用 `SciFiTheme.panelSoft` 背景和 `filterShelfHeight`，下方用 horizontal divider 分隔 image grid。
6. 新增 `filterShelfHeight` 和 `embeddedWideMinimumWidth`，分别随 accessibility Dynamic Type 和现有 filter/grid 宽度调整。
7. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 embedded Gallery 在窄 regular 宽度下可从左侧 filter rail 回退为顶部 filter shelf。
8. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
