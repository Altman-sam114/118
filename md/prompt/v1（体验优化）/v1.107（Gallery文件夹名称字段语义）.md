# v1.107 Gallery 文件夹名称字段语义

## 版本号

- 版本：`v1.107`
- 阶段：`v1（体验优化）`
- 任务名：Gallery 文件夹名称字段语义

## 版本分配依据

- 最新已完成版本为 `v1.106`，已补齐 Models 未跟踪 GGUF 导入编辑器 Display name 和 Family 字段级 VoiceOver 语义。
- 本轮转向 Gallery 的 folder create/rename 编辑器，继续做小步 UI/可访问性优化，让 folder name 输入框本身暴露明确 label、value 和下一步提示，递增为 `v1.107`。

## 背景

`FolderNameEditor` 当前 toolbar 的 Save / Cancel 已有 ready/name-required 和 close-without-saving 语义，但 `TextField("Folder name", text: $name)` 本身没有字段级 accessibility value/hint。VoiceOver 用户进入新建或重命名 folder 表单时，不容易直接知道当前草稿名称、空值状态，以及保存前必须填写 folder name。本轮只补输入框语义，不改变 folder create、rename、delete、filter、sort 或 SwiftData 行为。

## 目标

1. 给 `FolderNameEditor` 的 `TextField("Folder name", text: $name)` 增加明确 accessibility label。
2. value 区分空值和当前 trimmed name，空值读作 `No folder name`。
3. hint 说明保存前必须输入 folder name，并且该名称会用于 Gallery folder。
4. 保留现有 Save/Cancel toolbar 语义、Save disabled gate、trim 后保存、dismiss 和可见 UI 不变。

## 非目标

- 不修改 folder create、rename、delete、filter、sort、selection 或 image-folder assignment 行为。
- 不修改 `GalleryFolder` SwiftData schema、file storage、tag editing、image detail、Gallery grid、filter rail 或 sort menu。
- 不修改 Models、Prompt、Generate、Plan、StoreKit、Mac Catalyst、native backend、Xcode project 或 workflow。
- 不新增 simulator、真机 VoiceOver 或完整本地 build 要求。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，本轮只改 SwiftUI 表单语义。
- `FolderNameEditor` 是 Gallery folder 新建/重命名的局部编辑器，适合承载字段级 label/value/hint。
- Save toolbar 已负责 ready/name-required 状态；输入框补充当前值和用途语义即可，不需要改数据流。
- 测试策略按 `md/test/test.md`：SwiftUI 源码变更需跑本地轻量检查、普通 Swift parse、native bridge Swift parse，并通过 `main` push 触发 GitHub Actions 结果包。
