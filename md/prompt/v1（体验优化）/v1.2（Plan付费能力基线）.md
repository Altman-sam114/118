# v1.2（Plan 付费能力基线）

## 版本号

v1.2

## 版本分配依据

v1.0 和 v1.1 已完成 iPad Gallery 与 Generate 的宽屏体验基线。人工总目标还包含“优化付费功能”。当前项目没有 StoreKit、订阅、entitlement、账号或 Settings/Plan 入口。直接实现真实购买需要 App Store Connect 产品 ID、商业规则和人工决策，因此本轮选择一个可验收的小步：建立 Plan/付费能力信息架构基线，为后续 StoreKit 接入留出清晰 UI 和架构边界。

## 背景

Local Diffusion 当前核心入口是 Generate、Models、Gallery、Prompts。付费能力若后续进入产品，应该有一个清楚的位置展示当前本地计划、未来可升级能力和 StoreKit 状态，但本轮不得伪造付费成功、不得让用户误以为已经可以购买。

## 目标

- 在 Root 导航中新增 `Plan` 入口。
- 新增一个简洁暗色科幻风的 Plan 页面，展示：
  - 当前计划是本地 Local 体验。
  - 当前构建未配置 StoreKit 产品。
  - 后续可纳入付费层的能力清单，但不启用购买。
- 保持 iPhone compact 和 iPad regular 都能访问 Plan。
- 不接入真实支付、不新增 StoreKit 依赖、不新增 SwiftData schema。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不实现真实购买、订阅恢复、收据校验、App Store Server API、账号系统或 entitlement 持久化。
- 不限制现有 Generate、Models、Gallery、Prompts 功能。
- 不新增 macOS/Catalyst target。
- 不修改 native backend、文件存储、下载流程或生成流程。
- 不创建假“Upgrade”成功态，不伪造用户已付费。

## 当前架构依据

- `RootContentView` 管理顶层入口，新增 Plan 应在这里接入。
- UI 层只负责展示计划状态和后续付费能力边界，不承担购买业务逻辑。
- 当前 Xcode project 是显式文件引用；为避免本轮扩大到 project 文件维护，可把小型 Plan view 放在现有 `RootContentView.swift` 中，后续若页面变复杂再拆文件并同步 project/workflow。
- Swift 源码变更至少运行两条 Swift parse，push 后由 CI 结果包重验证。

## 实现步骤

1. 阅读必读文件、本提示词、`RootContentView.swift` 和 `md/test/test.md`。
2. 确认当前分支和工作区：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short`
3. 在 `AppSection` 增加 `.plan`，标题建议为 `Plan`，图标建议为 `creditcard` 或等价 SF Symbol。
4. 将 `sections` 增加 Plan，确保 iPhone TabView 与 iPad sidebar 都可进入。
5. 在 `sectionContent` 增加 `PlanView()`。
6. 新增小型 `PlanView` 和必要 row/subview：
   - 使用 `NavigationStack + Form`。
   - 使用 `SciFiTheme`、`.sciFiScreen()`、`.bottomTabBarClearance()` 和现有 8px 风格。
   - 显示当前 Local plan、StoreKit 未配置状态、未来付费能力清单。
   - 不提供可点击的假购买按钮；如有按钮必须明确 disabled 或仅为后续占位。
7. 同步文档：
   - `README.md` Current implementation 增加 Plan/付费能力基线。
   - `md/flow/flow.md` 更新用户入口和 UI 层职责。
   - `md/flow/flowchart.md` 更新 SwiftUI 页面列表。
   - `update_log.md` 追加 v1.2 记录。

## 验收标准

- Root 导航包含 Plan，iPhone TabView 和 iPad sidebar 均可进入。
- Plan 页面明确当前是 Local plan，且当前构建未配置 StoreKit 产品。
- 页面不提供真实购买入口，不伪造付费状态。
- 现有 Generate、Models、Gallery、Prompts 行为不变。
- 本地轻量检查和两条 Swift parse 通过。
- `origin/main` 最新 v1.2 run 成功，Agent C 下载并核对结果包通过。

## 风险和禁止项

- 禁止用占位 UI 冒充真实 StoreKit 购买能力。
- 禁止新增收费限制导致现有免费功能被锁。
- 禁止新增 SwiftData schema 或 entitlement 持久化。
- 禁止把 macOS/Catalyst 支持混入本轮。
- 禁止跳过 Agent C artifact 核对。
