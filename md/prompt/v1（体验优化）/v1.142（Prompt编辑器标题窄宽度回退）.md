# v1.142 / Prompt 编辑器标题窄宽度回退

## 版本号

- 版本：`v1.142`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.141 / Prompt 分类标题窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- Prompt list headers 已完成窄宽度回退；`PromptTemplateEditor.promptEditor` 在普通 Dynamic Type 下仍固定横排 title + character count，是下一个小而可验收的 UI 韧性优化。

## 背景

`PromptTemplateEditor.promptEditor` 的 header 在 accessibility Dynamic Type 下已纵向堆叠 title + character count，但普通 Dynamic Type 固定 `HStack { title; Spacer(); count }`。在窄 sheet / Split View 下可能挤压。本轮用 `ViewThatFits(in: .horizontal)` 对齐既有模式。

## 目标

1. 只修改 `PromptTemplateEditor.promptEditor` 中普通 Dynamic Type 的 title/count header 分支。
2. `ViewThatFits(in: .horizontal)` 优先横排，不足时回退纵向。
3. accessibility 纵向布局不变。
4. 保留 TextEditor、placeholder、character count accessibility、Save/Cancel 行为不变。

## 非目标

- 不修改 Generate 页 prompt editor、ParameterEditor、category editor、SwiftData、native、StoreKit、Mac Catalyst、workflow。
- 禁止本地 Xcode / Simulator / 真机构建测试。

## 实现步骤

普通分支改为：

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        Text(title)...
        Spacer()
        promptCharacterCount(text, title: title)
    }

    VStack(alignment: .leading, spacing: 4) {
        Text(title)...
        promptCharacterCount(text, title: title)
    }
}
```

同步 README / flow / flowchart / update_log。
