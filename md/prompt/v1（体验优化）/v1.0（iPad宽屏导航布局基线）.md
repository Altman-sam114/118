# v1.0（iPad 宽屏导航布局基线）

## 版本号

v1.0

## 版本分配依据

v0.x 已完成多 Agent 治理、`main` 直推、CI 结果包、native Release asset 恢复和 checksum 校验。人工总目标要求持续优化 mac 版、iPad 版、UI、设计和付费功能。本轮进入第一个产品体验阶段，但当前工程事实仍是 iOS/iPadOS：`SUPPORTED_PLATFORMS = "iphoneos iphonesimulator"`，`SUPPORTS_MACCATALYST = NO`。因此 v1.0 先选择可小步验收的 iPad 宽屏导航和 Gallery 布局基线，不同时开启 macOS/Catalyst 或付费功能。

## 背景

当前 `RootContentView` 在 regular size class 使用顶层 `NavigationSplitView`。`GalleryView` 内部也使用自己的 `NavigationSplitView`，当用户在 iPad 上从 Root sidebar 进入 Gallery 时，会出现 split view 嵌套 split view 的结构风险。这个问题会影响宽屏信息架构、返回行为、toolbar 位置和 Gallery 详情打开路径，是后续 mac/Catalyst 和付费功能入口设计前应先稳定的基础体验。

相关事实：

- `RootContentView` 负责 Generate、Models、Gallery、Prompts 四个全局入口。
- iPhone compact 使用 `TabView`，iPad regular 使用 `NavigationSplitView`。
- `GalleryView` 当前同时承担 folder/tag sidebar、image grid detail 和 image detail navigation。
- `focusedGalleryImageID` 从 Generate 跳到 Gallery 后需要继续打开对应图片详情。
- UI 必须保持简洁、暗色、明显科幻风格。

## 目标

- 消除 iPad regular 布局中的 Root split + Gallery split 嵌套问题。
- 保留 iPhone compact 下 Gallery 的现有两栏/列表筛选体验，不破坏 TabView。
- 在 iPad regular 下，让 Gallery 在 Root detail 内呈现单层内容：筛选入口、图片网格、详情 navigation 应在同一个 detail 层级内工作。
- 保留 `focusedGalleryImageID` 自动打开新生成图片详情的行为。
- 保留 folder/tag filter、sort、refresh、folder add/rename/delete、image detail reuse/regenerate/delete/share 等既有功能。
- 继续使用现有 Sci-Fi theme，不引入浅色、默认 material 或营销式 UI。

## 非目标

- 不启用 Mac Catalyst，不新增 macOS target，不修改 `SUPPORTED_PLATFORMS` 或 `SUPPORTS_MACCATALYST`。
- 不实现付费功能、订阅、StoreKit、账号系统或 entitlement。
- 不重构 SwiftData schema、文件存储、native backend 或生成流程。
- 不修改 `ImageGenerationBackend`、native bridge 或 XCFramework。
- 不做大范围视觉重设计，不拆分大型文件超过本轮需要。
- 不创建 PR，不引入 `develop`、`smalldata_test` 或其他分支流程。

## 当前架构依据

- UI 层只能展示、输入、导航和反馈，不直接处理 native bridge。
- Gallery 删除图片仍必须通过 `AppFileStore` 和 SwiftData 元数据同步。
- `GenerationViewModel.load(image:)` 是 Gallery 复用参数到 Generate 的既有路径。
- Root 负责跨页面 selection 切换，Gallery 不应绕过 Root 做全局导航。
- Swift 源码变更至少需要本地 Swift parse；push 后云端 CI 还会运行 Swift parse、native preflight 和 iPhoneOS build。

## 实现步骤

1. 按 `AGENTS.md` 顺序读取必读文件、本提示词、`RootContentView.swift`、`GalleryView.swift`、`ParameterEditor.swift` 和 `md/test/test.md`。
2. 确认当前分支是 `main`、upstream 是 `origin/main`、工作区干净：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short`
3. 在 `RootContentView` 中继续用 `horizontalSizeClass == .regular` 选择 iPad sidebar layout，不修改 iPhone TabView。
4. 为 `GalleryView` 增加布局模式或等价参数，例如：
   - compact / standalone：保留当前内部 `NavigationSplitView`，供 iPhone TabView 使用。
   - embeddedWide / rootDetail：在 Root regular detail 中避免内部 `NavigationSplitView`，改用单层 `NavigationStack` + 内容区域。
5. 在 Root regular 的 Gallery 分支传入宽屏嵌入模式；compact TabView 继续使用原有模式。
6. 宽屏嵌入模式至少提供：
   - 当前 filter 的可见入口，可用 toolbar menu、segmented/menu picker、或者靠左的非嵌套 filter rail。
   - 图片 grid 和 sort/refresh 操作。
   - `NavigationStack(path: $detailPath)` 打开 `ImageDetailView`。
7. 保持 `openFocusedImageIfAvailable()` 逻辑适用于 compact 和 regular 两种 Gallery 布局。
8. 避免把 UI 直接变成浅色或系统默认灰，继续使用 `SciFiTheme`、`.sciFiScreen()`、`.sciFiPanel()` 和现有按钮样式。
9. 如新增共享 UI helper，应局限在 `GalleryView.swift` 或现有 shared 样式文件，避免无关抽象。
10. 同步必要文档：
    - `md/flow/flow.md`：更新导航与 Gallery 宽屏布局说明。
    - `md/flow/flowchart.md`：如流程图中 iPad 导航说明需要调整，做小幅同步。
    - `README.md`：Current implementation 中补充 iPad Gallery 避免嵌套 split 的宽屏布局基线。
    - `update_log.md`：追加 v1.0 记录。

## main push 和云端 CI 结果包要求

Agent B 必须在 `main` 上提交并 push：

```bash
git add LocalDiffusion/Views/RootContentView.swift LocalDiffusion/Views/Gallery/GalleryView.swift LocalDiffusion/Views/Shared/ParameterEditor.swift md/flow/flow.md md/flow/flowchart.md README.md update_log.md 'md/prompt/v1（体验优化）/v1.0（iPad宽屏导航布局基线）.md'
git commit -m "v1.0: iPad 宽屏导航布局基线"
git push origin main
```

若 `ParameterEditor.swift` 或 `flowchart.md` 未修改，不要强行暂存。提交必须只包含本轮相关文件。

预期 workflow 名称仍为 `Local Diffusion CI Results`。预期 artifact 名称：

```text
localdiffusion-ci-v1.0-main-<short_sha>-run<run_id>-attempt<run_attempt>
```

Agent C 必须下载最新 artifact 到 `/private/tmp/localdiffusion-c-review-<run_id>/`，核对：

- `ci-artifact-manifest.json` 的 `branch=main`。
- `commitSha` 等于 `origin/main` 最新 commit。
- `runId`、`runAttempt` 与下载 run 一致。
- `nativeBackendAssetOutcome=success`，expected/actual SHA 一致。
- JUnit/摘要显示 git diff、plutil、native asset、Swift parse、native backend、xcodebuild 全部通过。

## 验收标准

- iPad regular 的 Root sidebar 进入 Gallery 时，Gallery 不再嵌套自己的 `NavigationSplitView`。
- iPhone compact 的 Gallery 仍可使用原有 folder/tag sidebar + image detail flow。
- Generate 完成后 `focusedGalleryImageID` 仍能切到 Gallery 并打开对应图片详情。
- Gallery 的 filter、sort、refresh、folder 管理、图片详情、reuse、regenerate、delete、share 行为不丢失。
- UI 保持暗色科幻风，不引入浅色或默认系统灰风格。
- 本地轻量检查按要求记录；云端 v1.0 artifact 通过 Agent C 核对。

## 风险和禁止项

- 禁止把 macOS/Catalyst target 和 iPad Gallery 优化混在本轮。
- 禁止把付费功能入口、订阅、StoreKit 或账号系统混入本轮。
- 禁止 UI 直接调用 native bridge 或改动生成后端。
- 禁止删除 Gallery 现有 folder/tag/detail 功能来简化布局。
- 禁止用旧 run、旧 artifact 或本地输出替代 Agent C 最新结果包核对。
