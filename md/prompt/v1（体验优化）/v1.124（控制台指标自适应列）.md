# v1.124 / 控制台指标自适应列

## 版本号

- 版本：`v1.124`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.123 / Stepper 标签窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.120 已让 Generate iPad regular 双栏在窄宽度下回退单列，v1.121-v1.123 已收敛共享参数区窄宽度风险；本轮继续收敛 Generate 右侧控制台 metrics 在窄栏中的横向压缩风险。

## 背景

`GenerationView.consoleMetrics` 使用 `LazyVGrid(columns: consoleMetricColumns, spacing: 10)` 展示 Steps 和 Canvas 两个状态 metric。当前 `consoleMetricColumns` 在 accessibility Dynamic Type 下回退单列，但普通 Dynamic Type 下固定返回两个 `.flexible()` 列。iPad Split View、Stage Manager 或窄 regular 右侧面板中，固定两列会让每个 metric 变窄。改为 adaptive column 可以在空间足够时保持双列，在空间不足时自然回退为单列，且不改变 metric 内容、参数、生成状态或 VoiceOver 语义。

## 目标

1. 只修改 `GenerationView.consoleMetricColumns` 的普通 Dynamic Type 分支。
2. accessibility Dynamic Type 继续返回单列。
3. 普通 Dynamic Type 使用 `GridItem(.adaptive(minimum: 150), spacing: 10)`，让 Steps/Canvas metrics 根据可用宽度自适应列数。
4. 保持 `consoleMetrics` 的两个 `SciFiMetric` 内容、颜色、图标、数值来源和顺序不变。
5. 保持 Generate 双栏/单列布局、generation gate、run/progress/result、model/prompt/parameter 行为不变。

## 非目标

- 不修改 `wideGenerationLayout`、`singleColumnGenerationLayout`、`compactGenerationLayout` 或 `ViewThatFits` 外层布局。
- 不修改 `SciFiMetric` 组件、status pills、console overview、run section、result section、model picker、prompt editor、ParameterEditor 或 Gallery handoff。
- 不修改 `GenerationParameters`、`GenerationViewModel`、SwiftData、file storage、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GenerationView` 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，属于 SwiftUI UI 展示层。
- `consoleMetrics` 只展示 `viewModel.parameters.steps` 和 `width x height`，不写入状态。
- `consoleMetricColumns` 是纯布局 helper。
- `SciFiMetric` 已在 accessibility Dynamic Type 下内部纵向排列；本轮只调整 grid 列数选择。

## 实现步骤

1. 阅读 `consoleMetrics` 与 `consoleMetricColumns`。
2. 保留 `dynamicTypeSize.isAccessibilitySize` 分支返回 `[GridItem(.flexible())]`。
3. 将普通 Dynamic Type 分支从固定两个 `.flexible()` 列改为：

```swift
return [GridItem(.adaptive(minimum: 150), spacing: 10)]
```

4. 不修改 `SciFiMetric` 调用、title/value/systemImage/color。
5. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Generate 控制台 metrics 支持普通字号下自适应列数。
6. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
