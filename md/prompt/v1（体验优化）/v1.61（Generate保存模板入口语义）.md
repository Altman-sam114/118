# v1.61 Generate 保存模板入口语义

## 版本号

v1.61

## 版本分配依据

v1.60 已完成 Models toolbar refresh / add menu 的 VoiceOver 入口语义。本轮继续 Agent X 的 iPad / Mac / UI 体验优化循环，转向 Generate toolbar 中保存模板入口的状态语义。

## 背景

Generate 页面已具备 prompt 编辑器、运行门禁、取消、结果预览、Gallery handoff 和共享参数控件的可访问语义。当前 toolbar 的 `Save Template` 按钮会在 positive prompt 为空时禁用，但 VoiceOver 没有明确说明按钮当前是 ready 还是需要 prompt，也没有说明按钮会把当前 prompt 与参数保存为 Prompt Library 模板。

## 目标

- 为 Generate toolbar `Save Template` button 增加 VoiceOver value，暴露 `Ready` / `Positive prompt required` 状态。
- 为该按钮增加 VoiceOver hint，说明 ready 时会把当前 prompts 和参数保存为 Prompt Library 模板，缺 prompt 时需要先填写 positive prompt。
- 保持现有可视文案、SF Symbol、toolbar placement、disabled 条件、sheet 打开和模板保存行为不变。
- 同步更新 README、flow、flowchart 和 update_log。

## 非目标

- 不修改 PromptTemplateEditor、PromptTemplate SwiftData schema、模板保存字段、名称/分类输入、Prompt Library 加载逻辑。
- 不修改 positive / negative prompt binding、参数编辑器、生成门禁、生成/取消/结果保存、Gallery handoff。
- 不修改 SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。
- 不新增 keyboard shortcut、command menu、autosave、付费模板包或模板导出功能。

## 当前架构依据

- Generate UI 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，UI 层只负责展示、输入和用户反馈。
- `Save Template` 只设置 `showingSaveTemplate = true` 并通过现有 sheet 插入 `PromptTemplate`。
- 本轮只添加 SwiftUI accessibility modifiers 和只读辅助计算，不触碰 SwiftData 模型、生成 ViewModel、文件存储或 native backend。

## 实现步骤

1. 为 `Save Template` toolbar button 增加 `accessibilityValue(...)`。
2. 为同一 button 增加 `accessibilityHint(...)`。
3. 如需减少重复判断，可添加只读 computed properties，但必须保持 disabled 条件与当前 positive prompt trim 判断一致。
4. 不改动 Button action、Label、sheet、PromptTemplateEditor 初始化参数或保存 closure。
5. 更新 README、flow、flowchart 和 update_log。
