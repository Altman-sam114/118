# v1.141 / Prompt 分类标题窄宽度回退

## 版本号

- 版本：`v1.141`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.140 / Prompt 模板行标题窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.140 已补齐模板行 header；`PromptCategoryHeader.headerContent` 在普通 Dynamic Type 下仍固定横排 title + category actions，是下一个对称、可验收的 UI 韧性优化。

## 背景

`PromptCategoryHeader.headerContent` 在 accessibility Dynamic Type 下已纵向堆叠，但普通 Dynamic Type 固定 `HStack { title; Spacer(); categoryActions }`。分类名与 actions menu 在窄 list / iPad split 下可能挤压。本轮用 `ViewThatFits(in: .horizontal)` 对齐既有模式。

## 目标

1. 只修改 `PromptCategoryHeader.headerContent` 的普通 Dynamic Type 分支。
2. `ViewThatFits(in: .horizontal)` 优先横排 title + optional actions，不足时回退纵向。
3. accessibility 纵向布局不变。
4. 保留 uncategorized 不显示 actions、rename/clear 行为与 accessibility 语义不变。

## 非目标

- 不修改 template row、editor、search、ParameterEditor、SwiftData、native、StoreKit、Mac Catalyst、workflow。
- 禁止本地 Xcode / Simulator / 真机构建测试。

## 实现步骤

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        title
        Spacer()
        if !group.isUncategorized {
            categoryActions
        }
    }

    VStack(alignment: .leading, spacing: 8) {
        title
        if !group.isUncategorized {
            categoryActions
        }
    }
}
```
