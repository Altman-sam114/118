# v1.125 / 进度阶段文本可读性

## 版本号

- 版本：`v1.125`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.124 / 控制台指标自适应列`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.120-v1.124 已持续收敛 Generate 的 iPad regular、窄宽度和控制台可读性；本轮继续处理 Generate Run 区域中进度阶段文本在窄栏或大字号下的垂直可读性。

## 背景

`GenerationView.runSection` 在 `viewModel.isGenerating` 时使用 `ProgressView(value:)` 展示当前 `viewModel.progress.stage`。该 stage 文字来自生成流程，可能随 native backend 阶段或未来本地化变长。当前 `Text(viewModel.progress.stage)` 没有显式垂直固定尺寸，在 iPad 右侧窄栏、sheet 或较大字号下可能被父布局压缩，降低可读性。项目中其它较长说明文字已广泛使用 `.fixedSize(horizontal: false, vertical: true)`，本轮将同一策略补齐到进度阶段 label。

## 目标

1. 只修改 `GenerationView.runSection` 中 `ProgressView` label 的可见 stage 文本。
2. 给 `Text(viewModel.progress.stage)` 增加 `.fixedSize(horizontal: false, vertical: true)`。
3. 保持 `ProgressView(value:)` 的 fraction、tint、accessibility label/value/hint 不变。
4. 保持取消按钮、生成状态、progress model、generation gate、保存结果和 Gallery handoff 不变。
5. 让 Generate 在 iPad 窄栏、大字号或未来较长 stage 文案下更可读。

## 非目标

- 不修改 `viewModel.progress.stage`、`viewModel.progress.fraction`、`generationProgressAccessibilityValue` 或 `progressPercentText`。
- 不修改 `GenerationViewModel`、native backend progress callback、取消逻辑、生成保存逻辑或 SwiftData。
- 不修改 `GenerationGatePanel`、Generate button、Cancel button、result section、ParameterEditor、model picker、prompt editor、Gallery handoff、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不加 `.lineLimit(nil)`、固定高度、缩小字体或截断策略；当前默认允许多行，本轮只防止垂直压缩。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GenerationView` 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，属于 SwiftUI UI 展示层。
- `runSection` 只根据 `viewModel.isGenerating` 展示当前进度或可执行操作。
- `ProgressView` 的 `value` 使用 `viewModel.progress.fraction`，可见 label 使用 `viewModel.progress.stage`。
- 本轮只改变 label layout，不触碰状态层、数据层、文件存储或 native backend。

## 实现步骤

1. 阅读 `runSection` 中 `ProgressView(value:)`。
2. 在 `Text(viewModel.progress.stage)` 的 `.foregroundStyle(SciFiTheme.primaryText)` 后追加：

```swift
.fixedSize(horizontal: false, vertical: true)
```

3. 不添加 `.lineLimit(nil)`，不修改 `ProgressView`、accessibility modifiers 或 Cancel button。
4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Generate progress stage label 支持垂直换行可读性。
5. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
