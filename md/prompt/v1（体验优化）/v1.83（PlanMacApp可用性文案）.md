# v1.83 / Plan Mac app 可用性文案

## 版本号

v1.83

## 版本分配依据

- v1.82 已完成 Plan Capability Matrix `StoreKit purchases` 能力文案优化，并通过 `origin/main` 云端结果包验收。
- 本轮从付费能力透明度转向 Mac/iPad 体验透明度，聚焦 Plan Availability 中 `Mac app` row 的当前平台状态说明。

## 背景

Plan 的 Availability 区用于区分当前可用 Local tools、planning-only paid candidates、Purchase UI 前置条件和 Mac app 状态。`Mac app` 当前 detail 写为 `This iOS target does not currently ship a Mac or Catalyst app...`，说明了 Mac/Catalyst 未发货，但没有正面说明当前 iPhone/iPad app 是可用状态。对用户来说，更清楚的事实是：当前可用范围是 iPhone/iPad，Mac 或 Catalyst 支持还未启用。

## 目标

- 将 Plan Availability `Mac app` detail 改为 `The iPhone and iPad app is available in this build; a Mac/Catalyst app is not enabled.`。
- 将该 row 的 accessibility hint 改为 `Clarifies Mac support is separate from the current iPhone and iPad app and still requires platform settings, a native Mac/Catalyst backend slice, signing decisions, and dedicated UI validation.`。
- 保持该 row 的 title、status、icon、layout 和 Availability 数据结构不变。
- 保持 Mac Catalyst 未启用、Mac readiness blockers、native Mac/Catalyst slice 未存在、signing/notarization 未配置等事实不变。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode platform、target、scheme、signing、notarization、distribution setup 或 native Mac/Catalyst slice。
- 不修改 StoreKit、购买、entitlement persistence、paid gate、SwiftData schema、文件存储、workflow 或推理链路。
- 不修改 Availability row 顺序、status token、icon、layout 或除该 detail/hint 以外的可见与可访问文案。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- `PlanAvailabilityItem.detail` 会进入 `AvailabilityRow` -> `PlanStatusRow` 的可见说明和 accessibility label。
- `PlanAvailabilityItem.accessibilityHint` 由 `AvailabilityRow` 附加到 row 上，用于 VoiceOver 解释该可用性状态。
- 该 row 当前 status 为 `Not enabled`，表达 Mac app 当前未启用。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `availabilityItems`、`AvailabilityRow` 和 `PlanStatusRow`。
2. 只修改 `Mac app` 的 detail 与 accessibility hint。
3. 不改 title、status、icon、layout、Mac readiness facts 或任何业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。

- 禁止启用 Mac Catalyst、修改 Xcode platform、添加 native Mac/Catalyst slice、修改 signing/notarization/distribution setup、StoreKit、SwiftData schema、workflow、native backend 或 Xcode project。
- 禁止跳过 Agent C artifact 核对或用旧 run 冒充 v1.83。
