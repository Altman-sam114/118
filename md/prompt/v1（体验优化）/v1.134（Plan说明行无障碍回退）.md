# v1.134 / Plan 说明行无障碍回退

## 版本号

- 版本：`v1.134`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.133 / Plan 状态行标题徽章回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.132-v1.133 已改善 Plan summary rows 和 shared status rows 的宽度回退；本轮继续在 Plan 页面内改善 StoreKit/Mac 长说明行在 accessibility Dynamic Type 下的可读性。

## 背景

`PlanNoteRow` 当前使用 `Label` 横向展示图标和长说明文本，服务 Current Build 的 StoreKit purchase status note 和 Platform Status 的 Mac support status note。文本本身已经允许换行，但在 accessibility Dynamic Type 下，横向图标列仍会占用宽度，使长说明更容易变成窄文本柱。Plan 页面已有多个组件在 accessibility Dynamic Type 下回退纵向布局；本轮让 `PlanNoteRow` 也遵循同一规则。

## 目标

1. 修改 `PlanNoteRow` 的内部布局。
2. 普通 Dynamic Type 下保持既有横向 `Label` 视觉结构。
3. accessibility Dynamic Type 下回退为纵向 `VStack`：
   - 顶部显示同一个 system image 和 icon color。
   - 下方显示同一个 note 文本，继续使用 `.font(.callout)`、`SciFiTheme.secondaryText` 和 `.fixedSize(horizontal: false, vertical: true)`。
4. 保留 `.frame(maxWidth: .infinity, minHeight: 44, alignment: .leading)`、accessibility label/value/hint、可见文案、图标、颜色和所有 Plan 数据不变。
5. 不改变 StoreKit 未配置、paid candidates planning-only 或 Mac/Catalyst 未启用的业务边界。

## 非目标

- 不启用 StoreKit、购买按钮、restore flow、receipt validation、entitlement persistence、feature flag 或 paid unlock gate。
- 不修改 paid candidates 的业务含义、付费文案或可购买状态。
- 不修改 Mac Catalyst、Xcode project platform settings、native backend slice、signing、sandboxing、notarization 或 workflow。
- 不修改 `PlanStatusSummaryRow`、`PlanStatusRow`、Plan 数据数组、row 顺序、summary rows、status badges 或 section footer。
- 不修改 Generate、Models、Gallery、Prompts、SwiftData、AppFileStore、GenerationViewModel、native backend 或测试规范。
- 不做 simulator / Mac Catalyst / Stage Manager 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `RootContentView.swift` 中的 `PlanView` 属于 SwiftUI UI 展示层，只负责展示 Local plan、付费候选规划状态、StoreKit 边界和平台 readiness。
- `PlanNoteRow` 是 Plan 页面内部共享 UI 组件，当前用于 StoreKit purchase status 和 Mac support status 两条长说明。
- Plan 页面当前明确不请求 App Store product、不存储 entitlement、不展示购买 UI；本轮不得改变该边界。

## 实现步骤

1. 阅读 `PlanNoteRow`、`currentBuildContent` 和 `platformStatusContent`。
2. 给 `PlanNoteRow` 读取 `dynamicTypeSize` 环境。
3. 抽出：
   - `noteText`：保留当前文本样式。
   - `noteIcon`：保留当前 system image 和 icon color。
   - `horizontalRow`：保留现有 `Label` 横向布局。
   - `verticalRow`：新增 `VStack(alignment: .leading, spacing: 8)`，先显示 `noteIcon`，再显示 `noteText`。
4. `body` 中根据 `dynamicTypeSize.isAccessibilitySize` 选择 `verticalRow` 或 `horizontalRow`。
5. 保持 frame、accessibility element、label、value 和 hint 不变。
6. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Plan note rows 在 accessibility Dynamic Type 下回退纵向。
7. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
