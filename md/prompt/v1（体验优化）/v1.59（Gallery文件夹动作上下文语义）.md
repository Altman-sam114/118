# v1.59 Gallery 文件夹动作上下文语义

## 版本号

v1.59

## 版本分配依据

v1.58 已完成 Prompt Library 搜索入口 submit 与 VoiceOver 匹配范围语义。本轮继续 Agent X 的 iPad / Mac / UI 体验优化循环，选择 Gallery filter rail 中 folder 行 action 的上下文语义作为低风险小轮次。

## 背景

Gallery filter rail 的 folder 行已经具备图片数量、filter 语义、44pt 最小高度和 pointer hover affordance。folder 行同时提供 swipe actions 和 context menu，用于 Rename / Delete 文件夹。当前这些动作视觉上可用，但 VoiceOver 在多文件夹场景下缺少具体 folder 名上下文，iPad 长按、Mac 右键和辅助功能用户不容易确认操作对象。

## 目标

- 为 Gallery folder row 的 Rename / Delete swipe actions 增加 folder 名上下文 VoiceOver label 和 hint。
- 为同一 folder row 的 Rename / Delete context menu actions 增加 folder 名上下文 VoiceOver label 和 hint。
- 保持现有可视文案、SF Symbol、menu/action 顺序、swipe/context menu 触发方式和 action 行为不变。
- 同步更新 README、flow、flowchart 和 update_log。

## 非目标

- 不修改 Gallery filter selection、folder rename/delete 实现、确认弹窗、图片归档规则或 tag 过滤。
- 不修改 Gallery grid、image tile、detail view、reuse/regenerate/share/delete、folder/tag organization。
- 不修改 SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。
- 不新增 keyboard shortcut、command menu、bulk folder management、folder sorting 或新 UI。

## 当前架构依据

- Gallery UI 位于 `LocalDiffusion/Views/Gallery/GalleryView.swift`，UI 层只负责展示、输入和用户反馈。
- folder rename/delete 仍由 `editingFolder`、`pendingFolderDeletion`、`renameFolder(id:name:)` 和 `deleteFolder(id:)` 负责。
- 本轮只给 SwiftUI `Button` 增加 accessibility modifiers，不触碰 SwiftData 模型、文件存储或推理边界。

## 实现步骤

1. 在 `ForEach(folders)` 的 `.swipeActions` 中，为 Rename action 添加 `accessibilityLabel("Rename folder: <name>")` 和对应 hint。
2. 在同一 `.swipeActions` 中，为 Delete action 添加 `accessibilityLabel("Delete folder: <name>")` 和对应 hint。
3. 在 `.contextMenu` 中对同两个 Button 添加相同语义。
4. 不改动 Button action closure、Label 文案、role、tint 或 menu 顺序。
5. 更新文档记录 Gallery folder action context 语义。
