# v1.3（Mac 平台可行性基线）

## 版本号

v1.3

## 版本分配依据

v1.0 和 v1.1 已完成 iPad 宽屏体验基线，v1.2 已新增 Plan/付费能力状态入口。人工总目标同时要求优化 Mac 版。当前项目事实显示 Xcode target 仍只支持 iPhone/iPad，native XCFramework 也只有 iOS 真机和 iOS simulator slices，因此本轮选择小步建立 Mac 平台状态基线，避免误称已经支持 Mac。

## 背景

当前 `LocalDiffusion.xcodeproj/project.pbxproj` 中 `SUPPORTED_PLATFORMS = "iphoneos iphonesimulator"`，`SUPPORTS_MACCATALYST = NO`，`TARGETED_DEVICE_FAMILY = "1,2"`。当前 `LocalDiffusionNative.xcframework` 的 `Info.plist` 只包含 `ios-arm64` 和 `ios-arm64_x86_64-simulator`。如果直接打开 Mac Catalyst，云端和本机很可能因为 native asset 缺少 macOS/Catalyst slice 而失败。

## 目标

- 在 Plan 页面增加平台状态说明：
  - 当前支持 iPhone / iPad 本地体验。
  - Mac Catalyst 当前未启用。
  - Mac 支持前置条件包括 Xcode 平台设置、native backend Mac/Catalyst slice、签名/分发和 UI 验证。
- 同步 README、flow、flowchart、update_log，明确当前不是 Mac 版本。
- 保持现有 Generate、Models、Gallery、Prompts、Plan 行为不变。
- 继续使用 `main` 直推和 GitHub Actions artifact 验收。

## 非目标

- 不启用 `SUPPORTS_MACCATALYST`。
- 不修改 `SUPPORTED_PLATFORMS`、`TARGETED_DEVICE_FAMILY` 或 Xcode target。
- 不重建或替换 `LocalDiffusionNative.xcframework`。
- 不新增 macOS target、Mac Catalyst entitlement、签名配置或分发流程。
- 不承诺当前版本可在 Mac 上运行。
- 不修改 StoreKit、付费、SwiftData schema、文件存储或 native bridge ABI。

## 当前架构依据

- 顶层入口仍由 `RootContentView` 管理。
- Plan 页面是当前承载产品状态、付费状态和平台状态的最小 UI 入口。
- Mac 真支持会跨 Xcode project、native Release asset、CI、UI 验证和发布配置；本轮只能记录真实状态与前置条件。
- native backend 只能通过 `ImageGenerationBackend` 暴露给 UI；平台状态 UI 不能直接调用 native bridge。

## 实现步骤

1. 阅读必读文件、本提示词、`RootContentView.swift`、`LocalDiffusion.xcodeproj/project.pbxproj` 和 `LocalDiffusionNative.xcframework/Info.plist`。
2. 确认当前分支和远端：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short`
3. 在 `PlanView` 增加平台状态 section，建议使用 `LabeledContent` 或 `Label`：
   - `iPhone / iPad`：Available。
   - `Mac Catalyst`：Not enabled。
   - 一行说明 Mac 支持需要 native backend Mac/Catalyst slice、Xcode 平台配置、签名和 UI smoke 验证。
4. 不添加任何可点击的假启用按钮或“立即支持 Mac”的文案。
5. 同步文档：
   - `README.md` Current implementation 增加平台状态基线。
   - `md/flow/flow.md` 更新 Plan 职责和未来扩展点。
   - `md/flow/flowchart.md` 更新 Plan 节点描述。
   - `update_log.md` 追加 v1.3 记录。

## 验收标准

- Plan 页面明确展示平台状态，不误称 Mac 已支持。
- 文档与项目事实一致：iPhone/iPad 当前可用，Mac Catalyst 未启用。
- 不修改 Xcode 平台、native asset、StoreKit 或数据层。
- 本地轻量检查、Swift parse 和沙箱外 iPhoneOS build 通过，或清晰记录无法本地 build 的环境原因。
- `origin/main` 最新 v1.3 run 成功，Agent C 下载并核对结果包通过。

## 风险和禁止项

- 禁止只改 UI 文案却声称已经支持 Mac。
- 禁止在 native asset 缺少 Mac/Catalyst slice 时打开 Catalyst。
- 禁止把 iPad regular 布局等同于 Mac 验收。
- 禁止新增不可构建的 Xcode 平台配置。
- 禁止跳过 Agent C artifact 核对。
