# v1.99 Generate 结果图库入口语义

## 版本号

v1.99

## 版本分配依据

v1.98 已完成 Models 未跟踪文件删除确认语义。当前继续 v1 体验优化阶段，选择 Generate 结果区一个可独立验收的 VoiceOver 语义小改进，因此递增为 v1.99。

## 背景

Generate 结果区的图片预览已经会说明图片尺寸以及是否保存到 Gallery，但 `View in Gallery` 按钮当前只暴露 `Ready` / `Unavailable`，没有在按钮名称和值里明确这是打开最新已保存结果，也没有把无 Gallery 记录时的不可用原因说清楚。VoiceOver 用户在结果区快速扫控件时，需要知道该入口和当前生成图片记录的关系。

## 目标

- 为 Generate 结果区 `View in Gallery` 按钮增加明确的 accessibility label，说明它会打开已保存的生成图片。
- 扩展 accessibility value，区分 `Saved to Gallery` 和 `Gallery record unavailable`。
- 扩展 accessibility hint，说明 ready 时会在 Gallery 中打开最新生成图片，unavailable 时需要先完成一次可保存的生成。
- 保持按钮可见文案、图标、禁用条件和 `onShowGallery()` 行为不变。

## 非目标

- 不修改图片生成、保存、取消、错误处理或 `latestGeneratedImageID` 数据流。
- 不修改 Gallery 导航、Root 跳转、SwiftData schema、文件存储或 native backend。
- 不新增 StoreKit、购买、恢复、receipt、subscription、entitlement persistence 或付费 gate。
- 不修改 Mac Catalyst、Xcode platform 设置、workflow 或 native Release asset。
- 不做大范围 UI 重构或样式改版。

## 当前架构依据

- UI 层只在 `LocalDiffusion/Views/**` 展示状态、输入和导航反馈。
- `GenerationView` 只能通过 `GenerationViewModel.latestGeneratedImageID` 判断是否有可跳转的 Gallery 记录。
- 结果图片文件和 SwiftData `GeneratedImage` 记录仍由 `GenerationViewModel` 管理，UI 不直接操作文件或数据层。
- SwiftUI accessibility 语义应使用文本 label/value/hint，保留 44pt 控件和现有暗色科幻风格。

## 实现步骤

1. 阅读必读文件、`GenerationView.swift` 和最新 v1.98 提示词。
2. 在 `GenerationView` 中为 `View in Gallery` 按钮添加 `accessibilityLabel(Text("View saved image in Gallery"))`。
3. 新增或复用 helper，生成 `Saved to Gallery. Ready to open the latest generated image.` / `Gallery record unavailable. No saved Gallery record is available for this preview.` 之类的 value。
4. 新增或复用 helper，生成 ready/unavailable 两种 hint。
5. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录本轮只改变 Generate 结果 Gallery 入口辅助功能语义。
