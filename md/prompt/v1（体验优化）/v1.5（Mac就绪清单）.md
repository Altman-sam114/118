# v1.5（Mac 就绪清单）

## 版本号

v1.5

## 版本分配依据

v1.3 已把 Mac Catalyst 当前未启用的事实展示到 Plan，v1.4 已把 Plan 付费能力整理为能力矩阵。人工总目标要求持续优化 Mac 版、iPad 版、UI、设计和付费功能；当前工程事实仍未满足真正 Mac 支持的前置条件，因此 v1.5 选择一个可验收的小步：把 Mac 支持从一句状态说明扩展为可执行的 readiness checklist。

## 背景

当前 Xcode project 仍只支持 `iphoneos iphonesimulator`，`SUPPORTS_MACCATALYST = NO`。当前 native backend 资产也只服务 iOS 设备和 iOS simulator 构建。直接启用 Mac Catalyst 需要 native Mac/Catalyst slice、签名和分发策略、平台 UI smoke、结果包验证策略和人工决策。本轮必须继续 truthful：Plan 可以说明 Mac 支持准备项，但不得暗示 Mac 版已可用。

## 目标

- 在 `PlanView` 中新增 Mac readiness checklist section。
- 每项展示标题、细节和状态，至少覆盖：
  - Xcode target platform：Requires configuration。
  - Native backend Mac/Catalyst slice：Requires native build。
  - Window and sidebar QA：Planned。
  - Distribution and signing：Requires decision。
- 状态必须用文字和图标表达，不能只靠颜色区分。
- 保持当前简洁暗色科幻风格、Form 结构和 Dynamic Type 友好。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、`TARGETED_DEVICE_FAMILY` 或 Xcode project。
- 不新增 macOS target、Mac entitlement、签名配置、分发脚本或 native Mac/Catalyst slice。
- 不接入 StoreKit，不新增产品 ID、价格、购买、恢复、receipt、subscription 或 entitlement。
- 不修改 Generate、Models、Gallery、Prompts、SwiftData schema、文件存储、native bridge ABI 或 CI workflow。
- 不把 iPad regular 布局宣传成 Mac 版本。

## 当前架构依据

- `RootContentView` 管理 Generate、Models、Gallery、Prompts、Plan 五个入口。
- `PlanView` 当前在 `RootContentView.swift` 内，是纯 UI 展示层，只能展示状态和准备项。
- `ImageGenerationBackend` 是推理边界，Plan 不得触碰 native bridge。
- 当前 UI 共享样式来自 `SciFiTheme`、`.sciFiScreen()`、`.bottomTabBarClearance()` 和 `.sciFiPanel()`。
- SwiftUI 技能要求状态差异不能只靠颜色，要支持 Dynamic Type 和 VoiceOver；本轮 checklist 行应使用 `Label`/`LabeledContent` 或等价结构。

## 实现步骤

1. 阅读必读文件、本提示词、`RootContentView.swift` 和 Xcode project 当前平台设置。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且没有无关改动：

```bash
git fetch origin
git switch main
git pull --ff-only origin main
git status --short
```

3. 在 `RootContentView.swift` 中为 Mac readiness 增加小型 UI 数据结构，建议字段：
   - title
   - detail
   - status title
   - systemImage
   - color
4. 在 `PlanView` 的 Platform Status 附近新增 `Mac Readiness` section。
5. checklist 必须明确所有 Mac 项都是未完成准备项，不允许出现 “Available”、“Enabled”、“Ready to ship”、“Buy/Upgrade”等误导性状态。
6. 保留现有 Platform Status、Capability Matrix 和 Availability 语义。
7. 同步文档：
   - `README.md` Current implementation 和 maintenance log。
   - `md/flow/flow.md` Plan UI 职责和未来扩展点。
   - `md/flow/flowchart.md` Plan 节点。
   - `update_log.md` v1.5 记录。

- `origin/main` 最新 v1.5 run 成功，Agent C 下载并核对结果包通过。
