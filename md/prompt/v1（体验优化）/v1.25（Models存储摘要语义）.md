# v1.25 / Models 存储摘要语义

## 版本号

v1.25

## 版本分配依据

- 当前体验优化阶段已完成到 v1.24。
- v1.24 优化了 Add Model 表单错误提示的可读性和 VoiceOver 语义。
- Models 页面顶部 `StorageSummaryRow` 已具备 Dynamic Type 视觉布局，但 VoiceOver 会逐项读取标题、状态 pill 和 Grid 内容，缺少一个组合后的存储摘要。

## 背景

`StorageSummaryRow` 展示 ready/total 模型数量、tracked bytes、models directory on-disk bytes 和 untracked files/bytes：

```swift
StorageSummaryRow(
    readyCount: readyModels.count,
    totalCount: models.count,
    trackedBytes: trackedModelBytes,
    directoryBytes: modelDirectoryBytes,
    untrackedCount: untrackedModelFiles.count,
    untrackedBytes: untrackedModelBytes
)
```

这些数值来自现有 SwiftData 查询和 `AppFileStore` reconcile 结果，是 Models 页面判断本地模型存储状态的重要摘要。本轮只添加辅助功能语义，不改变任何统计值、视觉布局、reconcile、导入、删除或下载行为。

## 目标

- 为 `StorageSummaryRow` 增加组合后的 accessibility label/value/hint。
- 保留现有可视布局、Grid 文案、ready pill、颜色、Dynamic Type 分支和 `.sciFiPanel(isHighlighted:)`。
- 辅助功能语义应表达：
  - 这是模型存储摘要。
  - ready/total 模型数量。
  - tracked storage。
  - model directory on-disk storage。
  - untracked file count and size。
- 使用 `ByteCountFormatter.fileSizeString(...)` 复用现有格式，不新增 formatter 或数据源。
- 保持 `readyModels`、`trackedModelBytes`、`modelDirectoryBytes`、`untrackedModelFiles` 和 `untrackedModelBytes` 计算方式不变。
- 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`，记录 Models storage summary 的 VoiceOver 摘要改善。

## 非目标

- 不修改 `StorageSummaryRow` 的视觉布局、字体、颜色、Grid 行或 ready pill 文案。
- 不修改模型统计、文件大小统计、未跟踪文件扫描、导入、删除、reconcile 或下载行为。
- 不修改 `AppFileStore`、`HuggingFaceDownloadManager`、`LocalModel`、SwiftData schema 或文件存储路径。
- 不修改 Add Model、ModelRow、ModelDetailView、UntrackedModelFileRow 或任何 action。
- 不修改 native backend、生成流程、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。

## 当前架构依据

- `ModelLibraryView` 属于 UI 展示层，只负责展示模型、存储摘要、用户操作和反馈。
- `StorageSummaryRow` 的输入来自上层已计算好的计数和字节数；本轮不得改变这些输入或计算。
- SwiftUI 可访问性要求：摘要类信息应能以单个明确元素被朗读，避免用户必须从分散的短标签中拼出整体含义。
