# v1.72 / Plan 紧凑页脚语义

## 版本号

v1.72

## 版本分配依据

- v1.71 已完成 iPad sidebar 的 Plan hint 语义补齐，并通过 `origin/main` 云端结果包验收。
- 本轮继续收窄 Plan 页面可访问性一致性：regular 宽屏 `PlanPanel` footer 已带面板上下文，compact Form section footer 也应提供同等级上下文。

## 背景

Plan regular 布局中的 `PlanPanel` footer 已使用 `accessibilityElement(children: .ignore)`、`accessibilityLabel("\(title) note")` 和 `accessibilityValue(footer)`，VoiceOver 能知道 footer 属于哪个面板。compact Form 中 Mac Readiness、Capability Matrix 和 Entitlement Rules 的 footer 当前只是普通 `Text`，可见文案正确，但缺少同样的 section note 上下文。

## 目标

- 为 Plan compact Form 的 Mac Readiness、Capability Matrix、Entitlement Rules footer 增加 section note 上下文。
- 保持 footer 可见文案、Form/Section 结构、布局、Plan 页面事实和业务逻辑不变。
- 与 regular `PlanPanel` footer 的可访问性语义保持一致。

## 非目标

- 不修改 Plan 可见文案、Plan 图标、能力矩阵数据、Entitlement Rules 数据、Availability、Mac readiness 数据、StoreKit 状态、product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不启用 Mac Catalyst，不修改 Xcode project、workflow、SwiftData、文件存储、native backend 或推理链路。
- 不新增截图验收、simulator smoke、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- `PlanView.compactLayout` 使用 SwiftUI `Form` 和 `Section` 展示 compact Plan 页面。
- regular 布局的 `PlanPanel` footer 已经为 VoiceOver 提供 `"<Panel> note"` label 和 footer value。
- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `PlanView.compactLayout`、`macReadinessSection`、`capabilityMatrixSection`、`entitlementRulesSection` 和 `PlanPanel` footer 语义。
2. 添加一个仅供 `PlanView` 使用的小 helper，接收 section title 和 footer text，返回带 contextual accessibility label/value 的 `Text`。
3. 将 Mac Readiness、Capability Matrix、Entitlement Rules 的 compact section footer 从裸 `Text` 改为该 helper。
4. 不修改 footer 文案本身，不修改 regular `PlanPanel`、section body、data arrays、StoreKit/Mac 事实或导航。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
