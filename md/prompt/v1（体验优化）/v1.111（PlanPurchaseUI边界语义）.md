# v1.111 / Plan Purchase UI 边界语义

## 版本号

- 版本：`v1.111`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.110 / Plan Mac QA 边界语义`。
- v1.110 已通过最新 `origin/main` artifact 验收：commit `9cdfa4384dbfbcd400e85ae3770567b6b5e12667`，run `28812664937`，attempt `1`，artifact `localdiffusion-ci-v1.110-main-9cdfa43-run28812664937-attempt1`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。本轮继续选择一个小而可验收的 paid-planning 透明度优化，递增为 `v1.111`。

## 背景

Plan 页面已经明确 Local plan 当前可用、paid candidates planning-only、StoreKit 未配置、没有 purchase state / entitlement / App Store product request，并且 Mac/Catalyst 未启用。Availability 区域中 `Purchase UI` 行当前说明：

```text
Purchase UI remains hidden until StoreKit products and entitlement mapping exist.
```

这已经表达购买 UI 未出现，但还可以更直接地说明当前没有 purchase sheet、buy button 或 unlock entry point，避免用户把 Plan 页面误认为可购买或可解锁的入口。

本轮只补强 Availability 的 `Purchase UI` 行可见 detail 和 VoiceOver hint，不启用真实 StoreKit、不新增按钮、不触碰 entitlement 或 paid gate。

## 目标

1. 只修改 `PlanView.availabilityItems` 中 `title: "Purchase UI"` 的 `detail`。
2. 让可见 detail 明确：
   - Purchase UI remains hidden。
   - 当前没有 purchase sheet、buy button 或 unlock entry point。
   - 后续需要 StoreKit products、entitlement mapping、restore/receipt decisions。
3. 同步修改同一 item 的 `accessibilityHint`，让 VoiceOver 用户听到当前没有可购买或解锁入口。
4. 保持 `Purchase UI` 行 title、status token `Requires configuration`、system image、排序、row renderer 和 Availability footer 不变。
5. 保持 Plan overview、Current Build、Capability Matrix、Entitlement Rules、Mac Readiness、Platform Status、Root navigation 和所有业务逻辑不变。

## 非目标

- 不启用 StoreKit，不新增 product IDs、purchase sheet、buy button、unlock button、restore flow、receipt validation、subscriptions、entitlement persistence、paid gate、App Store Connect 请求或真实购买流程。
- 不启用 Mac Catalyst，不修改 Xcode target/platform/scheme/signing/sandboxing/notarization/distribution，不新增 native Mac/Catalyst backend slice。
- 不修改 SwiftData schema、`AppFileStore`、`HuggingFaceDownloadManager`、`GenerationViewModel`、`ImageGenerationBackend`、native bridge、模型下载、图片生成、Gallery、Prompt Library 或 Models 行为。
- 不修改 `PlanAvailabilityItem` 数据结构、`AvailabilityRow`、`PlanStatusRow`、status enums、navigation、workflow、native backend asset metadata 或测试规范。
- 不重复修改 v1.109 的 Plan overview purchase boundary 或 v1.110 的 Mac QA boundary。

## 当前架构依据

- `PlanView` 位于 `LocalDiffusion/Views/RootContentView.swift`，属于 SwiftUI UI 展示层。
- `availabilityItems` 是静态 UI 数据；`AvailabilityRow` 将 `PlanAvailabilityItem.detail` 和 `accessibilityHint` 传给共享 `PlanStatusRow`。
- `PlanStatusRow` 已把 title/detail/status 组合为一个 row-level accessibility element，本轮只需改静态文本即可同步可见文案和 VoiceOver 语义。
- 本轮不涉及状态层、数据层、文件存储、native backend、StoreKit 或平台构建配置。

## 实现步骤

1. 确认当前分支为 `main`，并已同步最新 `origin/main`。
2. 阅读 `LocalDiffusion/Views/RootContentView.swift` 中 `availabilityItems`、`AvailabilityRow` 和 `PlanStatusRow`。
3. 只修改 `Purchase UI` item：
   - `detail` 建议表达为：`Purchase UI remains hidden; this build shows no purchase sheet, buy button, or unlock entry point until StoreKit products, entitlement mapping, and restore/receipt decisions exist.`
   - `accessibilityHint` 建议表达为：`Clarifies there is no purchase sheet, buy button, or unlock entry point in this build, and Purchase UI requires StoreKit products, entitlement mapping, and restore or receipt decisions before it can be added.`
4. 如需微调英文，必须保留同等含义：当前没有购买/解锁入口，且未来需要 StoreKit products、entitlement mapping、restore/receipt decisions。
5. 不修改 Availability 其他行、Plan overview、Current Build、Capability Matrix、Entitlement Rules、Mac Readiness、Platform Status、PlanPanel、PlanStatusRow、Root navigation 或任何业务逻辑。

   - `md/flow/flowchart.md` Plan 节点。
