# v1.32 / Plan Mac 可用性说明

## 版本号

v1.32

## 版本分配依据

- 当前体验优化阶段已完成到 v1.31。
- v1.3、v1.5 已建立 Mac 平台状态和 Mac readiness checklist，明确 Mac Catalyst 当前未启用。
- v1.19 已将 Plan Availability 区升级为可读状态行，但当前 Availability 只说明 Core local tools 和 Purchase UI，缺少一个直接表达 “Mac app 当前未交付” 的可用性行。
- 本轮继续人工总目标中的 Mac 版、iPad/UI、付费能力规划优化，选择一个可独立验收的小步：在 Plan Availability 区补充 Mac app 的当前可用性事实，不启用 Mac Catalyst，不改 StoreKit。

## 背景

Plan 页面已经在 Platform Status 中展示 iPhone/iPad 可用、Mac Catalyst 未启用，并在 Mac Readiness 中列出 Xcode target platform、native backend slice、window/sidebar QA、distribution/signing 等前置条件。

Availability 区是用户快速判断“当前能用什么、什么还不可用”的入口。当前它说明核心本地工具可用、Purchase UI 需要配置，但没有直接列出 Mac app 当前未启用。新增一条 Mac app 状态行可以减少误解：iPad regular 布局不是 Mac 支持，当前 app 仍只交付 iPhone/iPad。

## 目标

- 只在 `LocalDiffusion/Views/RootContentView.swift` 的 `availabilityItems` 中新增一个非交互 `PlanAvailabilityItem`。
- 新增行建议：
  - title：`Mac app`
  - detail：说明当前 iOS build 不交付 Mac/Catalyst app，Mac app 仍需要 Xcode platform configuration、native Mac/Catalyst backend slice、signing/distribution decision 和 window/sidebar QA。
  - status：`Not enabled`
  - status icon：`xmark.circle`
  - status color：`SciFiTheme.amber`
  - row icon：`desktopcomputer`
- 复用现有 `AvailabilityRow -> PlanStatusRow -> PlanStatusBadge`，不新增新组件。
- 保持 Plan compact Form、iPad regular 双栏/单列回退、状态 badge 视觉和 VoiceOver label/value 结构不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、Xcode project、签名、分发或 scheme 设置。
- 不修改 native XCFramework、native bridge ABI、Release asset、CI workflow 或 Mac/Catalyst slice。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增/删除 capability matrix、entitlement rules 或 Mac readiness checklist 项。
- 不修改 Root 导航、compact `TabView`、iPad `NavigationSplitView`、Generate、Models、Gallery、Prompts、SwiftData schema、文件存储、native backend 或生成流程。
- 不做 simulator screenshot、VoiceOver 目检、Mac build、StoreKit 测试或真机 GGUF 生成验收。

## 当前架构依据

- `PlanView` 位于 `RootContentView.swift`，属于 SwiftUI UI 展示层，只陈述当前 Local plan、StoreKit 未配置、Mac 未启用和未来候选能力。
- `availabilityItems` 是 Plan Availability 区的数据源，渲染通过 `AvailabilityRow` 复用 `PlanStatusRow` 的可读视觉和 VoiceOver label/value 语义。
- `LocalDiffusion.xcodeproj/project.pbxproj` 当前 Debug/Release 均为 `SUPPORTED_PLATFORMS = "iphoneos iphonesimulator"` 且 `SUPPORTS_MACCATALYST = NO`，本轮只能把这个事实展示出来，不能改变工程配置。
- Mac Readiness 已列出 Mac app 的真实前置条件；Availability 新增行应与这些事实保持一致。
