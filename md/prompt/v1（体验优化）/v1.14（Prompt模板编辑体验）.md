# v1.14（Prompt模板编辑体验）

## 版本号

v1.14

## 版本分配依据

v1.9 到 v1.13 已依次优化 Generate、Models、Prompts、Gallery 和共享参数控件在 iPad、Dynamic Type 与辅助功能场景下的可读性。当前 `PromptTemplateEditor` 仍使用两个裸 `TextEditor` 编辑 positive/negative prompts，缺少可见字段标题、placeholder、字符计数和明确辅助功能语义。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升 Prompt 模板编辑 sheet 的 prompt 输入体验，不改变模板保存、加载或生成参数语义。

## 背景

`PromptTemplateEditor` 同时服务 Prompt Library 新建/编辑模板，以及 Generate 页的 Save Template 流程。模板名称、分类和参数编辑已通过 Form 与共享 `ParameterEditor` 表达；但 positive/negative prompt 编辑区域只有 `TextEditor` 本体，空内容时缺少上下文，VoiceOver 读法不够明确，大字号下高度也偏紧。Generate 页已有带标题、placeholder、字符计数、科幻面板和描边的 prompt editor，本轮应让模板编辑器与该视觉语言保持一致。

## 目标

- 只优化 `PromptTemplateEditor` 的模板编辑 UI 展示层。
- 为 positive prompt 和 negative prompt 编辑区增加清晰可见标题。
- 为两个编辑区增加空内容 placeholder，保留 `TextEditor` 多行编辑能力。
- 显示字符计数，帮助用户判断模板内容长度。
- 编辑区使用当前暗色科幻 panelSoft 背景和轻描边，保持现有视觉体系。
- 根据 `dynamicTypeSize.isAccessibilitySize` 调整编辑区最小高度，避免辅助功能字号下输入区域过矮。
- 为两个 prompt 编辑区补足明确的 accessibility label/value/hint。
- 保持 `ParameterEditor(parameters: $parameters)`、模板保存、模板加载、分类、名称 trim 和 Save disabled 行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不修改 `GenerationParameters` 默认值、范围、归一化、sampler、seed、size 或任何生成参数语义。
- 不修改 `PromptTemplate` SwiftData schema、模板保存/加载路径、分类重命名/清空逻辑。
- 不修改 `GenerationViewModel`、`ImageGenerationBackend`、native bridge、文件存储、Hugging Face 下载或 Gallery 逻辑。
- 不接入 StoreKit，不新增产品 ID、购买、恢复、receipt、subscription 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 Xcode project 平台设置、native XCFramework、CI workflow。
- 不重构整个 Prompt Library，不拆分大文件，不改变已有列表行、分类菜单或共享参数编辑器。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `PromptTemplateEditor` 属于 UI 展示层，只负责模板字段输入、导航栏 Save/Cancel 和参数编辑 binding。
- 模板持久化仍由调用方闭包完成；本轮不得改变 `onSave(name, category, parameters)` 的调用时机和参数来源。
- Prompt Library 和 Generate 通过同一个 `PromptTemplateEditor` 保存模板，本轮 UI 改动会同时影响这两个入口。
- `ParameterEditor` 是共享参数控件，v1.13 已完成可读性优化；本轮不得再次扩大到该组件。
- 既有 v1.9-v1.13 已使用 `dynamicTypeSize.isAccessibilitySize` 进行局部布局回退，本轮可沿用该策略。

## 实现步骤

1. 阅读必读文件、本提示词、`PromptLibraryView.swift` 中 `PromptTemplateEditor`，以及 `GenerationView.swift` 中已有 prompt editor 作为视觉参考。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `PromptTemplateEditor` 中加入 `@Environment(\.dynamicTypeSize)`。
4. 将 `Section("Prompts")` 内两个裸 `TextEditor` 替换为带标题、placeholder、字符计数、背景和描边的局部 prompt editor。
5. Positive prompt 建议标题为 `Positive Signal`，placeholder 为 `Describe the image this template should generate`，accent 使用 `SciFiTheme.cyan`。
6. Negative prompt 建议标题为 `Negative Mask`，placeholder 为 `Artifacts, styles, or details to avoid`，accent 使用 `SciFiTheme.magenta`。
7. 普通字号下 positive prompt 最小高度不低于当前 96pt，negative prompt 不低于当前 72pt；accessibility Dynamic Type 下分别提高到更舒适高度。
8. placeholder 只在对应文本为空时显示，不写入参数值。
9. 给 `TextEditor` 添加明确 accessibility：
   - label 使用字段标题。
   - value 为空时描述为空 prompt；非空时可用当前文本。
   - hint 说明该字段用于保存模板的 positive/negative prompt。
10. 保持 Save、Cancel、name/category trim、Save disabled、`ParameterEditor` 和 `onSave` 闭包行为不变。
11. 同步文档：
    - `README.md`：说明 Prompt library/template editor 有带标签、placeholder 和 Dynamic Type-friendly 的 prompt 编辑区。
    - `md/flow/flow.md`：更新 Prompts UI 职责。
    - `md/flow/flowchart.md`：更新 Prompts 节点说明。
    - `update_log.md`：追加 v1.14 记录。

- `PromptTemplateEditor` 的 positive prompt 和 negative prompt 不再是裸 `TextEditor`。
- 两个编辑区都有可见标题、placeholder 和字符计数。

- accessibility Dynamic Type 下两个编辑区最小高度提高，避免输入区域过矮。
- VoiceOver 能读出对应字段 label、空/非空 value 和编辑用途 hint。
- 模板名称、分类、Save disabled、Save/Cancel、模板保存闭包和 `ParameterEditor` 行为保持不变。
- 不新增 StoreKit、Mac Catalyst、Xcode project、native backend、SwiftData schema、文件存储或 CI workflow 改动。
- 本地轻量检查和 Swift parse 通过；建议的 iPhoneOS build 通过或明确记录沙箱限制。
