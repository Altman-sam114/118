# v1.115 / Plan Purchase UI 恢复订阅入口边界语义

## 版本号

- 版本：`v1.115`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.114 / Plan StoreKit Gate 恢复收据边界语义`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- 本轮继续推进 Plan 页面付费能力透明度，只做一个可验收的 Availability 文案与 VoiceOver 语义优化，不实现真实 StoreKit、恢复购买、订阅管理或商品加载。

## 背景

Plan 页面 `Purchase UI` 行已经说明当前 Purchase UI hidden，且没有 purchase sheet、buy button 或 unlock entry point。随着 StoreKit gate 行已经排除 restore button、receipt validation path 和 entitlement mapping resolution，Availability 的 `Purchase UI` 行还可以更明确地排除用户最容易期待的恢复入口、订阅管理入口和产品加载状态，避免误解当前页面存在隐藏购买 UI 或产品加载中的状态。

## 目标

1. 只增强 `PlanView.availabilityItems` 中 `Purchase UI` 行的 `detail`。
2. 同步增强同一行的 `accessibilityHint`。
3. 明确当前没有 restore entry point、manage subscription entry point 或 product loading state。
4. 保持该行 title、status、icon、排序、row renderer 和所有业务逻辑不变。

## 非目标

- 不启用 StoreKit、product IDs、purchase UI、purchase sheet、buy button、unlock entry point、restore entry point、manage subscription entry point、product loading state、restore flow、receipt validation、subscriptions、entitlement persistence、paid gate、feature flags 或 unlock logic。
- 不修改 Plan overview、Current Build、Capability Matrix、Entitlement Rules、Mac Readiness、Platform Status 或 navigation。
- 不启用 Mac Catalyst，不修改 Xcode project、platform、scheme、signing、sandboxing、notarization 或 native Mac/Catalyst slice。
- 不修改 SwiftData schema、native backend、模型下载、生成、Gallery、Prompt Library、Models、workflow 或测试规范。

## 当前架构依据

- `PlanView` 位于 `LocalDiffusion/Views/RootContentView.swift`，属于 SwiftUI UI 展示层。
- `PlanAvailabilityItem.detail` 会由 `AvailabilityRow` 传入 `PlanStatusRow` 并组成 row-level accessibility label。
- `PlanAvailabilityItem.accessibilityHint` 会追加为该行 VoiceOver hint。
- 本轮只改静态展示文案和辅助功能语义，不触碰状态层、数据层、文件存储、native backend、StoreKit 或平台配置。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `PlanView.availabilityItems`、`AvailabilityRow` 和 `PlanStatusRow`。
2. 只修改 `Purchase UI` item：
   - detail 继续说明 Purchase UI hidden，没有 purchase sheet、buy button、unlock entry point，并补充没有 restore entry point、manage subscription entry point 或 product loading state。
   - accessibilityHint 同步说明没有这些购买、恢复、订阅管理或产品加载入口/状态。
3. 不修改该 item 的 title、status、systemImage、数组顺序或 row renderer。
4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
5. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
