# v1.65 / Gallery 文件夹编辑器保存语义

## 版本号

v1.65

## 版本分配依据

- v1.64 已完成 Gallery detail action accessibility 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段做小步、可验收的 Gallery 文件夹编辑器可访问性增强。

## 背景

Gallery 文件夹列表、folder rename/delete actions、筛选栏和详情操作已经具备较明确的 VoiceOver 语义。`FolderNameEditor` 用于新建和重命名文件夹，但 toolbar Save/Cancel 仍主要依赖可见标题和 disabled 状态，VoiceOver 对 folder name required、ready 和取消不保存的语义不够明确。

## 目标

- 为 `FolderNameEditor` 的 Cancel 按钮补充关闭并丢弃未保存编辑的提示。
- 为 `FolderNameEditor` 的 Save 按钮补充 ready/folder name required 状态。
- Save hint 说明会保存当前文件夹名称。
- 保持 New Folder 和 Rename Folder 两种入口现有行为。

## 非目标

- 不修改 folder 新建、重命名、删除、筛选、排序、图片分配、tag 保存或 Gallery navigation 行为。
- 不修改 `onSave(name.trimmingCharacters(...))`、disabled 条件、dismiss 时机、SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 GitHub Actions workflow。
- 不新增购买、订阅、entitlement、价格、product ID 或 Mac 平台开关。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。
- Gallery 文件夹编辑通过 `GalleryFolder` SwiftData 记录完成，不触碰 native bridge 或生成后端。
- 当前核心流程文档要求 Gallery 的筛选栏、详情页参数/操作和 Organization 控件提供清晰辅助功能语义和足够触控区域。

## 实现步骤

1. 阅读 `GalleryView.swift` 中 `FolderNameEditor`、`showingAddFolder` sheet 和 `editingFolder` sheet。
2. 保留 `onSave(name.trimmingCharacters(in: .whitespacesAndNewlines))` 和 `dismiss()` 现有顺序。
3. 给 Cancel 增加稳定 label/value/hint，说明关闭 editor 且不保存 folder name 更改。
4. 给 Save 增加稳定 label、ready/name-required value 和保存当前 folder name 的 hint。
5. 如需 helper，限制在 `FolderNameEditor` 内，避免跨文件抽象。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 语义事实。
