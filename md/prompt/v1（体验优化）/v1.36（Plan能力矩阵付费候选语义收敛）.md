# v1.36 / Plan 能力矩阵付费候选语义收敛

## 版本号

v1.36

## 版本分配依据

- 当前体验优化阶段已完成到 v1.35。
- v1.33 已把 Plan Availability 的 paid candidates 行和 Entitlement Rules 的 paid candidate 状态收敛为 `Planning only`，明确未售卖、未解锁。
- 当前 Plan Capability Matrix 仍显示 `Planned`，且 Batch queue controls、Curated prompt packs、Workflow export 三个 paid candidate 的说明没有统一表达 `Planning only / not sold / not unlocked`。
- 本轮选择一个小步付费规划语义目标：只收敛 Capability Matrix 的 paid candidate 文案，不新增 StoreKit、购买、entitlement 或 paid gate。

## 背景

Plan 页是当前付费能力规划和 Mac readiness 的事实入口。它必须真实说明当前 Local tools 可用、Purchase UI 未配置、paid candidates 仍只是规划候选。

v1.33 已经在 Availability 和 Entitlement Rules 中明确 paid candidates 是 `Planning only`。但 Capability Matrix 仍使用 `Planned` 状态，并且三条付费候选说明偏宽泛，容易被理解为功能正在交付或可解锁。本轮只统一这些展示语义，让 Plan 内部不同区域对 paid candidates 的说法一致。

## 目标

- 在 `LocalDiffusion/Views/RootContentView.swift` 中收敛 Plan Capability Matrix 语义：
  - `PlanCapabilityStatus.planned.title` 从 `Planned` 改为 `Planning only`。
  - `Batch queue controls` detail 改为明确 planning-only paid candidate，未售卖、未解锁。
  - `Curated prompt packs` detail 改为明确 planning-only paid candidate，未售卖、未解锁。
  - `Workflow export` detail 改为明确 planning-only paid candidate，未售卖、未解锁。
  - `Capability Matrix` footer 中的 `Planned` 改为 `Planning-only`，并说明 planning-only 和 configuration-gated items 都不是 purchases 或 active entitlements。
- 同步 README、flow、flowchart、update_log，记录 Plan Capability Matrix 现在与 Availability/Entitlement Rules 一致表达 paid candidates 的 planning-only 状态。

## 非目标

- 不新增 `import StoreKit`、product IDs、价格、购买、恢复、receipt、subscription、entitlement persistence 或 paid gate。
- 不把 Batch queue、curated prompt packs、workflow export 写成可购买、已授权、已解锁、可用或即将自动开放。
- 不修改 `PlanAvailabilityStatus`、`PlanEntitlementRuleStatus`、Availability rows 或 Entitlement Rules 的既有事实，除非只是文档描述同步。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、Xcode project、签名、分发或 scheme 设置。
- 不修改 native XCFramework、native bridge ABI、Release asset、CI workflow、SwiftData schema、文件存储、GenerationViewModel、Gallery、Models、Prompts 或 Generate 业务行为。
- 不修改 v1.34/v1.35 pointer hover 行为。

## 当前架构依据

- `RootContentView` 的 Plan 区属于 SwiftUI 展示层，只负责展示当前 Local plan、付费规划状态、平台状态和 Mac readiness。
- `PlanCapabilityStatus` 只用于 Capability Matrix 的视觉/VoiceOver 状态 token，不驱动购买或 entitlement。
- Availability 和 Entitlement Rules 已经把 paid candidates 表达为 `Planning only`；Capability Matrix 应使用同一事实语言。
- 当前项目没有 StoreKit 配置或 entitlement persistence，本轮不能新增任何真实付费能力。
