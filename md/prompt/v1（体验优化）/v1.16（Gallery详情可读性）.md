# v1.16（Gallery详情可读性）

## 版本号

v1.16

## 版本分配依据

v1.12 已优化 Gallery 图块可读性，v1.14 和 v1.15 分别优化 Prompt Template Editor 与 Generate prompt editor 的多行输入体验。当前 Gallery 图片详情页仍使用系统 `LabeledContent` 展示 prompt、negative prompt、seed、尺寸、sampler 等参数，长文本在 iPad split/detail 或 accessibility Dynamic Type 下容易被尾部压缩；Reuse、Regenerate、Share 和 Save Tags 操作也缺少明确的辅助功能 hint 与统一 44pt 行高。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升 Gallery 图片详情页参数和操作区可读性，不改变图片复用、再生成、删除、分享、标签或文件夹业务行为。

## 背景

Gallery detail 是用户复查生成结果、复用参数和再生成的重要入口。图块层已经在 v1.12 支持大字号可读元数据和 VoiceOver 摘要，但详情页的参数区仍把长 prompt 当作普通 key/value 行处理。`swiftui-pro` design/accessibility 规则要求避免固定横向挤压，按钮保持真实文本 label 和足够触控区域。本轮应只改善详情页展示层，让参数更容易扫描和朗读。

## 目标

- 只优化 `LocalDiffusion/Views/Gallery/GalleryView.swift` 中 `ImageDetailView` 的 UI 展示层。
- 将 Gallery detail 的参数行从默认 `LabeledContent` 改为项目风格的可读参数行。
- Prompt 和非空 negative prompt 使用纵向长文本布局，普通字号也不被尾部压缩。
- accessibility Dynamic Type 下所有参数行使用纵向布局，避免 label/value 横向挤压。
- 参数行补足明确的 accessibility label/value。
- Reuse Parameters、Reuse and Generate、Share PNG、Save Tags 操作保持带文本 `Label`，补足至少 44pt 行高和明确 hint。
- Tags 输入在 accessibility Dynamic Type 下允许更舒适的多行范围。
- 保持当前暗色 Sci-Fi panel、cyan/mint accent 和 Form 结构。
- 保持 Reuse、Regenerate、Delete、Share、Folder picker、Tags 保存、SwiftData 保存和文件删除行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不修改 `GeneratedImage`、`GalleryFolder`、`GenerationParameters` 或任何 SwiftData schema。
- 不修改 Gallery filter、sort、folder/tag 过滤、图片详情导航、reuse/regenerate/delete/share/tag/folder 的业务语义。
- 不修改 `GenerationViewModel`、`AppFileStore`、`ImageGenerationBackend`、native bridge、Hugging Face 下载或 Models 逻辑。
- 不接入 StoreKit，不新增购买、恢复、receipt、subscription、product ID 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 Xcode project 平台设置、native XCFramework、CI workflow。
- 不重构 Gallery 整页、图片网格、filter rail 或图块。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `ImageDetailView` 属于 UI 展示层，只负责展示图片、参数、组织字段和操作入口。
- Gallery 复用/再生成仍通过父级闭包调用 `GenerationViewModel.load(image:)` 和 `generation.generate(...)`；本轮不得改变闭包调用条件或顺序。
- 删除仍由 `GalleryView.deleteImage(_:)` 处理文件和 SwiftData 元数据一致性；本轮不得改变删除路径。
- Folder picker 和 tags TextField 直接更新 `GeneratedImage` 元数据并保存 context；本轮只改善输入区可读性。
- v1.9-v1.15 已用 `dynamicTypeSize.isAccessibilitySize` 做局部大字号布局回退，本轮可沿用该策略。

## 实现步骤

1. 阅读必读文件、本提示词、`GalleryView.swift` 中 `ImageDetailView`、父级 `imageDetail(for:)` 和 `deleteImage(_:)`。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `ImageDetailView` 中读取 `@Environment(\.dynamicTypeSize)`。
4. 新增局部 Gallery detail 参数行组件或等价 helper：
   - 普通字号下短参数可保持横向 label/value。
   - Prompt 和非空 Negative 使用纵向长文本布局。
   - accessibility Dynamic Type 下所有参数行纵向显示。
   - value 允许换行，不用 `minimumScaleFactor` 或过小字体压缩。
   - 添加 accessibility label/value。
5. 用该参数行替换 `Section("Parameters")` 中现有 `LabeledContent` 参数行。
6. 为 Reuse Parameters、Reuse and Generate、Share PNG 和 Save Tags 操作补足 `frame(maxWidth: .infinity, minHeight: 44, alignment: .leading)` 或等价触控区域。
7. 给这些操作添加明确 `accessibilityHint`，说明操作结果但不改变动作。
8. Tags TextField 在 accessibility Dynamic Type 下允许更多行，普通字号保持紧凑。
9. 保持 image preview、folder picker、tag parsing、save context、delete confirmation、dismiss、share file existence check 等行为不变。
10. 同步文档：
    - `README.md`：说明 Gallery detail 参数和操作区支持 Dynamic Type-friendly 可读布局。
    - `md/flow/flow.md`：更新 Gallery UI 职责。
    - `md/flow/flowchart.md`：更新 Gallery 节点说明。
    - `update_log.md`：追加 v1.16 记录。
