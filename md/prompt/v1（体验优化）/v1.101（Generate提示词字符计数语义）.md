# v1.101 Generate 提示词字符计数语义

## 版本号

v1.101

## 版本分配依据

v1.100 已完成 Generate 空模型状态摘要语义。当前继续 v1 体验优化阶段，选择 Generate prompt editor header 中字符计数这一处可独立验收的 VoiceOver 字段上下文改进，因此递增为 v1.101。

## 背景

Generate 的 Positive Signal 和 Negative Mask header 会显示 `N chars` 字符计数。当前 `promptCharacterCount(_:)` 没有字段上下文，VoiceOver 单独聚焦时只能读出类似 `42 chars`，不说明这是 Positive Signal 还是 Negative Mask 的字符计数。用户在两个 prompt 区之间移动时，需要明确当前计数属于哪个 prompt field。

## 目标

- 为 Generate prompt header 的字符计数补充字段上下文。
- `promptCharacterCount` 接收当前 prompt title，用于生成 VoiceOver label。
- 字符计数的 accessibility label 使用 `<title> character count`。
- 字符计数的 accessibility value 使用 `0 characters` / `1 character` / `N characters`。
- 保留可见 `N chars` 文案、prompt binding、focus、清除按钮和布局不变。

## 非目标

- 不修改 Positive Signal / Negative Mask 的 TextEditor binding、placeholder、focus 或 prompt 内容。
- 不修改 Clear Prompt button、Save Template、generation gate、model selection、参数编辑或 Gallery handoff。
- 不修改 Prompt Library、共享 `ParameterEditor`、SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或 Release asset。
- 不做 simulator VoiceOver 实机朗读或视觉重设计。

## 当前架构依据

- UI 层只在 `LocalDiffusion/Views/**` 展示状态、输入、导航和用户反馈。
- `GenerationView` 的 prompt editor header 已经拿到 `title`，可在 call-site 内传给字符计数 helper。
- 这是纯 SwiftUI accessibility metadata，不应改变 prompt 数据流、generation request 组装或参数保存。
- SwiftUI 可访问性应使用文本 label/value 提供可扫描语义，同时保留现有暗色科幻视觉样式。

## 实现步骤

1. 阅读必读文件、`GenerationView.swift`、最新 v1.100 提示词和本轮相关代码。
2. 将 `promptCharacterCount(_:)` 改为 `promptCharacterCount(_:title:)`。
3. 在 accessibility Dynamic Type 和普通 header 两处调用中传入 `title`。
4. 在 `promptCharacterCount` 内保留可见 `Text("\(count) chars")`，并新增：
   - `.accessibilityLabel(Text("\(title) character count"))`
   - `.accessibilityValue(Text(promptCharacterCountAccessibilityValue(count)))`
5. 新增私有 helper `promptCharacterCountAccessibilityValue(_:)`，处理 `1 character` / `N characters`。
6. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录本轮只改变 Generate prompt 字符计数辅助功能语义。
