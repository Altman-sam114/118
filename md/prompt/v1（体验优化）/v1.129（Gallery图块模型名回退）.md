# v1.129 / Gallery 图块模型名回退

## 版本号

- 版本：`v1.129`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.128 / Gallery 预览下一步语义`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.128 已补齐 Gallery detail preview 的下一步语义；本轮继续优化 Gallery grid 图块在 iPad 分栏、窄宽度和较长 GGUF 模型名下的可读性。

## 背景

`ImageTile.metadataView` 在 accessibility Dynamic Type 下已经使用纵向日期/模型名布局，但普通 Dynamic Type 分支固定使用横向 `HStack`，并把模型名限制为 1 行。GGUF 模型文件名常较长，Gallery grid 在 iPad split、Stage Manager 或较窄 adaptive tile 中容易截断模型名。本轮只给普通字号 metadata 增加横向优先、空间不足时纵向回退的布局，不改变图片图块的导航、筛选、排序或 VoiceOver 输出。

## 目标

1. 只修改 `ImageTile.metadataView` 的普通 Dynamic Type 分支。
2. 使用 `ViewThatFits(in: .horizontal)` 包裹既有横向日期/模型名布局。
3. 在横向布局放不下时回退到纵向日期/模型名布局，模型名最多 2 行并允许垂直展开。
4. 保持 accessibility Dynamic Type 现有纵向布局不变。
5. 保持 image tile preview、prompt 文案、accessibility label/value/hint、NavigationLink、filter/sort、detail navigation、SwiftData 和文件存储逻辑不变。

## 非目标

- 不修改 Gallery detail、folder/tags、删除确认、Refresh Gallery、Sort menu 或 filter rail。
- 不修改 image grid column minimum、tile panel 样式、thumbnail frame、prompt line limit、metadata 字体或颜色。
- 不修改 `ImageTile` 的 accessibility label/value/hint。
- 不修改 SwiftData schema、AppFileStore、GenerationViewModel、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GalleryView.swift` 位于 `LocalDiffusion/Views/Gallery/`，属于 SwiftUI UI 展示层。
- `ImageTile` 只展示 `GeneratedImage` 元数据并通过 `NavigationLink` 打开详情。
- Gallery 图块已经提供 pointer hover、prompt-focused VoiceOver label 和 model/date/output metadata value；本轮只改善可见 metadata 的布局回退，不改变数据或交互。

## 实现步骤

1. 阅读 `ImageTile.metadataView`。
2. 保留 accessibility Dynamic Type 分支。
3. 将普通 Dynamic Type 分支改为：

```swift
ViewThatFits(in: .horizontal) {
    HStack(...)
    VStack(...)
}
```

4. 横向候选保持既有日期、spacer、1 行模型名。
5. 纵向候选显示日期和最多 2 行模型名，并对模型名使用 `.fixedSize(horizontal: false, vertical: true)`。
6. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Gallery 图块 metadata 在窄宽度下可回退堆叠以保留模型名可读性。
7. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
