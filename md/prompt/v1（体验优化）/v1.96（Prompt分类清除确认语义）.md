# v1.96 / Prompt 分类清除确认语义

## 版本号

v1.96

## 版本分配依据

- v1.95 已完成 Gallery folder delete confirmation buttons 的 folder 名称上下文和确认/取消 VoiceOver 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续做窄范围 UI accessibility 优化，转向 Prompt Library 的分类清除确认弹窗，补齐 destructive / cancel 按钮的分类名上下文。

## 背景

Prompt Library 的分类 action menu 已经提供分类名上下文，但 `.confirmationDialog("Clear category?")` 内的两个按钮目前只有可见文案：

- destructive `Clear Category`；
- cancel `Cancel`。

弹窗 message 已说明模板会留在 library 并移动到 Uncategorized，但按钮本身没有分类名上下文，也没有明确确认清除分类或取消保留分类的辅助功能说明。

## 目标

- 为 category clear confirmation dialog 的 destructive `Clear Category` button 增加：
  - accessibility label：`Clear category: <category name>`；
  - accessibility value：待清除 category 名称；
  - accessibility hint：说明该 category 下的 templates 会移动到 Uncategorized，templates 保留在 library。
- 为 cancel `Cancel` button 增加：
  - accessibility label：`Cancel clearing category: <category name>`；
  - accessibility value：待清除 category 名称；
  - accessibility hint：说明保留 category 并关闭确认。
- 可新增小型 private helper，用于从 `pendingCategoryClear?.category` 生成稳定 category 名称和值。
- 保留 dialog title、message、按钮可见文案、destructive/cancel roles、`clearCategory(_:)` 行为、`pendingCategoryClear = nil` 清理和 `categoryClearBinding` 不变。

## 非目标

- 不修改 `clearCategory(_:)` 的数据行为：不得删除 PromptTemplate，不得改变 category 清空和 `updatedAt` 更新逻辑。
- 不修改 `categoryClearBinding`、`pendingCategoryClear` 的展示/清理机制、category menu、rename category、template edit/load/save、search、grouping 或 sorting 行为。
- 不修改 SwiftData schema、file storage、`GenerationViewModel`、native backend、bridge、workflow、Xcode project、StoreKit、paid gates、Mac Catalyst 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Prompt Library 页面位于 `LocalDiffusion/Views/Prompts/PromptLibraryView.swift`，属于 UI 展示层。
- `PromptCategoryEditorState` 已保存待操作 category 字符串，可直接用于 confirmation dialog button accessibility。
- `clearCategory(_:)` 只将同 category 的 templates 移到 Uncategorized 并保存 SwiftData；本轮不进入 schema、文件存储或生成状态边界。

## 实现步骤

1. 阅读 `.confirmationDialog("Clear category?")`、`pendingCategoryClear`、`categoryClearBinding`、`clearCategory(_:)` 和已有 category action menu 语义。
2. 新增或复用 helper 获取待清除 category 名称。
3. 在 destructive `Clear Category` button 后追加 accessibility label/value/hint。
4. 在 cancel `Cancel` button 后追加 accessibility label/value/hint。
5. 核对没有改动 dialog title/message、button roles、visible copy、clearCategory 行为、pending state 清理或 template grouping logic。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
