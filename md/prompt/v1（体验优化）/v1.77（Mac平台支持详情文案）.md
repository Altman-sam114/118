# v1.77 / Mac 平台支持详情文案

## 版本号

v1.77

## 版本分配依据

- v1.76 已完成 Mac readiness 平台支持标题优化，并通过 `origin/main` 云端结果包验收。
- 本轮继续优化 Plan 的 Mac readiness copy，把第一项 detail 从工程目标术语收敛为更直接的 iPhone/iPad app target 当前状态说明。

## 背景

Plan 的 Mac Readiness checklist 用于说明未来 Mac 支持仍需多个前置条件。第一项 title 已是 `Apple platform support`，但 detail 仍写 `Project still targets iPhoneOS and iPhone Simulator only.`，偏向工程配置表述。本轮只调整该 row 的 detail，让用户理解当前构建仍只配置 iPhone/iPad app target，不暗示已经启用 Mac 或 Catalyst。

## 目标

- 将 Mac Readiness 第一项 detail 改为 `This build is still configured for the iPhone and iPad app target only.`。
- 保持该 row 的 title、status、icon、accessibility hint 和 Mac readiness 数据结构不变。
- 保持 Mac app not enabled、Mac Catalyst 未启用、native backend slice 未配置等事实不变。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode target/platform、signing、native backend slice、XCFramework、workflow、SwiftData、文件存储或推理链路。
- 不修改 StoreKit product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不修改 Mac readiness status enum、row layout、Plan 导航或除该 detail 以外的可见文案。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- `MacReadinessItem.detail` 会进入 `PlanStatusRow` 的可见说明和 accessibility label。
- 该 row 当前 status 为 `.requiresConfiguration`，表达未来 Mac 支持仍需平台配置。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `macReadinessItems`、`MacReadinessRow` 和 `PlanStatusRow`。
2. 只将第一项 Mac readiness detail 改为 `This build is still configured for the iPhone and iPad app target only.`。
3. 不改 title、status、hint、icon、layout、Mac support facts 或任何业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
