# v1.120 / Generate iPad 布局宽度回退

## 版本号

- 版本：`v1.120`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.119 / Generate 控制台状态汇总语义`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- 上轮 Explorer 子 agent 已指出 Generate iPad regular 布局缺少宽度不足时的 `ViewThatFits` 回退；本轮选择这个小而可验收的 iPad UI 韧性优化。

## 背景

Generate 当前在 `horizontalSizeClass == .regular` 且非 accessibility Dynamic Type 时直接使用双栏 `wideGenerationLayout`。在 Stage Manager、Split View 或窄窗口场景下，regular size class 不一定代表有足够横向空间。左栏最小宽度、分隔线和右栏最小宽度叠加后，双栏可能比可用宽度更紧，影响 iPad 多窗口可读性。

Plan 已在 regular layout 中使用 `ViewThatFits(in: .horizontal)` 优先显示双栏，不适合时回退单列。Generate 可以采用同一思路，保持 iPad 宽屏双栏优先，同时让窄 regular 宽度自动选择已有单列布局。

## 目标

1. 只调整 `GenerationView.generationLayout` 的 regular / 非 accessibility Dynamic Type 分支。
2. 使用 `ViewThatFits(in: .horizontal)` 优先显示 `wideGenerationLayout`，在横向空间不足时回退 `singleColumnGenerationLayout`。
3. 保持 accessibility Dynamic Type 仍直接使用单列。
4. 保持 compact size class 仍使用 compact/single-column。
5. 不改变 visible content ordering、section 内容、status semantics、prompt/model/parameter/run/result 逻辑。

## 非目标

- 不修改 `wideGenerationLayout` 内部栏宽、分隔线、Form 内容或视觉样式。
- 不修改 `singleColumnGenerationLayout` 的 section 顺序。
- 不修改 prompt、model picker、generation gate、run button、progress、result、Gallery handoff、SwiftData、file storage、native backend 或 StoreKit。
- 不启用 Mac Catalyst、不新增 Mac target、不修改 Xcode project、workflow 或测试规范。
- 不做 simulator screenshot 或真实 iPad Stage Manager 人工目检；本轮交给 Swift parse 和云端 build 覆盖语法/构建风险。

## 当前架构依据

- `GenerationView` 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，属于 SwiftUI UI 展示层。
- `generationLayout` 当前根据 `horizontalSizeClass` 与 `dynamicTypeSize.isAccessibilitySize` 在 compact、single-column、wide layout 之间切换。
- `singleColumnGenerationLayout` 已是 Generate 的可读回退布局，包含 console、model、prompts、parameters、run、result。
- `wideGenerationLayout` 已是 iPad regular 双栏布局，左侧放 model/prompts/parameters，右侧放 console/run/result。
- `RootContentView.PlanView.regularLayout` 已有 `ViewThatFits(in: .horizontal)` 优先双栏、回退堆叠的项目内模式。

## 实现步骤

1. 阅读 `GenerationView.generationLayout`、`singleColumnGenerationLayout`、`wideGenerationLayout` 和 `RootContentView.PlanView.regularLayout` 的 `ViewThatFits` 模式。
2. 在 `GenerationView.generationLayout` 中保留：
   - regular + accessibility Dynamic Type：`singleColumnGenerationLayout`
   - compact：`compactGenerationLayout`
3. 将 regular + 非 accessibility Dynamic Type 从直接 `wideGenerationLayout` 改为：

```swift
ViewThatFits(in: .horizontal) {
    wideGenerationLayout
    singleColumnGenerationLayout
}
```

4. 不改其他 layout 函数内容。
5. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Generate iPad regular 会优先双栏并在窄宽度回退单列。
6. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
