# v1.79 / Mac 分发签名详情文案

## 版本号

v1.79

## 版本分配依据

- v1.78 已完成 Mac readiness native backend 详情文案优化，并通过 `origin/main` 云端结果包验收。
- 本轮继续优化 Plan 的 Mac readiness copy，把第四项 detail 从具体发布通道枚举收敛为更清楚的 release channel、signing、sandboxing、notarization 产品决策说明。

## 背景

Plan 的 Mac Readiness checklist 用于说明未来 Mac 支持仍需多个前置条件。第四项 title 是 `Distribution and signing`，当前 detail 写 `Developer ID, sandboxing, notarization, and App Store path require a product decision.`。这列出了具体分发路径，但对用户来说更重要的是 Mac release channel、signing、sandboxing、notarization 都还没有产品决策。本轮只调整该 row 的 detail，不改变真实 signing、distribution 或 Mac app 状态。

## 目标

- 将 Mac Readiness 第四项 detail 改为 `Mac release channel, signing, sandboxing, and notarization still need a product decision.`。
- 保持该 row 的 title、status、icon、accessibility hint 和 Mac readiness 数据结构不变。
- 保持 Mac app not enabled、Mac Catalyst 未启用、signing/distribution 未配置等事实不变。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode target/platform、signing、entitlements、native backend slice、XCFramework、workflow、SwiftData、文件存储或推理链路。
- 不修改 StoreKit product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不修改 Mac readiness status enum、row layout、Plan 导航或除该 detail 以外的可见文案。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- `MacReadinessItem.detail` 会进入 `PlanStatusRow` 的可见说明和 accessibility label。
- 该 row 当前 status 为 `.requiresDecision`，表达未来 Mac 发布、签名和分发路径仍需产品决策。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `macReadinessItems`、`MacReadinessRow` 和 `PlanStatusRow`。
2. 只将第四项 Mac readiness detail 改为 `Mac release channel, signing, sandboxing, and notarization still need a product decision.`。
3. 不改 title、status、hint、icon、layout、Mac support facts 或任何业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
