# v1.51 / Models 空状态语义

## 版本号

v1.51

## 版本分配依据

- 当前体验优化阶段已完成到 v1.50。
- Models 已具备 Add Model 键盘提交、Download ready/missing 状态、Storage Matrix 摘要、行内控制模型名上下文、未跟踪文件操作上下文、状态消息可读语义等 VoiceOver 优化。
- `ModelLibraryView` 在 `models.isEmpty` 且 `untrackedModelFiles.isEmpty` 时仍使用通用 `EmptyStateView("No models")`，没有向 VoiceOver 明确暴露“没有 tracked models 或 untracked GGUF files”的状态，也没有给两个下一步入口补充操作提示。
- 本轮选择 Models 空状态的小步语义优化，不修改模型下载、导入、未跟踪文件扫描、删除、SwiftData、文件存储或 native backend 行为。

## 背景

Models 页承担本地 GGUF 模型准备入口。首次打开或清空模型后，用户会看到 “No models” 空状态和 “Download from Hugging Face”“Import GGUF File” 两个入口。当前可视信息足够，但 VoiceOver 用户只能听到通用空状态和按钮文本，缺少当前存储状态与下一步操作说明。

本轮只补充空状态和按钮的辅助功能语义。视觉布局、按钮文案、sheet、fileImporter、导入 loading、toolbar、storage sections 和 reconcile 行为保持不变。

## 目标

- 在 `LocalDiffusion/Views/Models/ModelLibraryView.swift` 的 Models 空状态分支中：
  - 保留现有 `EmptyStateView(systemImage:title:message:)`、两个按钮、按钮样式和 disabled 逻辑。
  - 让 `EmptyStateView` 作为组合 VoiceOver 元素读取。
  - 增加空状态 label/value/hint，明确当前没有 tracked models 或 untracked GGUF files，并提示可使用下载或导入入口添加本地模型。
  - 给 “Download from Hugging Face” 按钮增加 hint，说明会打开 Hugging Face 模型下载表单。
  - 给 “Import GGUF File” 按钮增加 ready/importing value 和 hint，说明会打开本地 GGUF 文件选择器。
- 同步 README、flow、flowchart、update_log，记录 Models 空状态会暴露无模型/无未跟踪文件状态和下一步入口语义。

## 非目标

- 不修改 `models`、`untrackedModelFiles`、`storageSections`、`reconcileModelFiles()`、`handleModelFileImport(_:)`、`importUntrackedModel`、下载、暂停、恢复、取消、删除或详情 sheet 行为。
- 不修改 Add Model URL 解析、重复检测、文件导入路径、未跟踪文件扫描、Storage Matrix 或模型行控制逻辑。
- 不修改 SwiftData schema、文件存储、`HuggingFaceDownloadManager`、`GenerationViewModel`、native backend、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不新增 StoreKit import、product ID、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线、Mac build、StoreKit 测试或真机 GGUF 生成验收。

## 当前架构依据

- `ModelLibraryView` 属于 `LocalDiffusion/Views/Models/ModelLibraryView.swift` 的 UI 展示层，只负责模型列表展示、用户输入、导航、导入和反馈入口。
- 空状态是否显示已经由 `models.isEmpty` 和 `untrackedModelFiles.isEmpty` 决定，本轮只为该 UI 状态增加辅助功能表达。
- 辅助功能 label/value/hint 属于 SwiftUI 展示层，不应进入 SwiftData、文件存储、下载管理器或 native bridge 边界。
- 按钮应保持独立可操作元素，不能把整个空状态 `VStack` 合并成一个元素而遮蔽下载和导入按钮。
