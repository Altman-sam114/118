# v1.136 / Models 行标题窄宽度回退

## 版本号

- 版本：`v1.136`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.135 / Prompt 模板指标窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.135 已补齐 Prompt metric pills 窄宽度回退；Models 列表 `ModelRow.header` 在普通 Dynamic Type 下仍固定横排 identity + status pill，是下一个小而可验收的 UI 韧性优化。

## 背景

`ModelRow.header` 在 accessibility Dynamic Type 下已经使用纵向 model identity + status pill 布局，但普通 Dynamic Type 分支固定使用 `HStack(alignment: .firstTextBaseline)`。模型显示名与 `family | sourceFilename` 常较长，在 iPad split、Stage Manager 或窄 list 宽度下可能横向挤压状态 pill。项目内已在 Gallery metadata、Prompt metric pills、Plan status rows 使用 `ViewThatFits(in: .horizontal)`；本轮把同一模式补齐到 Models 行标题，不改变下载/删除/详情业务或 VoiceOver 语义。

## 目标

1. 只修改 `ModelRow.header` 的普通 Dynamic Type 分支。
2. 使用 `ViewThatFits(in: .horizontal)` 优先显示既有横向 identity + status pill 布局。
3. 横向空间不足时回退到既有纵向 identity + status pill 布局。
4. 保持 accessibility Dynamic Type 现有纵向布局不变。
5. 保持 model 名称/来源文案、status pill、progress、footer controls、message row、删除/暂停/恢复/详情行为不变。

## 非目标

- 不修改 `footer`、`controls`、`UntrackedModelFileRow`、Add Model sheet、detail sheet、download manager 或 storage summary。
- 不修改 status text/icon/color 计算、progress binding 或 model selection。
- 不修改 SwiftData schema、AppFileStore、GenerationViewModel、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager / Split View / VoiceOver 实机目检；禁止本地 Xcode / Simulator / 真机构建测试；完整构建交给 GitHub Actions。

- `ModelRow` 只展示模型元数据、下载进度和行内控制。
- 本轮只改善普通字号下 header 的布局回退，不改变数据或交互。
