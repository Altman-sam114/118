# v1.66 / Models 未跟踪导入保存语义

## 版本号

v1.66

## 版本分配依据

- v1.65 已完成 Gallery folder editor Save/Cancel accessibility 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段做小步、可验收的 Models 本地 GGUF 导入表单可访问性增强。

## 背景

Models 页面已经为 toolbar、空状态、未跟踪文件行、Add Model 下载表单和模型行控制补充了较完整的 VoiceOver 语义。`UntrackedModelImportEditor` 用于把模型目录中未跟踪的 GGUF 文件导入模型库，但 toolbar Import/Cancel 仍主要依赖可见标题和 disabled 状态，VoiceOver 对 display name required、ready 和取消不导入的语义不够明确。

## 目标

- 为 `UntrackedModelImportEditor` 的 Cancel 按钮补充关闭并不导入当前文件的提示。
- 为 `UntrackedModelImportEditor` 的 Import 按钮补充 ready/display name required 状态。
- Import hint 说明会用当前 display name 和 family 把该 GGUF 文件加入模型库。
- 保持未跟踪 GGUF 导入现有行为。

## 非目标

- 不修改未跟踪文件扫描、导入、删除、下载状态机、模型详情、辅助模型选择、文件存储或 SwiftData schema。
- 不修改 `onImport(name, family)`、disabled 条件、dismiss 时机、native backend、StoreKit、Mac Catalyst、Xcode project 或 GitHub Actions workflow。
- 不新增购买、订阅、entitlement、价格、product ID 或 Mac 平台开关。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。
- Models 页面通过 `AppFileStore` 与 SwiftData 元数据管理本地 GGUF 文件；本轮只增强导入表单语义，不改变数据层。
- 当前核心流程文档要求 Models 的下载、导入、未跟踪文件操作、空状态和行内控制提供明确辅助功能语义。

## 实现步骤

1. 阅读 `ModelLibraryView.swift` 中 `UntrackedModelImportEditor` 和未跟踪文件导入入口。
2. 保留 `onImport(name, family)` 和 `dismiss()` 现有顺序。
3. 给 Cancel 增加稳定 label/value/hint，说明关闭 editor 且不导入当前 GGUF 文件。
4. 给 Import 增加稳定 label、ready/display-name-required value 和导入当前 GGUF 文件的 hint。
5. 如需 helper，限制在 `UntrackedModelImportEditor` 内，避免跨文件抽象。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 语义事实。
