# v1.85 / Plan Platform Status Mac support note 文案

## 版本号

v1.85

## 版本分配依据

- v1.84 已完成 Plan Mac Readiness footer 文案优化，并通过 `origin/main` 云端结果包验收。
- 本轮继续沿 Mac/iPad 体验透明度做小步优化，聚焦 Plan `Platform Status` 中 `Mac support status` note 的可见文案和 VoiceOver hint。

## 背景

Plan 的 Platform Status 区已经用 summary rows 表达 `iPhone / iPad` 当前 Available、`Mac Catalyst` 当前 Not enabled。下面的 `Mac support status` note 仍写为 `Mac support requires Xcode platform changes...`，偏工程实现语气，也没有正面说明当前 iPhone/iPad app 可用。为了减少用户误解，本轮将该 note 改成更明确的 planning 状态：Mac support 仍是 planned，当前 iPhone/iPad app 可用，Mac/Catalyst app 未启用，并列出真实前置条件。

## 目标

- 将 `Mac support status` note 的 text 改为 `Mac support remains planned. The iPhone and iPad app is available in this build; a Mac/Catalyst app is not enabled until platform support, a native backend Mac/Catalyst slice, signing decisions, and dedicated UI validation are ready.`。
- 将该 note 的 accessibility hint 改为 `Clarifies that the current build provides the iPhone and iPad app while Mac/Catalyst support remains planned and not enabled.`。
- 保持 `accessibilityLabel: "Mac support status"`、`systemImage: "checklist"`、`iconColor: SciFiTheme.cyan` 不变。
- 保持 Platform Status summary rows、Mac Readiness、Availability rows、Plan layout 和业务逻辑不变。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode platform、target、scheme、signing、notarization、distribution setup 或 native Mac/Catalyst slice。
- 不修改 StoreKit、购买、entitlement persistence、paid gate、SwiftData schema、文件存储、workflow 或推理链路。
- 不修改 Platform Status summary rows、Mac Readiness rows/footer、Availability rows/footer、Capability Matrix、Entitlement Rules、Plan data structures 或 layout。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- `Platform Status` 中的 note 通过 `PlanNoteRow` 渲染；该 row 已使用可换行文本并通过 `accessibilityLabel` 与 `accessibilityHint` 暴露给 VoiceOver。
- 该 note 是静态文案，不触碰状态层、SwiftData、文件存储、native backend 或 StoreKit。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `platformStatusContent` 和 `PlanNoteRow`。
2. 只修改 `Mac support status` note 的 `text` 与 `accessibilityHint`。
3. 不改 `accessibilityLabel`、`systemImage`、`iconColor`、Platform Status summary rows、Plan layout 或业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
