# v1.30 / Gallery 组织语义

## 版本号

v1.30

## 版本分配依据

- 当前体验优化阶段已完成到 v1.29。
- v1.16 已优化 Gallery 详情页参数和主要操作的可读性，v1.17 已优化 Gallery 筛选栏。
- Gallery 详情页 `Organization` 区的 Folder picker、Tags 输入和 Save Tags 按钮仍只有较通用的提示，VoiceOver 用户不容易一次听清当前 folder 归属、当前 tag 输入和是否存在未保存 tag 变更。

## 背景

Gallery 图片详情页允许用户：

- 将图片分配到某个 Gallery folder 或设为 None。
- 编辑 comma-separated tags。
- 点击 Save Tags 写回 SwiftData。

本轮只改善 Organization 区的辅助功能语义，不改变 folder 绑定、tag 解析、保存时机、SwiftData、文件存储、删除、分享、复用或再生成行为。

## 目标

- 为 Folder picker 增加明确 accessibility value：
  - 当前没有 folder 时朗读 `No folder`。
  - 当前 folder 存在时朗读 folder name。
  - 当前 `folderID` 找不到对应 folder 时朗读 `Folder unavailable`。
- 为 Tags 输入增加明确 accessibility label/value/hint：
  - label 表达这是 image tags。
  - value 读取当前 `tagText` 输入，空值时朗读 `No tags`。
  - hint 说明用 commas 分隔并通过 Save Tags 保存。
- 为 Save Tags 按钮增加 accessibility value/hint：
  - value 表达 `Unsaved changes` 或 `No changes`。
  - hint 说明保存当前 comma-separated tags。
- 保持 Organization 区视觉、Form row 背景、Dynamic Type 行数、folderBinding、`tagText.tagsFromCSV()`、SwiftData 保存和现有按钮样式不变。

## 非目标

- 不修改 Gallery filter、sort、grid、image tile、image detail 参数行、Reuse、Regenerate、Share PNG、Delete Image 或 folder rename/delete 流程。
- 不修改 `GeneratedImage.tags`、`GalleryFolder`、SwiftData schema、`AppFileStore`、`GenerationViewModel`、native backend、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。
- 不新增 simulator screenshot、真机 VoiceOver 目检、真实 tag/folder 实机测试或真机 GGUF 生成验收。

## 当前架构依据

- `ImageDetailView` 属于 UI 展示层，负责读取当前 `GeneratedImage` 与 `GalleryFolder` 列表并调用现有 SwiftData 保存逻辑。
- Folder 归属通过 `folderBinding` 直接写入 `image.folderID` 并保存；tags 只有点击 Save Tags 才写入 `image.tags`。
- SwiftUI 可访问性要求：表单控件应暴露当前值和下一步操作，避免用户只听到通用标签而不知道当前状态。
