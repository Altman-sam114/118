# v1.69 / Plan 概览规划图标

## 版本号

v1.69

## 版本分配依据

- v1.68 已完成 Generate 模型选择语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段收窄 Plan 付费规划页面的视觉语义，避免把 planning-only 状态误读为购买入口。

## 背景

Plan 概览卡片文案明确说明当前 build 不保存 purchase state、不授予 entitlement，也不会从该页面请求 App Store product。当前概览装饰图标使用 `creditcard`，视觉上偏向支付/购买，可能与 planning-only、StoreKit not configured 的事实不一致。

## 目标

- 将 Plan 概览装饰图标从支付语义调整为中性的规划/清单语义。
- 保持 Plan 概览文案、VoiceOver label/value/hint、面板样式和布局不变。
- 保持 Plan 页面关于 StoreKit 未配置、无购买状态、无 entitlement 的事实表达不变。

## 非目标

- 不修改 `AppSection.plan` 的主导航图标、TabView、sidebar、导航状态或可访问性语义。
- 不修改 Plan 文案、能力矩阵、Entitlement Rules、Availability、Mac readiness、StoreKit 状态、product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不启用 Mac Catalyst，不修改 Xcode project、workflow、SwiftData、文件存储、native backend 或推理链路。
- 不新增截图验收、simulator smoke、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，只负责展示、输入、导航和用户反馈。
- Plan 页面当前只展示本地规划事实，不接入 StoreKit 或 entitlement 状态机。
- `planOverviewIcon` 是 `.accessibilityHidden(true)` 的装饰图标，外层 `planOverview` 已通过 `accessibilityLabel`、`accessibilityValue` 和 `accessibilityHint` 暴露真实语义。
- `AppSection.plan.systemImage` 同时驱动 compact TabView 和 iPad sidebar，是主导航识别入口，本轮不触碰。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `AppSection.plan.systemImage`、`planOverview`、`planOverviewIcon` 和 Plan 相关文案。
2. 只将 `planOverviewIcon` 内的 `Image(systemName: "creditcard")` 改为中性规划图标 `Image(systemName: "checklist")`。
3. 保留字体、颜色、frame、panel、overlay 和 `.accessibilityHidden(true)` 不变。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
5. 不引入跨文件 helper 或新抽象。
