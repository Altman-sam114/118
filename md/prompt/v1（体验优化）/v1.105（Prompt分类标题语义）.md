# v1.105 Prompt 分类标题语义

## 版本号

- 版本：`v1.105`
- 阶段：`v1（体验优化）`
- 任务名：Prompt 分类标题语义

## 版本分配依据

- 最新已完成版本为 `v1.104`，已补齐 Prompt template row 的 positive / negative / category / 参数摘要语义。
- 本轮继续在 Prompt Library 内做小步 UI/可访问性优化，让分类标题成为可导航 heading，并暴露搜索过滤后的可见模板数量，递增为 `v1.105`。

## 背景

`PromptCategoryHeader` 当前只以视觉样式显示分类名。VoiceOver 用户在 Prompt Library 中按分类浏览时，分类标题没有 heading trait，也不会说明当前分类下有多少可见模板。由于 Prompt Library 的 section 数据来自 `groupedTemplates`，其中 `group.templates.count` 已经对应当前搜索过滤后的可见模板数量，适合直接作为分类标题的辅助功能 value。

## 目标

1. 将 `PromptCategoryHeader.title` 标记为 VoiceOver heading。
2. 为分类标题提供明确 label：`<category> prompt category`。
3. 为分类标题提供当前可见模板数量 value：`1 template` 或 `N templates`。
4. 保留分类标题可见文案、样式、布局、分类 action menu、搜索、编辑、加载和分类行为不变。

## 非目标

- 不修改 `groupedTemplates`、`filteredTemplates`、`matchesSearch(_:)` 或排序/分组逻辑。
- 不修改模板新增、编辑、加载、删除、分类重命名或分类清除行为。
- 不修改 Prompt row、Prompt editor、Generate、Gallery、Models、Plan、StoreKit、Mac Catalyst、native backend、Xcode project 或 workflow。
- 不新增 simulator、真机 VoiceOver 或完整本地 build 要求。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，本轮只改 SwiftUI 展示语义。
- Prompt Library 的数据仍由 SwiftData `PromptTemplate` 和现有查询/过滤逻辑提供。
- `PromptCategoryHeader` 是分类标题和分类菜单的局部视图，适合承载分类 heading 与可见模板数量语义。
- 测试策略按 `md/test/test.md`：SwiftUI 源码变更需跑本地轻量检查、普通 Swift parse、native bridge Swift parse，并通过 `main` push 触发 GitHub Actions 结果包。
