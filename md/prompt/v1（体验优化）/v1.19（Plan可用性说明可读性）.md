# v1.19 / Plan 可用性说明可读性

## 版本号

v1.19

## 版本分配依据

- 当前体验优化阶段已完成到 v1.18。
- v1.18 已为 Plan overview、status summary row 和 status row 补足辅助功能语义，但 Availability 区仍是两条普通 `Label`，视觉和 VoiceOver 语义未与其他 Plan 状态行统一。
- 本轮继续在 v1 阶段做小范围、可验收的 Plan / 付费规划信息架构优化。

## 背景

Local Diffusion 当前是 iOS SwiftUI + SwiftData 本地图像生成应用。Plan 页面只展示当前 Local plan、StoreKit 未配置、未来付费候选、entitlement rules、平台状态和 Mac readiness blockers，不实现购买、不存储 entitlement、不启用 Mac Catalyst。

当前 `PlanView` 的 Current Build、Platform Status、Mac Readiness、Capability Matrix 和 Entitlement Rules 已使用 `PlanStatusSummaryRow` / `PlanStatusRow` / `PlanStatusBadge`，状态可读且有明确 VoiceOver label/value。Availability 区仍使用普通 `Label`：

- Generate、Models、Gallery、Prompts 保持 Local plan 可用。
- Purchase UI 只能在 StoreKit 产品和 entitlement mapping 存在后添加。

这两条信息是付费规划边界的关键确认点，应与其他 Plan 状态信息保持一致。

## 目标

- 将 Plan Availability 区从普通 `Label` 升级为可读的状态行。
- 明确表达两类状态：
  - 当前核心本地工具仍可用。
  - Purchase UI 仍被 StoreKit product IDs 和 entitlement mapping 阻塞。
- 保持 iPhone compact Form 和 iPad regular 双栏布局顺序不变。
- 让 Availability 行具备清晰的文字、SF Symbol、状态 badge、颜色和 VoiceOver label/value。
- 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`，记录当前真实 Plan 行为。

## 非目标

- 不接入 StoreKit。
- 不新增 product ID、价格、订阅、购买、恢复、receipt 校验或 entitlement 持久化。
- 不修改 SwiftData schema、文件存储、native backend、bridge、GenerationViewModel 或推理流程。
- 不修改 Xcode project，不启用 Mac Catalyst，不修改 supported platforms。
- 不改变 Generate、Models、Gallery、Prompts 的可用性或导航行为。
- 不做大范围 `RootContentView.swift` 拆分或无关 SwiftUI 重构。

## 当前架构依据

- UI 层只能展示、输入、导航和反馈。
- Plan 页面在 `LocalDiffusion/Views/RootContentView.swift` 中，只能作为付费和平台规划状态展示。
- 当前本地核心工具必须保持可用，不能因付费规划 UI 增加门禁。
- 状态展示应复用本文件内已有 `PlanStatusRow` 和 `PlanStatusToken` 模式，保持暗色 Sci-Fi 视觉。
- `PlanStatusBadge` 由父 row 负责辅助功能状态 value，避免重复朗读。

## 验收标准

- Availability 区不再是两条普通 `Label`。
- Availability 中核心工具可用和 purchase UI 阻塞条件均有明确 status badge。
- VoiceOver 能通过父 row 读到清晰 label/value，不重复朗读 badge。
- iPhone compact 和 iPad regular 内容顺序与现有 Plan 页面一致。
- 没有 StoreKit、Mac Catalyst、SwiftData schema、native backend、CI workflow 或业务门禁变更。
- 本地要求的检查通过。
- `origin/main` 最新 commit 对应的 GitHub Actions 结果包通过 Agent C 核对。

## 风险和禁止项

- 禁止把 planning UI 写成真实付费入口。
- 禁止新增购买、恢复、订阅、receipt、product ID 或 entitlement 存储。
- 禁止改变当前 Local plan 的 Generate、Models、Gallery、Prompts 可用性。
- 禁止启用 Mac Catalyst 或修改 Xcode 平台设置。
- 禁止跳过 Agent C artifact 验收。
- 禁止把旧 artifact、旧 run 或本地输出冒充本轮云端结果。
