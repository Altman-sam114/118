# v1.60 Models 工具栏入口语义

## 版本号

v1.60

## 版本分配依据

v1.59 已完成 Gallery folder actions 的上下文 VoiceOver 语义。本轮继续 Agent X 的 iPad / Mac / UI 体验优化循环，转向 Models toolbar 的添加与刷新入口语义。

## 背景

Models 页面已具备空状态、Add Model 表单、未跟踪文件行、Storage Matrix、模型行 controls 和详情状态消息的可访问语义。当前 toolbar 中 `Refresh Storage` 按钮和 `Add` menu 是 iPad / Mac 高频入口，但 toolbar 级上下文仍较薄：用户不一定能从 VoiceOver 中理解刷新会重新核对模型目录、Add menu 包含 Hugging Face 下载和本地 GGUF 导入两个入口。

## 目标

- 为 `Refresh Storage` toolbar button 增加 VoiceOver hint，说明会重新核对 tracked / untracked GGUF 文件和模型目录。
- 为 `Add` toolbar menu 增加 VoiceOver label、value 和 hint，说明该菜单提供 Hugging Face 下载与本地 GGUF 导入入口，并暴露当前 import 状态。
- 为 menu 内 `Download from Hugging Face` 和 `Import GGUF File` actions 增加明确 hint。
- 保持现有可视文案、SF Symbol、toolbar placement、menu action closure、disabled 条件和业务行为不变。

## 非目标

- 不修改 `reconcileModelFiles()`、`showingAddModel`、`showingModelFileImporter`、`isImportingModelFile` 或 fileImporter 逻辑。
- 不修改 Hugging Face URL 解析、下载状态机、暂停/恢复/取消/删除、未跟踪文件 import/delete、Storage Matrix 或模型详情。
- 不修改 SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。
- 不新增 keyboard shortcut、command menu、bulk import、Mac Catalyst target 或付费功能门禁。

## 当前架构依据

- Models UI 位于 `LocalDiffusion/Views/Models/ModelLibraryView.swift`，UI 层只负责展示、输入和用户反馈。
- 下载与恢复由 `HuggingFaceDownloadManager` 管理；文件导入和目录核对由现有 ModelLibraryView 方法处理。
- 本轮只添加 SwiftUI accessibility modifiers，不跨越 SwiftData、文件存储、下载状态机或 native backend 边界。

## 实现步骤

1. 在 `Refresh Storage` toolbar button 上追加 `accessibilityHint(...)`。
2. 在 `Add` toolbar `Menu` 上追加 `accessibilityLabel("Add model options")`、`accessibilityValue(isImportingModelFile ? "Importing" : "Ready")` 和 `accessibilityHint(...)`。
3. 在 menu 内 `Download from Hugging Face` action 上追加 hint，说明会打开 Hugging Face GGUF 下载表单。
4. 在 menu 内 `Import GGUF File` action 上追加 value / hint，说明会打开本地 GGUF 文件选择器，并保留 import 中禁用状态。
5. 更新 README、flow、flowchart 和 update_log。
