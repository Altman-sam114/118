# v1.18（Plan可访问性语义）

## 版本号

v1.18

## 版本分配依据

v1.2-v1.8 已建立 Plan 入口、付费能力规划、Mac readiness、entitlement rules、iPad 双栏布局和 Dynamic Type-friendly 状态行。当前 Plan 页面内容完整，但 `PlanStatusSummaryRow` 与 `PlanStatusRow` 主要依赖 `.accessibilityElement(children: .combine)`，VoiceOver 可能把标题、说明和视觉状态徽章按视图树机械拼接，缺少稳定的 label/value 结构；`planOverview` 也没有显式把 Local plan、StoreKit 未配置和未保存 entitlement 这三个关键信息合并成一个清晰摘要。本轮继续人工总目标中的付费功能规划、Mac/iPad 信息架构和 UI/accessibility 优化，选择一个可独立验收的小步：收口 Plan 已有展示内容的辅助功能语义，不新增付费功能、不启用 StoreKit、不启用 Mac Catalyst。

## 背景

Plan 是当前付费能力和 Mac readiness 的事实入口。它必须明确表达“当前 Local plan 可用、StoreKit 未配置、Mac Catalyst 未启用、付费候选仍是规划状态”。这些内容已经在视觉上用状态徽章和文案呈现，但辅助技术需要更稳定的 label/value/hint，避免重复朗读状态徽章，或漏读状态。`swiftui-pro` accessibility 规则要求图片/状态/按钮等元素具备明确辅助功能语义，状态不能只靠颜色传达。本轮应只改善 Plan UI 的 VoiceOver 结构。

## 目标

- 只优化 `LocalDiffusion/Views/RootContentView.swift` 中 Plan 相关 UI 的辅助功能语义。
- `PlanStatusSummaryRow` 显式提供 `accessibilityLabel(title)` 和 `accessibilityValue(status.title)`。
- `PlanStatusRow` 显式提供包含 title/detail 的 `accessibilityLabel`，并把 `status.title` 作为 `accessibilityValue`。
- 视觉状态徽章 `PlanStatusBadge` 在父 row 已提供状态 value 时不重复暴露给辅助技术。
- `planOverview` 合并为清晰的辅助功能元素，明确 Local Plan、StoreKit not configured、no purchase state / entitlement / App Store product requested。
- 保持 Plan 现有视觉布局、暗色 Sci-Fi 风格、状态颜色、图标和 Dynamic Type 行为。
- 保持 Current Build、Platform Status、Mac Readiness、Capability Matrix、Entitlement Rules、Availability 的内容事实不变。
- 保持 compact Form 和 iPad regular 双栏/单列回退结构不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不新增 Plan capability、entitlement rule、Mac readiness item、价格、产品名称或营销内容。
- 不引入 `StoreKit` import，不请求产品、不提供购买/恢复按钮、不保存 purchase state、不授予 entitlement。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、Xcode project、native XCFramework 或 CI workflow。
- 不修改 Root 导航、Tab/Sidebar 入口、Generate、Models、Gallery、Prompts、SwiftData schema、文件存储或 native backend。
- 不重构 Plan 页面为多文件或大幅拆分组件。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- Plan 位于 `RootContentView.swift`，属于 UI 展示层，只陈述当前 Local plan、StoreKit 未配置、Mac 未启用和未来候选能力。
- `PlanStatusToken` 是视觉状态元数据，不能作为真实 entitlement 或平台可用性的业务状态。
- `PlanStatusSummaryRow` 用于 Current Build 和 Platform Status 的简短状态。
- `PlanStatusRow` 用于 Mac Readiness、Capability Matrix 和 Entitlement Rules 的 title/detail/status 展示。
- Plan 当前无 StoreKit 数据层、无 entitlement persistence、无 Mac target，本轮不得新增这些能力。
- v1.8 已完成状态行视觉和 Dynamic Type 可读性，本轮只补辅助功能 label/value/hint 的稳定结构。

## 实现步骤

1. 阅读必读文件、本提示词、`RootContentView.swift` 中 `PlanView`、`planOverview`、`PlanStatusBadge`、`PlanStatusSummaryRow`、`PlanStatusRow`、`MacReadinessRow`、`CapabilityRow`、`EntitlementRuleRow`。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 为 `planOverview` 增加明确的 accessibility element：
   - children 可忽略或组合，但最终 label/value 必须稳定。
   - label 可为 `Local Plan`。
   - value 应包含本地优先、StoreKit 未配置、未保存 purchase state/entitlement、未请求 App Store product。
4. 为 `PlanStatusBadge` 增加可选参数或等价机制，避免在 `PlanStatusSummaryRow` / `PlanStatusRow` 中重复朗读状态：
   - 最小实现可让 `PlanStatusBadge` 本身 `.accessibilityHidden(true)`，前提是所有使用点的父 row 明确提供状态 value。
   - 不改变 badge 的视觉样式。
5. 修改 `PlanStatusSummaryRow`：
   - `.accessibilityElement(children: .ignore)`。
   - `.accessibilityLabel(title)`。
   - `.accessibilityValue(status.title)`。
   - 可加简短 hint，例如说明这是 Plan 状态摘要；若会产生噪音，可不加。
6. 修改 `PlanStatusRow`：
   - `.accessibilityElement(children: .ignore)`。
   - `.accessibilityLabel("\(title). \(detail)")` 或等价清晰组合。
   - `.accessibilityValue(status.title)`。
   - 不改变 title/detail/status 文案。
7. 视需要为 Current Build、Platform Status、Mac readiness、Capability Matrix、Entitlement Rules 的说明性 `Label` 保持系统默认朗读，不做隐藏，避免丢失警示文案。
8. 同步文档：
   - `README.md`：说明 Plan 状态行和 overview 具备更明确的辅助功能语义。
   - `md/flow/flow.md`：更新 Plan UI 职责。
   - `md/flow/flowchart.md`：更新 Plan 节点说明。
   - `update_log.md`：追加 v1.18 记录。
