# v1.7（付费权益规则基线）

## 版本号

v1.7

## 版本分配依据

v1.2 建立 Plan 入口和付费能力信息架构，v1.4 增加能力矩阵，v1.6 优化 Plan 的 iPad regular 宽屏布局。人工总目标要求继续优化付费功能，但当前项目仍没有 StoreKit 产品 ID、App Store Connect 配置、entitlement 规则、账号或收据策略。直接接入购买会依赖人工商业决策和外部配置，因此本轮选择一个可验收的小步：在 Plan 中明确未来付费权益的规则基线和保护边界。

## 背景

当前 Plan 已说明 Local plan、StoreKit 未配置、未来能力候选、Mac readiness 和平台状态，但还没有把“哪些能力永远保持本地可用、哪些能力可作为未来付费候选、真实购买前必须满足哪些前置条件”整理成清晰规则。这个缺口会影响后续 StoreKit 接入设计，也容易让 UI 文案误导用户。本轮应继续保持 truthful：只做 UI 信息架构和文档基线，不实现真实付费。

## 目标

- 在 `PlanView` 中新增 `Entitlement Rules` 展示区，说明未来付费接入前的规则边界。
- 规则至少覆盖：
  - `Core local tools stay available`：Generate、Models、Gallery、Prompts 当前 Local plan 能力不因本轮被锁定。
  - `Paid candidates need product decisions`：batch queue、pro prompt packs、workflow export 等仍只是候选能力。
  - `StoreKit gate is required`：真实购买前需要 product IDs、entitlement mapping、restore/receipt flow 和 App Store Connect 配置。
  - `No entitlement persistence yet`：当前不保存购买状态、不授予 entitlement、不请求 App Store 产品。
- compact 布局仍用单列 `Form`；regular 布局把该区放入右栏，并保持宽屏可扫描。
- 使用文字、图标和状态标签表达规则状态，不只靠颜色。
- 保持简洁暗色科幻风格、Dynamic Type 友好和可读性。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不接入 StoreKit，不新增 `import StoreKit`、product ID、SKU、价格、订阅、购买按钮、恢复按钮、receipt 校验、App Store Server API 或 entitlement 持久化。
- 不限制、隐藏或锁定现有 Generate、Models、Gallery、Prompts 能力。
- 不修改 SwiftData schema、文件存储、native backend、native bridge、Xcode project、CI workflow 或 Mac Catalyst 设置。
- 不启用 Mac 支持，不声称 Mac 版可用。
- 不做真实购买、真机 GGUF 生成、simulator smoke 或截图验收，除非人工另行要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `RootContentView` 管理 Generate、Models、Gallery、Prompts、Plan 五个入口。
- `PlanView` 当前在 `LocalDiffusion/Views/RootContentView.swift` 内，是纯 SwiftUI 展示层。
- Plan 不能触碰 `GenerationViewModel`、SwiftData、`AppFileStore`、`ImageGenerationBackend` 或 native C bridge。
- 当前 `SUPPORTED_PLATFORMS` 仍为 iPhoneOS/iPhone Simulator，`SUPPORTS_MACCATALYST = NO`，native XCFramework 仍只服务 iOS 构建。
- SwiftUI 设计要求：避免固定屏幕宽度，不使用 `UIScreen.main.bounds`；状态不能只靠颜色；图标装饰应隐藏或搭配文本；支持 Dynamic Type。
- 带 footer 的 `Section` 应继续使用显式 `Section { } header: { } footer: { }` 形式。

## 实现步骤

1. 阅读必读文件、本提示词、`RootContentView.swift` 和相关文档。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

若 `HEAD` 与 `origin/main` 不一致，应先按项目规则处理同步，不得在落后分支上实现。

3. 在 `RootContentView.swift` 中为 entitlement rules 增加小型 UI 数据结构，建议字段：
   - `title`
   - `detail`
   - `status`
   - `systemImage`
4. 增加规则状态枚举，状态建议包括：
   - `protected`
   - `candidate`
   - `requiresConfiguration`
   - `notImplemented`
5. 在 `PlanView` 中新增规则数据：
   - `Core local tools`
   - `Paid feature candidates`
   - `StoreKit purchase gate`
   - `Entitlement persistence`
6. 在 compact `Form` 中把 `Entitlement Rules` 放在 `Capability Matrix` 之后、`Availability` 之前。
7. 在 regular 右栏中把 `Entitlement Rules` 放在 `Capability Matrix` 之后、`Availability` 之前。
8. 更新 `Availability` 文案，让它与新规则区互补，避免重复过多。
9. 同步文档：
   - `README.md`：说明 Plan 现在展示 entitlement rules baseline，仍无购买/entitlement 行为。
   - `md/flow/flow.md`：更新 Plan UI 职责和未来付费扩展点。
   - `md/flow/flowchart.md`：更新 Plan 节点。
   - `update_log.md`：追加 v1.7 记录。
