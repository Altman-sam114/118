# v1.56 / Plan 权益规则提示语义

## 版本号

v1.56

## 版本分配依据

- v1.53 已为 Plan Availability rows 增加行级 VoiceOver hint。
- v1.54 已为 Plan Mac readiness rows 增加行级 VoiceOver hint。
- v1.55 已为 Plan Capability Matrix rows 增加行级 VoiceOver hint。
- 本轮继续补齐 Plan 页付费规划语义，为 Entitlement Rules rows 增加行级 hint。

## 背景

Plan 页当前通过 Entitlement Rules 区说明当前 Local tools 受保护、paid candidates 仍是 planning only、StoreKit purchase gate 需要配置、entitlement persistence 尚未实现。`PlanStatusRow` 已提供 label/value，Capability、Availability 和 Mac readiness 已在各自 wrapper 上追加 item-specific hint。Entitlement Rules 仍只朗读 title/detail/status，缺少对每条规则的行级解释。

## 目标

- 为 Entitlement Rules 的所有 row 增加行级 VoiceOver hint。
- 对四类规则分别说明：
  - Core local tools 保持当前 Local plan 可用。
  - Paid candidates 仅规划，当前未售卖、未解锁、未授予 entitlement。
  - StoreKit purchase gate 需要 product IDs、entitlement mapping、restore、receipts 和 App Store Connect。
  - Entitlement persistence 当前没有存储 purchase state、没有授予 entitlement、没有请求 App Store product。
- 保持所有可视文案、状态、图标、顺序、布局和业务行为不变。

## 非目标

- 不接入 StoreKit。
- 不新增 product IDs、purchase/restore/receipt/subscription、entitlement persistence 或 paid gates。
- 不启用 Mac Catalyst。
- 不修改 Xcode project、native XCFramework、native bridge、SwiftData schema、文件存储、推理流程或 CI workflow。
- 不修改 shared `PlanStatusRow` 的通用行为，避免影响其他 Plan rows。
- 不新增 simulator VoiceOver 自动化、StoreKit 测试或真实购买测试。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，Plan 页面在 `LocalDiffusion/Views/RootContentView.swift`。
- `PlanEntitlementRuleItem` 是 Entitlement Rules 的数据源，`EntitlementRuleRow` 是行级 wrapper。
- 现有 `CapabilityRow`、`AvailabilityRow` 和 `MacReadinessRow` 已使用 wrapper 追加 `.accessibilityHint(item.accessibilityHint)`，本轮应复用该局部模式。

## 实现步骤

1. 在 `PlanEntitlementRuleItem` 增加 `accessibilityHint: String` 字段。
2. 为 `entitlementRuleItems` 四个元素填入对应 hint。
3. 在 `EntitlementRuleRow` 的 `PlanStatusRow(...)` 后追加 `.accessibilityHint(item.accessibilityHint)`。
4. 不改 title/detail/status/systemImage、row 顺序、layout、StoreKit facts 或 entitlement facts。
5. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
