# v1.86 / Plan Current Build purchase note 文案

## 版本号

v1.86

## 版本分配依据

- v1.85 已完成 Plan Platform Status Mac support note 文案优化，并通过 `origin/main` 云端结果包验收。
- 本轮继续做小步、可验收的 Plan 透明度优化，聚焦 Current Build 中 StoreKit purchase note 的可见文案和 VoiceOver hint。

## 背景

Plan 已经在 Capability Matrix、Entitlement Rules 和 Availability 中说明 paid candidates 是 planning-only，当前未售卖、未解锁。Current Build 的 StoreKit note 只说 purchases、restore、receipts、subscriptions 和 entitlements 未启用，还没有直接说明屏幕上的 paid candidates 只是规划，不能在这里购买或解锁。为了减少用户误解，本轮只补强该 note 的 visible text 和 VoiceOver hint。

## 目标

- 将 `StoreKit purchase status` note 的 text 改为 `Purchases, restore, receipts, subscriptions, and entitlements are not enabled in this build; paid candidates on this screen are planning only.`。
- 将该 note 的 accessibility hint 改为 `This build does not include purchase, restore, receipt, subscription, or entitlement flows, and paid candidates cannot be bought or unlocked here.`。
- 保持 `accessibilityLabel: "StoreKit purchase status"`、`systemImage: "exclamationmark.triangle"`、`iconColor: SciFiTheme.amber` 不变。
- 保持 Current Build summary rows、Capability Matrix、Entitlement Rules、Availability、Mac Readiness、Plan layout 和业务逻辑不变。

## 非目标

- 不启用 StoreKit，不新增 purchase UI、restore flow、receipt、subscription、entitlement persistence、paid gate、product IDs、App Store Connect 请求或真实购买流程。
- 不修改 Mac Catalyst、Xcode platform、target、scheme、signing、native Mac/Catalyst slice、SwiftData schema、文件存储、workflow 或推理链路。
- 不修改 Plan 数据结构、状态枚举、row 顺序、导航结构、布局、Capability Matrix、Entitlement Rules、Availability rows 或 Mac Readiness rows。
- 不新增截图验收、simulator smoke、真实 StoreKit 测试、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责展示当前计划和状态。
- Current Build 中的 purchase note 通过 `PlanNoteRow` 渲染；该 row 已使用可换行文本并通过 `accessibilityLabel` 与 `accessibilityHint` 暴露给 VoiceOver。
- 该 note 是静态文案，不触碰状态层、SwiftData、文件存储、native backend、StoreKit 或购买门禁。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `currentBuildContent` 和 `PlanNoteRow`。
2. 只修改 `StoreKit purchase status` note 的 `text` 与 `accessibilityHint`。
3. 不改 `accessibilityLabel`、`systemImage`、`iconColor`、Current Build summary rows、Plan layout 或业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
