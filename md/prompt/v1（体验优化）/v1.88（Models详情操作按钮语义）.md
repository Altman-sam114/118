# v1.88 / Models 详情操作按钮语义

## 版本号

v1.88

## 版本分配依据

- v1.87 已完成 Prompt Library Add toolbar 语义优化，并通过 `origin/main` 云端结果包验收。
- 本轮继续非 Plan 的体验优化，聚焦 Models detail sheet 操作按钮的 VoiceOver 上下文，让详情页与 v1.39 已完成的 Models 列表行按钮语义保持一致。

## 背景

Models 列表行的 `ModelRow.controlButton(...)` 已经把 `Model Details`、`Delete Model`、`Pause Download`、`Cancel Download`、`Resume Download` 和 `Download Model` 暴露为带模型名上下文的 VoiceOver label，例如 `Delete Model: <model name>`，并提供 `Applies this action to <model name>.` 的 hint。

但 `ModelDetailView.detailControls` 内的详情 sheet 操作按钮仍只暴露通用 `Label("Delete Model", ...)`、`Label("Pause Download", ...)` 等文本。用户能知道动作，但不能像列表行一样听到具体模型上下文。本轮只补齐详情 sheet 的模型名语义，不改下载、暂停、取消、删除或恢复行为。

## 目标

- 在 `ModelDetailView` 内新增一个私有 `detailControlButton(...)` helper。
- 用该 helper 统一生成 `detailControls` 中的：
  - `Delete Model`
  - `Pause Download`
  - `Cancel Download`
  - `Resume Download`
  - `Download Model`
- 每个详情操作按钮都增加：
  - `accessibilityLabel(Text("\(title): \(model.name)"))`
  - `accessibilityHint(Text("Applies this action to \(model.name)."))`
- 保留按钮 visible title、SF Symbol、颜色、role、状态分支、action 顺序和现有 `dismiss()` 调用时机不变。

## 非目标

- 不修改下载状态机、暂停/取消/恢复/删除逻辑、`HuggingFaceDownloadManager`、`AppFileStore` 或 SwiftData schema。
- 不修改 `ModelRow.controlButton(...)`，它是本轮对齐的既有模式来源。
- 不修改 Model detail sheet 的表单结构、Native Loading picker、辅助模型选择、status rows、message rows 或 close toolbar。
- 不修改 StoreKit、paid gates、Mac Catalyst、Xcode project、native backend、bridge、XCFramework、workflow 或 CI。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Models 页面位于 `LocalDiffusion/Views/Models/ModelLibraryView.swift`，属于 UI 展示层。
- `ModelDetailView` 只展示模型元数据和触发外部注入的操作闭包；下载状态和文件处理仍由 `HuggingFaceDownloadManager`、`AppFileStore` 和外层视图负责。
- `detailControls` 已按 `progress.status` 决定按钮集合，本轮只改变按钮的辅助功能语义。

## 实现步骤

1. 阅读 `ModelRow.controlButton(...)` 和 `ModelDetailView.detailControls`。
2. 在 `ModelDetailView` 内新增 `detailControlButton(...)` helper，参数包含 title、systemImage、color、role 和 action。
3. 将 `detailControls` 中现有按钮替换为 helper 调用。
4. 对每个状态分支核对按钮集合、颜色、role 和 action 顺序不变，尤其 `Delete Model` 仍先 `dismiss()` 再 `delete()`。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
