# v1.70 / Plan 导航规划图标

## 版本号

v1.70

## 版本分配依据

- v1.69 已完成 Plan 概览装饰图标从支付语义到中性规划语义的调整，并通过 `origin/main` 云端结果包验收。
- 本轮继续收窄 Root 层 Plan 入口的视觉语义，让进入页面前的 Tab/sidebar 图标与 Plan 页面 planning-only、StoreKit not configured 的事实一致。

## 背景

Plan 页面当前反复说明 StoreKit 未配置、无 purchase state、无 entitlement、付费候选仍是 planning-only。v1.69 已把 Plan 概览装饰图标改成 `checklist`，但 Root 层 `AppSection.plan.systemImage` 仍使用 `creditcard`，会同时出现在 compact TabView 和 iPad sidebar，仍可能暗示购买入口。

## 目标

- 将 Root 层 Plan 主导航图标从支付语义调整为中性规划/清单语义。
- 保持 TabView/sidebar 结构、导航状态、sidebar 文本和 accessibility label/value/hint 不变。
- 保持 Plan 页面事实、付费规划状态和所有业务逻辑不变。

## 非目标

- 不修改 Plan 页面文案、Plan 概览、能力矩阵、Entitlement Rules、Availability、Mac readiness、StoreKit 状态、product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不启用 Mac Catalyst，不修改 Xcode project、workflow、SwiftData、文件存储、native backend 或推理链路。
- 不新增截图验收、simulator smoke、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- `AppSection.systemImage` 是 Root 层导航视觉入口，驱动 compact `TabView` 和 iPad `NavigationSplitView` sidebar。
- `SidebarSectionRow` 已提供 `section.title`、selected/not-selected value 和 workspace hint；本轮只替换 icon 名称，不改变可访问性文本。
- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `AppSection.systemImage`、TabView、sidebar row 和 Plan 页面状态。
2. 只将 `AppSection.systemImage` 的 `case .plan: "creditcard"` 改为 `case .plan: "checklist"`。
3. 不修改 `sections` 顺序、TabView tag、sidebar selection binding、`SidebarSectionRow`、Plan 页面内容或 StoreKit/Mac 相关事实。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
