# v1.121 / 参数控件窄宽度回退

## 版本号

- 版本：`v1.121`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.120 / Generate iPad 布局宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.120 已让 Generate iPad regular 窄宽度可回退单列；本轮继续收敛单列内共享参数控件的横向压缩风险。

## 背景

`ParameterEditor` 是 Generate 和 Prompt Library 模板编辑器共用的参数 UI。当前 `seedControl` 和 `sizePresetControl` 只有在 accessibility Dynamic Type 下才改为纵向布局，普通 Dynamic Type 仍固定使用 `HStack`。在 iPad Split View、Stage Manager、窄 regular 宽度或较长本地化文案下，这类行内控件容易横向挤压：Seed 文本框和随机按钮可能互相抢宽，Size label 和 preset menu 也可能压缩。

项目内已经在 Plan 和 Generate 布局选择中使用 `ViewThatFits(in: .horizontal)` 做宽度回退。本轮将同一思路用于共享参数控件，保持普通宽度下横排，空间不足时回退既有纵向结构。

## 目标

1. 只优化 `ParameterEditor.seedControl` 和 `ParameterEditor.sizePresetControl` 的普通 Dynamic Type 分支。
2. 使用 `ViewThatFits(in: .horizontal)` 优先显示现有横排结构，横向空间不足时回退纵向结构。
3. 保持 accessibility Dynamic Type 仍直接使用纵向布局。
4. 保持 Seed、Randomize Seed、Canvas size preset 的绑定、文案、按钮样式、辅助功能 label/value/hint 和 44pt 命中区不变。
5. 让 Generate 与 Prompt Library 中复用的参数控件同时受益。

## 非目标

- 不修改 steps、CFG、width、height、sampler 控件。
- 不修改 `GenerationParameters`、参数归一化、生成门禁、prompt/model/result/Gallery handoff、SwiftData、file storage、native backend 或 StoreKit。
- 不调整 `GenerationView` 布局、Prompt Library 布局、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager 截图目检；本轮用 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `ParameterEditor` 位于 `LocalDiffusion/Views/Shared/ParameterEditor.swift`，属于共享 SwiftUI UI 展示层。
- `GenerationView` 和 `PromptLibraryView` 都复用 `ParameterEditor(parameters:)`。
- `seedControl` 已有 horizontal 和 accessibility vertical 两套内容，`randomizeSeedButton` 在 accessibility / non-accessibility 分支保持不同可见 label style。
- `sizePresetControl` 已有 horizontal 和 accessibility vertical 两套内容，`sizePresetMenu` 已提供 label/value/hint。
- 本轮只改布局选择，不触碰状态层、数据层、文件存储或 native backend。

## 实现步骤

1. 阅读 `ParameterEditor.seedControl`、`randomizeSeedButton`、`sizePresetControl` 和 `sizePresetMenu`。
2. 在 `seedControl` 中保留 accessibility Dynamic Type 直接纵向。
3. 将普通 Dynamic Type 的固定 `HStack` 改为：

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        seedField
        randomizeSeedButton
    }

    VStack(alignment: .leading, spacing: 8) {
        seedField
        randomizeSeedButton
    }
    .frame(maxWidth: .infinity, alignment: .leading)
}
.foregroundStyle(SciFiTheme.primaryText)
```

4. 在 `sizePresetControl` 中保留 accessibility Dynamic Type 直接纵向。
5. 将普通 Dynamic Type 的固定 `HStack` 改为：

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        Text("Size")
        Spacer()
        sizePresetMenu
    }

    VStack(alignment: .leading, spacing: 8) {
        Text("Size")
        sizePresetMenu
    }
    .frame(maxWidth: .infinity, alignment: .leading)
}
.foregroundStyle(SciFiTheme.primaryText)
```

6. 不修改 `randomizeSeedButton` 和 `sizePresetMenu` 的样式或辅助功能语义。
7. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明共享参数控件支持窄宽度回退。
8. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
