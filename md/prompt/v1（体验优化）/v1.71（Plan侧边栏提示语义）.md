# v1.71 / Plan 侧边栏提示语义

## 版本号

v1.71

## 版本分配依据

- v1.70 已完成 Root 层 Plan 主导航图标从支付语义到中性规划语义的调整，并通过 `origin/main` 云端结果包验收。
- 本轮继续收窄 iPad sidebar 的 Plan 入口可访问语义，让 VoiceOver hint 与 Plan 页面实际包含的 planning-only paid capability status 一致。

## 背景

Root 层 `AppSection.sidebarAccessibilityHint` 为 iPad sidebar 行提供 workspace hint。Plan 入口当前提示为 “Opens local plan and platform readiness.”，没有提到该页面还展示 planning-only paid capability status。Plan 页面本身已经明确说明 StoreKit 未配置、无 purchase state、无 entitlement、付费候选仍是 planning-only，本轮只把 sidebar 入口 hint 与这个事实对齐。

## 目标

- 更新 Plan sidebar accessibility hint，说明该入口打开 Local plan、planning-only paid capability status 和 platform readiness。
- 保持 sidebar label/value、selected/not-selected 状态、44pt 命中区、hover affordance、TabView 和导航状态不变。
- 保持 Plan 页面事实、付费规划状态和所有业务逻辑不变。

## 非目标

- 不修改可见 UI 文案、Plan 页面文案、Plan 图标、能力矩阵、Entitlement Rules、Availability、Mac readiness、StoreKit 状态、product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不启用 Mac Catalyst，不修改 Xcode project、workflow、SwiftData、文件存储、native backend 或推理链路。
- 不新增截图验收、simulator smoke、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- `SidebarSectionRow` 使用 `section.sidebarAccessibilityHint` 作为 iPad sidebar row 的 VoiceOver hint。
- `AppSection.sidebarAccessibilityHint` 是 Root 层导航语义集中点，本轮只改 Plan 分支字符串。
- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `AppSection.sidebarAccessibilityHint` 和 `SidebarSectionRow`。
2. 只修改 `case .plan` 的 hint 字符串，加入 planning-only paid capability status。
3. 不修改 `systemImage`、`sections` 顺序、TabView tag、sidebar selection binding、`SidebarSectionRow` 布局、Plan 页面内容或 StoreKit/Mac 相关事实。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
