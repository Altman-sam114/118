# v1.33 / Plan 付费候选可用性

## 版本号

v1.33

## 版本分配依据

- 当前体验优化阶段已完成到 v1.32。
- v1.4、v1.7 已建立 capability matrix 和 entitlement rules，说明 Batch queue、Curated prompt packs、Workflow export 仍是未来付费候选。
- v1.19 已将 Plan Availability 区升级为可读状态行，v1.32 已补充 Mac app Not enabled，但 Availability 仍缺少直接表达 paid candidates 当前不可购买、不可解锁的状态。
- 本轮继续人工总目标中的 UI、iPad/Mac 事实展示和付费能力规划优化，选择一个可独立验收的小步：让 Plan 直接显示 paid candidates 仍为 planning-only，不新增 StoreKit 或 entitlement 行为。

## 背景

Plan 页面已经展示当前 Local plan、StoreKit 未配置、Mac app 未启用、capability matrix、entitlement rules 和 availability rows。当前 Capability Matrix 与 Entitlement Rules 已经列出 paid candidate 相关事实，但 Availability 区只列出 Core local tools、Purchase UI 和 Mac app；用户快速扫读 Availability 时仍不能直接看到 Batch queue、Curated prompt packs、Workflow export 当前没有售卖或解锁。

此外，Entitlement Rules 里的 `.candidate` 状态显示为 `Candidate`，容易被误读成已有付费计划或已有权益候选。将可见状态收敛为 `Planning only`，并在 Availability 区新增同名状态行，可以让付费边界更清晰。

## 目标

- 在 `LocalDiffusion/Views/RootContentView.swift` 的 `availabilityItems` 中新增一个非交互 `PlanAvailabilityItem`：
  - title：`Paid candidates`
  - detail：说明 Batch queue、curated prompt packs、workflow export 当前未售卖、未解锁，仍需要 product decisions、StoreKit products 和 entitlement mapping。
  - status：`Planning only`
  - status icon：`clock`
  - status color：`SciFiTheme.cyan`
  - row icon：`sparkles`
- 将 `PlanEntitlementRuleStatus.candidate.title` 从 `Candidate` 调整为 `Planning only`。
- 将 entitlement rule item 标题从 `Paid feature candidates` 调整为 `Paid candidates`，detail 与 Availability 行保持一致但不重复过长。
- 复用现有 `PlanAvailabilityItem -> AvailabilityRow -> PlanStatusRow -> PlanStatusBadge`。
- 保持 Plan compact Form、iPad regular 双栏/单列回退、状态 badge 视觉和 VoiceOver label/value 结构不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不新增 `import StoreKit`、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不把 Batch queue、Curated prompt packs、Workflow export 描述成已售卖、可购买、已解锁或已有 entitlement。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、Xcode project、签名、分发或 scheme 设置。
- 不修改 native XCFramework、native bridge ABI、Release asset、CI workflow 或 Mac/Catalyst slice。
- 不修改 Root 导航、compact `TabView`、iPad `NavigationSplitView`、Generate、Models、Gallery、Prompts、SwiftData schema、文件存储、native backend 或生成流程。
- 不做 simulator screenshot、VoiceOver 目检、Mac build、StoreKit 测试或真机 GGUF 生成验收。

## 当前架构依据

- `PlanView` 位于 `RootContentView.swift`，属于 SwiftUI UI 展示层，只陈述当前 Local plan、StoreKit 未配置、Mac 未启用和未来候选能力。
- `availabilityItems` 是 Plan Availability 区的数据源，渲染通过 `AvailabilityRow` 复用 `PlanStatusRow` 的可读视觉和 VoiceOver label/value 语义。
- `entitlementRuleItems` 是 Plan Entitlement Rules 区的数据源，本轮只调整 candidate 的可见状态文案和说明，不引入真实 entitlement。
- 当前 StoreKit purchase gate、entitlement persistence 都明确为未配置/未实现；本轮必须保持这些事实不变。
