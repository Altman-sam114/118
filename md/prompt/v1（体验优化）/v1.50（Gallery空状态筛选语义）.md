# v1.50 / Gallery 空状态筛选语义

## 版本号

v1.50

## 版本分配依据

- 当前体验优化阶段已完成到 v1.49。
- Gallery 已具备 filter rail 计数、Sort 当前值、图块摘要、详情组织控件等 VoiceOver 语义。
- `GalleryView.imageGrid` 在 `visibleImages.isEmpty` 时仍使用通用 `EmptyStateView("No images")`，没有向 VoiceOver 暴露当前筛选条件，用户无法判断是整个图库为空，还是当前 folder/tag filter 没有结果。
- 本轮选择 Gallery 空状态的小步语义优化，不修改 Gallery 筛选、排序、导航、删除、复用、再生成、文件 reconcile 或 SwiftData 行为。

## 背景

Gallery 支持 All Images、Folder、Tag 三类筛选。空网格可能代表没有任何生成图片，也可能代表某个 folder 或 tag 当前没有匹配图片。当前可视文案固定为 “No images / Generated images will appear here.”，VoiceOver 也只会从通用 empty state 读取可见内容，缺少当前 filter context。

本轮只为 `visibleImages.isEmpty` 分支补充组合 VoiceOver label/value/hint。视觉布局、文案、filter title、image count、排序、列表、NavigationLink 和 detail 行为保持不变。

## 目标

- 在 `LocalDiffusion/Views/Gallery/GalleryView.swift` 的 `imageGrid` 空状态分支中：
  - 保留现有 `EmptyStateView(systemImage:title:message:)` 和 `.padding()`。
  - 增加 `.accessibilityElement(children: .ignore)`。
  - 增加 `.accessibilityLabel("Gallery empty state")`。
  - 增加 `.accessibilityValue(...)`，说明当前筛选条件与 `0 images`。
  - 增加 `.accessibilityHint(...)`，说明可以生成图片或更改 gallery filter。
- 新增的 value/hint 可以通过 `private var` helper 生成，必须使用现有 `selectedFilterTitle`，避免复制 filter title 逻辑。
- 同步 README、flow、flowchart、update_log，记录 Gallery 空状态会暴露当前筛选上下文。

## 非目标

- 不修改 `GalleryFilter`、`visibleImages`、`filterBinding`、`selectedFilterTitle`、`selectedFilterLabel` 或 filter row 行为。
- 不修改 Gallery sort、folder/tag 计数、NavigationSplitView/NavigationStack、image detail、delete、share、reuse、regenerate 或 reconcile 行为。
- 不修改 SwiftData schema、文件存储、GenerationViewModel、native backend、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不新增 StoreKit import、product ID、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线、Mac build、StoreKit 测试或真机 GGUF 生成验收。

## 当前架构依据

- `GalleryView` 属于 `LocalDiffusion/Views/Gallery/GalleryView.swift` 的 UI 展示层，只负责 Gallery 展示、筛选、导航和用户操作。
- `visibleImages` 已经集中表达当前 filter + sort 后的图片集合，本轮只在集合为空时改善 VoiceOver 输出。
- `selectedFilterTitle` 已经是当前 filter 的权威 UI 文本来源，空状态辅助功能文案应复用它。
- 辅助功能 label/value/hint 属于 UI 表达，不应进入 ViewModel、SwiftData、文件存储或 native 边界。
