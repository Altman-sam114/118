# v1.98 / Models 未跟踪文件删除确认语义

## 版本号

v1.98

## 版本分配依据

- v1.97 已完成 Models tracked model delete confirmation buttons 的模型名/多模型数量上下文和确认/取消 VoiceOver 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续做窄范围 UI accessibility 优化，转向 Models 的 untracked GGUF file 删除确认弹窗，补齐 destructive / cancel 按钮的文件名上下文。

## 背景

Models untracked file row 的 Import/Delete 操作已经提供具体文件上下文，但 `.confirmationDialog("Delete untracked model file?")` 内的两个按钮目前只有可见文案：

- destructive `Delete File`；
- cancel `Cancel`。

弹窗 message 已说明会从 Application Support 删除未关联 metadata 的 app-managed GGUF file，但按钮本身没有 pending filename，也没有明确确认删除或取消保留文件的辅助功能说明。

## 目标

- 为 untracked model file delete confirmation dialog 的 destructive `Delete File` button 增加：
  - accessibility label：`Delete untracked model file: <filename>`；
  - accessibility value：待删除 filename；
  - accessibility hint：说明会从 Application Support 删除该未跟踪 GGUF file，且它未连接到 model metadata。
- 为 cancel `Cancel` button 增加：
  - accessibility label：`Cancel deleting untracked model file: <filename>`；
  - accessibility value：待删除 filename；
  - accessibility hint：说明保留该 local file 并关闭确认。
- 可新增小型 private helper，用于从 `pendingUntrackedModelDeletion?.filename` 生成稳定 filename。
- 保留 dialog title、message、按钮可见文案、destructive/cancel roles、`performPendingUntrackedModelDeletion()` 行为、`pendingUntrackedModelDeletion = nil` 清理和 `untrackedModelDeleteBinding` 不变。

## 非目标

- 不修改 `performPendingUntrackedModelDeletion()` 的数据行为：不得改变 `fileStore.removeModelFile`、storage refresh、untracked scan refresh、错误提示或 pending state 清理。
- 不修改 tracked model delete flow、download pause/cancel/resume、untracked file import editor、import flow、storage reconcile、Add Model、row actions 或 detail controls。
- 不修改 SwiftData schema、file storage API、`GenerationViewModel`、native backend、bridge、workflow、Xcode project、StoreKit、paid gates、Mac Catalyst 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Models 页面位于 `LocalDiffusion/Views/Models/ModelLibraryView.swift`，属于 UI 展示层。
- `UntrackedModelFile` 已保存待删除文件的 `filename` 和 `bytes`，可直接用于 confirmation dialog button accessibility。
- `performPendingUntrackedModelDeletion()` 负责删除 Application Support 中未跟踪 GGUF file；本轮只改确认按钮语义，不进入文件存储 API、下载状态机或数据层边界。

## 实现步骤

1. 阅读 `.confirmationDialog("Delete untracked model file?")`、`pendingUntrackedModelDeletion`、`untrackedModelDeleteBinding`、`untrackedModelDeleteMessage`、`performPendingUntrackedModelDeletion()` 和 `UntrackedModelFile`。
2. 新增 private helper 获取待删除 filename。
3. 在 destructive `Button("Delete File", role: .destructive)` 后追加 accessibility label/value/hint。
4. 在 cancel `Button("Cancel", role: .cancel)` 后追加 accessibility label/value/hint。
5. 核对没有改动 dialog title/message、button roles、visible copy、performPendingUntrackedModelDeletion 行为、pending state 清理、tracked model delete flow 或 download state machine。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
