# v1.1（Generate 宽屏创作台）

## 版本号

v1.1

## 版本分配依据

v1.0 已完成 iPad Root/Gallery 宽屏导航基线，消除了 Gallery 在 Root detail 中嵌套 `NavigationSplitView` 的风险。人工总目标继续要求优化 iPad/mac/UI/设计/付费功能。本轮沿着 iPad 体验继续推进，选择 Generate 页宽屏创作台作为小步产品体验优化；macOS/Catalyst 和付费功能仍不混入本轮。

## 背景

当前 `GenerationView` 是单个 `NavigationStack + Form`。在 iPhone compact 上这很合适，但在 iPad regular 上，模型、prompt、参数、运行状态、结果预览都堆在一列，宽屏空间利用不足。Generate 是核心工作流入口，先稳定它的 iPad 创作台布局，有助于后续继续设计 mac/Catalyst 版本和付费功能入口。

## 目标

- iPhone compact 保持现有单列 `Form` 体验。
- iPad regular 在 Generate 内部使用双栏创作台布局：
  - 左栏：模型选择、positive/negative prompt、参数。
  - 右栏：状态摘要、运行 gate、Generate/Cancel 操作、结果预览与跳转 Gallery。
- 保留 `NavigationStack`，不要在 Generate 内新增 `NavigationSplitView`。
- 保留 Save Template toolbar、alert、默认模型选择、Generate/Cancel、Open Models、Edit Prompt、View in Gallery 等现有行为。
- 保持暗色科幻视觉，使用现有 `SciFiTheme`、按钮样式、8px 圆角和 `Form` section 样式。

## 非目标

- 不修改 `GenerationViewModel`、SwiftData schema、文件存储、native backend 或 bridge。
- 不启用 Mac Catalyst，不新增 macOS target，不修改 Xcode project 平台设置。
- 不实现付费功能、StoreKit、订阅、账号或 entitlement。
- 不把 `ParameterEditor` 从 `Form` 语义中强行抽离；如复用应仍在 `Form` 内呈现。
- 不做大范围视觉重设计，不引入新第三方依赖。

## 当前架构依据

- `RootContentView` 负责全局 Generate/Models/Gallery/Prompts 导航。
- `GenerationView` 只负责展示、输入、用户反馈和调用 `GenerationViewModel` 的既有方法。
- UI 只能依赖 `ImageGenerationBackend` 边界，不能直接触碰 native bridge。
- `ParameterEditor` 当前返回 `Section("Parameters")`，适合继续放在 `Form` 中。
- SwiftUI 宽屏布局应避免固定全屏尺寸，使用响应式 frame、scroll 和现有 Dynamic Type 友好的字体。

## 实现步骤

1. 阅读必读文件、本提示词、`GenerationView.swift`、`ParameterEditor.swift`、`RootContentView.swift` 和 `md/test/test.md`。
2. 确认当前分支和工作区：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short`
3. 在 `GenerationView` 中读取 `horizontalSizeClass`。
4. 将现有 `Form` 内容拆成可复用的紧凑单列内容和宽屏双栏内容。可以使用私有 helper view 片段，但不要引入跨文件抽象，除非能明显降低复杂度。
5. compact/standalone 路径应尽量保持现有 section 顺序和行为。
6. regular 路径保留一个 `NavigationStack`，内部用 `HStack` 或等价布局呈现两个 `Form`/scroll 区域：
   - 左侧宽度有合理 min/max，承载 Model、Prompts、Parameters。
   - 右侧承载当前顶部 console 摘要、Run section、Result section。
7. 右栏在没有结果时仍显示运行 gate 和主要操作，不显示空白大卡片。
8. 保持 `onAppear` 默认选中 ready model、Save Template sheet、alert、focus prompt、Open Models、View in Gallery 等行为。
9. 同步文档：
   - `README.md`：Current implementation 说明 Generate 在 iPad 上有双栏创作台。
   - `md/flow/flow.md`：Generate UI 职责补充 compact 单列 / iPad 双栏。
   - `md/flow/flowchart.md`：如现有图不准确，补充 Generate 宽屏布局节点。
   - `update_log.md`：追加 v1.1 记录。

## 验收标准

- iPhone compact 仍使用现有单列 Generate 表单体验。
- iPad regular 的 Generate 呈现双栏创作台，宽屏空间用于分离输入与运行/结果。
- 所有 Generate 既有行为保留：保存模板、默认模型选择、生成门禁、取消生成、跳转 Models、聚焦 prompt、查看 Gallery。
- 本地轻量检查和两条 Swift parse 通过。
- `origin/main` 最新 v1.1 run 成功，Agent C 下载并核对结果包通过。

## 风险和禁止项

- 禁止在本轮新增 macOS/Catalyst target 或付费功能。
- 禁止改动 native backend、SwiftData schema 或文件存储。
- 禁止把 `ParameterEditor` 放到非 `Form` 容器导致 section 样式破损。
- 禁止为了双栏布局删除现有 Generate 功能。
- 禁止用本地检查替代 Agent C 最新 artifact 核对。
