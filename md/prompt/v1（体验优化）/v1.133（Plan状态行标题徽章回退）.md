# v1.133 / Plan 状态行标题徽章回退

## 版本号

- 版本：`v1.133`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.132 / Plan 状态摘要行宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.132 已改善 Current Build 和 Platform Status summary rows 的宽度回退；本轮继续在 Plan 页面内推进同一类 iPad/Mac 阅读密度优化，但只扩展到共享 `PlanStatusRow` 的标题/状态区域。

## 背景

`PlanStatusRow` 负责 Mac readiness、Capability Matrix、Entitlement Rules 和 Availability 的普通状态行。它当前始终按标题、detail、状态徽章纵向排列。该布局在 compact 和 accessibility Dynamic Type 下可读，但在 iPad regular 双栏中，很多短标题和短状态仍占用额外垂直空间。为了保持 Plan 页面在未来 Mac/Catalyst 窗口验证前更接近桌面阅读节奏，本轮让普通 Dynamic Type 优先把标题和状态徽章放在同一行，空间不足时回退为既有纵向布局。

## 目标

1. 修改 `PlanStatusRow` 的内部布局。
2. 普通 Dynamic Type 下使用 `ViewThatFits(in: .horizontal)`：
   - 优先显示横向 header：左侧标题 label，右侧非全宽状态徽章。
   - 横向空间不足时回退既有纵向布局。
3. accessibility Dynamic Type 下直接保持既有纵向布局。
4. detail 文本继续在标题/状态区域下方可换行显示，保留 `.fixedSize(horizontal: false, vertical: true)`。
5. 保留所有可见文案、row 顺序、状态 token、颜色、图标、VoiceOver label/value/hint、Plan 数据和 StoreKit/Mac 边界不变。

## 非目标

- 不启用 StoreKit、购买按钮、restore flow、receipt validation、entitlement persistence、feature flag 或 paid unlock gate。
- 不修改 paid candidates 的业务含义、付费文案或可购买状态。
- 不修改 Mac Catalyst、Xcode project platform settings、native backend slice、signing、sandboxing、notarization 或 workflow。
- 不修改 `PlanStatusSummaryRow` 的 v1.132 行为，除非为了复用已存在的 `PlanStatusBadge(fillsWidth:)`。
- 不修改 Generate、Models、Gallery、Prompts、SwiftData、AppFileStore、GenerationViewModel、native backend 或测试规范。
- 不做 simulator / Mac Catalyst / Stage Manager 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `RootContentView.swift` 中的 `PlanView` 属于 SwiftUI UI 展示层，只负责展示 Local plan、付费候选规划状态、StoreKit 边界和平台 readiness。
- `PlanStatusRow` 是 Plan 页面内部共享 UI 组件，服务 Mac readiness、Capability Matrix、Entitlement Rules 和 Availability。
- `PlanStatusBadge` 在 v1.132 已支持 `fillsWidth` 参数，横向布局可复用非全宽 badge，不需要新增业务模型或状态。
- Plan 页面当前明确不请求 App Store product、不存储 entitlement、不展示购买 UI；本轮不得改变该边界。

## 实现步骤

1. 阅读 `PlanStatusRow`、`PlanStatusBadge`、`MacReadinessRow`、`CapabilityRow`、`EntitlementRuleRow` 和 `AvailabilityRow`。
2. 给 `PlanStatusRow` 读取 `dynamicTypeSize` 环境。
3. 抽出：
   - `headerLabel`：保留当前标题 `Label` 文案、图标和颜色。
   - `detailText`：保留当前 detail 文本样式和 fixed size。
   - `horizontalRow`：使用 `VStack`，顶部 `HStack` 放 `headerLabel`、spacer、`PlanStatusBadge(status:fillsWidth:false)`，下方放 `detailText`。
   - `verticalRow`：保留当前 `Label`、detail、全宽 `PlanStatusBadge` 顺序。
4. `body` 中普通 Dynamic Type 使用 `ViewThatFits(in: .horizontal)` 在 `horizontalRow` 和 `verticalRow` 间选择；accessibility Dynamic Type 直接使用 `verticalRow`。
5. 保持 `.accessibilityElement(children: .ignore)`、`.accessibilityLabel("\(title). \(detail)")` 和 `.accessibilityValue(status.title)` 不变。
6. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Plan status rows 在普通字号下可横排标题/状态并在窄宽度或 accessibility Dynamic Type 下回退。
7. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
