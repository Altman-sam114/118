# v1.127 / Result Gallery 命中区

## 版本号

- 版本：`v1.127`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.126 / Run 操作命中区`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.126 已补齐 Generate Run 区主要和次要操作的 44pt 最小命中区；本轮继续处理 Generate Result 区中生成结果到 Gallery 的关键 handoff 操作。

## 背景

`GenerationView.resultSection` 在生成成功并拿到 `generatedImageData` 后显示预览图和 `View in Gallery` 按钮。该按钮是用户进入 Gallery 检查、打标签、分享或复用已保存结果的主要入口。当前按钮使用 `SciFiSecondaryButtonStyle`，但 call site 没有显式 44pt 最小高度。为了和 v1.126 的 Run 区关键操作、同页 no-model Open Models 入口保持一致，本轮只给 `View in Gallery` 补齐 `.frame(minHeight: 44)`。

## 目标

1. 只修改 `GenerationView.resultSection` 中 `View in Gallery` 按钮布局。
2. 在 `.buttonStyle(SciFiSecondaryButtonStyle())` 后增加 `.frame(minHeight: 44)`。
3. 保持 `onShowGallery()` action、`latestGeneratedImageID == nil` disabled 条件、可见文案、system image、accessibility label/value/hint 不变。
4. 保持 result image preview、保存记录、Gallery handoff 语义和所有生成逻辑不变。

## 非目标

- 不修改 `SciFiSecondaryButtonStyle` 或全局按钮样式。
- 不修改 `resultImageAccessibilityValue`、`galleryHandoffAccessibilityValue`、`galleryHandoffAccessibilityHint`。
- 不修改 `onShowGallery`、`latestGeneratedImageID`、生成保存流程、Gallery 导航、Gallery 页面或 SwiftData。
- 不修改 Run 区按钮、progress、generation gate、model selection、prompt editor、ParameterEditor、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不引入固定宽度、固定总高度、缩小字体、单行限制或截断策略。
- 不做 simulator / Stage Manager / Split View / VoiceOver 触控截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GenerationView` 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，属于 SwiftUI UI 展示层。
- `resultSection` 只在存在生成图片数据时展示预览和 Gallery handoff。
- `View in Gallery` 按钮只调用 `onShowGallery()`，是否可用由 `viewModel.latestGeneratedImageID == nil` 控制。
- 本轮只改变按钮最小命中区约束，不触碰状态层、数据层、文件存储或 native backend。

## 实现步骤

1. 阅读 `resultSection` 中 `View in Gallery` 按钮。
2. 在按钮的 `.buttonStyle(SciFiSecondaryButtonStyle())` 后追加：

```swift
.frame(minHeight: 44)
```

3. 保持后续 `.disabled(...)` 和 accessibility modifiers 原样。
4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Generate result Gallery handoff 按钮显式保持 44pt 最小命中区。
5. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
