# v1.43 / 共享随机种子按钮语义

## 版本号

v1.43

## 版本分配依据

- 当前体验优化阶段已完成到 v1.42。
- v1.13 已为共享 `ParameterEditor` 的 Seed 文本框提供基础 label/value，并改善参数控件 Dynamic Type 可读性。
- v1.42 已补充共享 Reset Defaults 的重置范围和 prompt 保留语义。
- `ParameterEditor` 的 `Randomize Seed` 按钮当前有文本 label、icon-only 视觉分支和 44pt 命中区，但没有暴露当前 seed，也没有说明点击后会生成一个新的随机 seed。
- `ParameterEditor` 同时用于 Generate 和 Prompt Template Editor，因此本轮继续选择共享参数编辑器里的小步 VoiceOver 语义优化，不修改 seed 生成行为。

## 背景

Local Diffusion 的 seed 参数决定同一 prompt 和模型下的可复现生成路径。当前 Seed 文本框会朗读当前数值，但相邻的 Randomize Seed 按钮只朗读按钮名称。VoiceOver 用户聚焦按钮时需要额外切回文本框才能确认当前 seed，也无法从按钮本身听到它会生成新的随机 seed。

本轮只给 Randomize Seed 按钮增加当前 seed value 和操作 hint。普通字号下按钮继续视觉 icon-only，accessibility Dynamic Type 下继续显示 `Label("Randomize Seed", systemImage: "dice")`。按钮 action、随机范围、seed 字段、参数绑定和所有生成/模板行为保持不变。

## 目标

- 在 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的 `randomizeSeedButton` 两个分支上补充：
  - `.accessibilityValue("Current seed \(parameters.seed)")`
  - `.accessibilityHint("Generates a new random seed value.")`
- 保持现有 `Label("Randomize Seed", systemImage: "dice")`、普通字号 `.labelStyle(.iconOnly)`、`SciFiSecondaryButtonStyle(color: SciFiTheme.amber)`、tint 和 44pt frame 不变。
- 语义文案必须通用，不能写死 Generate screen，因为该组件也用于 Prompt Template Editor。
- 同步 README、flow、flowchart、update_log，记录共享随机 seed 按钮获得当前 seed 和新 seed 操作语义。

## 非目标

- 不修改 `randomizeSeed()` 的实现或 `Int.random(in: 0...Int(Int32.max))` 范围。
- 不修改 `seedField`、Seed 文本框格式、keyboard type、seed binding 或 `GenerationParameters` 默认值。
- 不修改 Reset Defaults、steps、CFG、size preset、width/height、sampler 或其他参数控件。
- 不修改 Generate、Prompt Library 的保存/加载模板行为。
- 不修改 SwiftData schema、文件存储、native backend、桥接头、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不修改平台、签名、分发或 scheme 设置。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线或真机 GGUF 生成验收。

## 当前架构依据

- `ParameterEditor` 属于 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的共享 UI 展示层，只负责展示和用户输入。
- Generate 和 Prompt Template Editor 通过 `@Binding var parameters: GenerationParameters` 复用同一个参数编辑器。
- `randomizeSeed()` 当前只在 UI 层写入 `parameters.seed`，不触碰 SwiftData、文件存储、ViewModel 或 native backend。
- 辅助功能语义属于 UI 表达，不应进入 ViewModel、数据层或 native 边界。
