# v1.140 / Prompt 模板行标题窄宽度回退

## 版本号

- 版本：`v1.140`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.139 / Models 存储矩阵标题窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.135 已补齐 metric pills 窄宽度回退；`PromptTemplateRow.header` 在普通 Dynamic Type 下仍固定横排 templateCopy + Edit/Load actions，是下一个对称、可验收的 UI 韧性优化。

## 背景

`PromptTemplateRow.header` 在 accessibility Dynamic Type 下已纵向堆叠，但普通 Dynamic Type 固定 `HStack { templateCopy; Spacer(); actions }`。模板名 + prompt 摘要与 Edit/Load 控件在窄 list / iPad split 下可能挤压。本轮用 `ViewThatFits(in: .horizontal)` 对齐既有模式。

## 目标

1. 只修改 `PromptTemplateRow.header` 的普通 Dynamic Type 分支。
2. `ViewThatFits(in: .horizontal)` 优先横排 templateCopy + actions，不足时回退纵向。
3. accessibility 纵向布局不变。
4. 保留 metric pills、row accessibility summary、Edit/Load 行为与文案不变。

## 非目标

- 不修改 `PromptCategoryHeader`、editor、search、ParameterEditor、SwiftData、native、StoreKit、Mac Catalyst、workflow。
- 禁止本地 Xcode / Simulator / 真机构建测试。

## 实现步骤

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        templateCopy
        Spacer()
        actions
    }

    VStack(alignment: .leading, spacing: 10) {
        templateCopy
        actions
    }
}
```
