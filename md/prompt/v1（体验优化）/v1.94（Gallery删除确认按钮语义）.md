# v1.94 / Gallery 删除确认按钮语义

## 版本号

v1.94

## 版本分配依据

- v1.93 已完成 Gallery detail Folder picker 的即时保存 VoiceOver hint，并通过 `origin/main` 云端结果包验收。
- 本轮继续 Gallery detail 体验优化，聚焦删除确认弹窗中的 destructive / cancel 按钮，让确认阶段也保留当前图片上下文。

## 背景

`ImageDetailView` toolbar 的 `Delete Image` 按钮已经提供：

- `Delete generated image` accessibility label；
- `imageActionAccessibilityValue`；
- 说明会先确认再删除图片文件和 metadata 的 hint。

但 `.confirmationDialog("Delete this generated image?")` 内部的两个按钮目前只有可见文案：

- destructive `Delete Image`；
- cancel `Cancel`。

进入确认弹窗后，VoiceOver 用户可能只能听到通用按钮名，缺少当前图片的 model/seed/size 上下文，也缺少确认删除与取消保留的明确说明。

## 目标

- 为 confirmation dialog 的 destructive `Delete Image` button 增加：
  - accessibility label：明确这是确认删除当前生成图片；
  - accessibility value：复用 `imageActionAccessibilityValue`；
  - accessibility hint：说明会删除图片文件和保存的 generation metadata。
- 为 confirmation dialog 的 cancel `Cancel` button 增加：
  - accessibility label：明确这是取消删除当前生成图片；
  - accessibility value：复用 `imageActionAccessibilityValue`；
  - accessibility hint：说明会保留图片文件和保存的 generation metadata。
- 保留 dialog 标题、message、按钮可见文案、destructive/cancel role、`onDelete(); dismiss()` 调用顺序和 toolbar 删除按钮语义不变。

## 非目标

- 不修改 `deleteImage(_:)`、`onDelete`、`dismiss()`、文件删除、SwiftData 删除、navigation 清理或保存顺序。
- 不新增二次确认、toast、异步删除、撤销、删除队列或 UI 重排。
- 不修改 Reuse/Regenerate/Share、Folder/Tags、filtering、sorting、thumbnail loading、Gallery layout 或 image tile。
- 不修改 `GeneratedImage`、SwiftData schema、`AppFileStore`、`GenerationViewModel`、native backend、bridge、workflow、Xcode project、StoreKit、paid gates、Mac Catalyst 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Gallery 页面位于 `LocalDiffusion/Views/Gallery/GalleryView.swift`，属于 UI 展示层。
- `imageActionAccessibilityValue` 已集中描述当前图片的 model、seed、requested size 和 output size，可直接复用于确认弹窗按钮。
- 删除实际行为由外层 `onDelete` 闭包和 `GalleryView.deleteImage(_:)` 完成；本轮只改变 SwiftUI accessibility 修饰符。

## 实现步骤

1. 阅读 `ImageDetailView` toolbar 删除按钮、confirmation dialog 和 `imageActionAccessibilityValue`。
2. 在 destructive `Button("Delete Image", role: .destructive)` 后追加 accessibility label/value/hint。
3. 在 cancel `Button("Cancel", role: .cancel)` 后追加 accessibility label/value/hint。
4. 核对没有改动按钮 role、可见文案、dialog title/message、`onDelete(); dismiss()`、deleteImage 实现或任何 Gallery 数据流。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
