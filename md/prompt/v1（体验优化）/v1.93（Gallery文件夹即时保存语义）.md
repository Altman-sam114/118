# v1.93 / Gallery 文件夹即时保存语义

## 版本号

v1.93

## 版本分配依据

- v1.92 已完成 Gallery detail Save Tags 的 draft tags、saved tags 与未保存状态 VoiceOver 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续 Gallery detail Organization 区域体验优化，聚焦 Folder picker 的“选择后立即保存”语义，补齐 folder 与 tags 保存时机的差异说明。

## 背景

`ImageDetailView` 的 Organization 区域中：

- Folder picker 使用 `folderBinding`，setter 会立即写入 `image.folderID` 并 `try? modelContext.save()`。
- Tags text field 是草稿态，必须点击 Save Tags 才写入 `image.tags`。
- v1.92 已让 Save Tags 暴露 draft tags、saved tags 和未保存状态。

当前 Folder picker 的 accessibility hint 是 `Assigns this image to a gallery folder.`，没有说明该选择会立即保存。VoiceOver 用户可能误以为 Folder 与 Tags 一样需要额外保存按钮。

本轮只修改 Folder picker 的可访问 hint，让保存时机更明确。

## 目标

- 将 Gallery detail Folder picker 的 accessibility hint 改为明确说明：
  - 该控件会更改当前图片的 folder；
  - 选择会立即保存。
- 保留 `Image folder` accessibility label、`currentFolderAccessibilityValue`、Picker title、options、tags、`folderBinding` getter/setter 和视觉布局不变。
- 保留 Tags text field 与 Save Tags 的 v1.92 语义不变。

## 非目标

- 不新增 Save Folder 按钮。
- 不把 folder selection 改成草稿态。
- 不修改 `folderBinding`、`image.folderID`、`GalleryFolder`、SwiftData schema、folder creation/rename/delete 或 folder filtering。
- 不修改 Tags parsing、draft/saved tags 比较、Save Tags action、reuse/regenerate/share/delete、image deletion、file storage 或 GenerationViewModel。
- 不修改 StoreKit、paid gates、Mac Catalyst、Xcode project、native backend、bridge、workflow 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Gallery 页面位于 `LocalDiffusion/Views/Gallery/GalleryView.swift`，属于 UI 展示层。
- `folderBinding` 当前已经立即保存 folder selection；本轮只让辅助功能文本反映现有事实，不改变数据流。
- UI 不得越过 `ImageGenerationBackend` 或触碰 native bridge；本轮不涉及推理链路。

## 实现步骤

1. 阅读 `ImageDetailView` 的 Organization section、`folderBinding` 和 `currentFolderAccessibilityValue`。
2. 将 Folder picker 的 `accessibilityHint` 改为类似 `Changes this image's folder and saves the selection immediately.`。
3. 核对没有改动 Picker 选项、binding、保存逻辑、Tags section、layout 或样式。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
