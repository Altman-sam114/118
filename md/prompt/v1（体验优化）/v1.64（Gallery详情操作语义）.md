# v1.64 / Gallery 详情操作语义

## 版本号

v1.64

## 版本分配依据

- v1.63 已完成 Prompt editor Save/Cancel accessibility 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段做小步、可验收的 Gallery 详情操作可访问性增强。

## 背景

Gallery 详情页已有图片预览、参数行、Folder/Tags/Save Tags 的 VoiceOver 语义。Actions section 中的 `Reuse Parameters`、`Reuse and Generate`、`Share PNG` 和 toolbar `Delete Image` 仍主要依赖可见标题与基础 hint，VoiceOver 对“当前是哪一张图、会发生什么”的上下文不够集中。

## 目标

- 为 `Reuse Parameters` 按钮补充稳定 label、当前图片上下文 value 和加载参数到 Generate 的提示。
- 为 `Reuse and Generate` 按钮补充稳定 label、当前图片上下文 value 和会立即开始新本地生成的提示。
- 为 `Share PNG` 补充稳定 label、当前图片上下文 value 和分享当前 PNG 文件的提示。
- 为 toolbar `Delete Image` 补充稳定 label、当前图片上下文 value 和先确认再删除文件与元数据的提示。

## 非目标

- 不修改复用参数、再生成、分享、删除、确认弹窗或 dismiss 行为。
- 不修改 Gallery 筛选、排序、folder/tag 保存、SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 GitHub Actions workflow。
- 不新增购买、订阅、entitlement、价格、product ID 或 Mac 平台开关。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。
- Gallery 详情复用通过 `GenerationViewModel.load(image:)` 和 Root navigation handoff 完成，不能直接访问 native bridge。
- 当前核心流程文档要求 Gallery 的详情参数、操作和 Organization 控件在 accessibility Dynamic Type 下保持可读并提供清晰辅助功能语义。

## 实现步骤

1. 阅读 `GalleryView.swift` 中 `ImageDetailView` 的 Actions、Organization 和 toolbar。
2. 保留 `onReuse()`、`onRegenerate()`、`ShareLink(item:)`、delete confirmation 与 `onDelete()` 的现有行为。
3. 增加一个局部 helper，用模型名和输出尺寸组成当前图片操作上下文。
4. 给 Reuse、Regenerate、Share、Delete 操作补充 `accessibilityLabel`、`accessibilityValue` 和更具体的 `accessibilityHint`。
5. 不改按钮样式、布局、启用条件、文件存在判断或确认 dialog。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 语义事实。

## 验收标准

- Reuse、Regenerate、Share、Delete 操作都向 VoiceOver 暴露当前图片上下文。
- Reuse 说明会把参数加载回 Generate。
- Regenerate 说明会加载参数并启动新的本地生成。
- Share 说明会分享当前 PNG 文件。
- Delete 说明会先确认再删除图片文件和 metadata。
- 所有本地轻量检查和 Swift parse 通过。
- `origin/main` 最新 commit 的 CI artifact 经 Agent C 下载核对并通过。
