# v1.97 / Models 删除确认语义

## 版本号

v1.97

## 版本分配依据

- v1.96 已完成 Prompt Library category clear confirmation buttons 的 category 名称上下文和确认/取消 VoiceOver 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续做窄范围 UI accessibility 优化，转向 Models 的 tracked model 删除确认弹窗，补齐 destructive / cancel 按钮的模型名或多选数量上下文。

## 背景

Models row controls 和 detail sheet 操作按钮已经提供模型名上下文，但 `.confirmationDialog("Delete model?")` 内的两个按钮目前只有可见文案：

- destructive `Delete Model` 或 `Delete Models`；
- cancel `Cancel`。

弹窗 message 已说明单模型名称或多模型数量，以及会删除 model metadata 和 local file，但按钮本身没有 pending model 名称/数量上下文，也没有明确确认删除或取消保留模型文件和 metadata 的辅助功能说明。

## 目标

- 为 tracked model delete confirmation dialog 的 destructive button 增加：
  - 单模型 accessibility label：`Delete model: <model name>`；
  - 多模型 accessibility label：`Delete <count> models`；
  - accessibility value：单模型名称，或多模型数量和名称列表；
  - accessibility hint：说明会移除 selected model metadata 和 local file(s) from Application Support。
- 为 cancel `Cancel` button 增加：
  - 单模型 accessibility label：`Cancel deleting model: <model name>`；
  - 多模型 accessibility label：`Cancel deleting <count> models`；
  - accessibility value：与 destructive button 使用同一 pending target 描述；
  - accessibility hint：说明关闭确认并保留 model metadata 和 local file(s)。
- 可新增小型 private helper，基于 `pendingModelDeletion?.modelIDs` 与 `pendingModelDeletion?.names` 生成单模型/多模型 label、value 和 hint。
- 保留 dialog title、message、按钮可见文案、destructive/cancel roles、`performPendingModelDeletion()` 行为、`pendingModelDeletion = nil` 清理和 `modelDeleteBinding` 不变。

## 非目标

- 不修改 `performPendingModelDeletion()` 的数据行为：不得改变 `downloads.prepareForDeletion`、`fileStore.removeModelFile`、resume data 删除、`modelContext.delete`、失败回滚提示或 storage refresh。
- 不修改 `requestDeleteModels(_:)`、`deleteModels(at:)`、`ModelDeletionState` 的 ID/name 捕获规则、row controls、detail controls、untracked file delete flow、download pause/cancel/resume、Add Model、import 或 storage reconcile 行为。
- 不修改 SwiftData schema、file storage、`GenerationViewModel`、native backend、bridge、workflow、Xcode project、StoreKit、paid gates、Mac Catalyst 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Models 页面位于 `LocalDiffusion/Views/Models/ModelLibraryView.swift`，属于 UI 展示层。
- `ModelDeletionState` 已保存待删除 models 的 `modelIDs` 和 `names`，可直接用于 confirmation dialog button accessibility。
- `performPendingModelDeletion()` 负责文件和 SwiftData 删除；本轮只改确认按钮语义，不进入文件存储、下载状态机或数据层边界。

## 实现步骤

1. 阅读 `.confirmationDialog("Delete model?")`、`pendingModelDeletion`、`modelDeleteBinding`、`modelDeleteButtonTitle`、`modelDeleteMessage`、`performPendingModelDeletion()` 和 `ModelDeletionState`。

6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。

- 单模型删除确认 destructive button 暴露待删除 model 名称、当前 model value 和删除 metadata/local file 的 hint。

- Cancel button 暴露取消删除当前 pending model(s) 的 label、value 和保留 metadata/local files 的 hint。
