# v1.149 / Gallery 详情参数行窄宽度回退

## 版本号

- 版本：`v1.149`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.148 / Plan 概览窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- `GalleryDetailParameterRow` 在非 long-form 且普通 Dynamic Type 下固定横排 title/value；窄详情宽度下仍可能挤压。本轮补 `ViewThatFits`，并同步 v1.148 遗漏的 `flow.md` Plan overview 描述。

## 背景

`GalleryDetailParameterRow` 当前只在 accessibility 或 `isLongForm` 时纵向堆叠；普通短字段固定 `HStack`。iPad split / 窄 detail pane 下短字段也可能需要回退。本轮对非 stack 路径加 `ViewThatFits(in: .horizontal)`。

## 目标

1. 修改 `GalleryDetailParameterRow.body`：`shouldStack` 仍直接纵向。
2. 非 stack 路径使用 `ViewThatFits` 优先横排 title/value，不足时回退纵向。
3. 保留 accessibility label/value、字体颜色、long-form 行为。
4. 同步 `flow.md` 中 Plan overview 窄宽度回退描述（v1.148 遗漏）。

## 非目标

- 不修改 Gallery filter/sort/tile、delete、folder/tags 业务、native、StoreKit、Mac Catalyst、workflow。
- 禁止本地 Xcode / Simulator / 真机构建测试。

## 实现步骤

```swift
if shouldStack {
    VStack(...) { titleText; valueText }
} else {
    ViewThatFits(in: .horizontal) {
        HStack(alignment: .firstTextBaseline) {
            titleText
            Spacer(minLength: 12)
            valueText.multilineTextAlignment(.trailing)
        }

        VStack(alignment: .leading, spacing: 4) {
            titleText
            valueText
        }
    }
}
```

同步 README / flow / flowchart / update_log。
