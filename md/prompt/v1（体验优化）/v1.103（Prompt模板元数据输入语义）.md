# v1.103 Prompt 模板元数据输入语义

## 版本号

- 版本：`v1.103`
- 阶段：`v1（体验优化）`
- 任务名：Prompt 模板元数据输入语义

## 版本分配依据

- 最新已完成版本为 `v1.102`，已覆盖 Prompt 模板编辑器 Positive / Negative prompt 字符计数的字段化 VoiceOver 语义。
- 本轮继续在 Prompt Library 内做小步可验收 UI/可访问性优化，递增为 `v1.103`。

## 背景

`PromptTemplateEditor` 的 Save toolbar 已暴露 `Ready` / `Template name required`，但 Template section 中的 `Name` 和 `Category` TextField 仍只依赖可见 placeholder。VoiceOver 用户在进入字段时不能直接知道：

- Template name 是保存前必填项。
- Category 是可选分组字段，留空会进入 Uncategorized。
- Category rename editor 的 Name 是保存 category rename 前必填项。

这属于 UI 层语义补强，不需要改变 SwiftData、文件存储、生成、模板保存、分类重命名或搜索逻辑。

## 目标

1. 在 `PromptTemplateEditor` 中为模板 `Name` TextField 增加明确 accessibility label/value/hint。
2. 在 `PromptTemplateEditor` 中为模板 `Category` TextField 增加明确 accessibility label/value/hint。
3. 在 `PromptCategoryNameEditor` 中为 category rename `Name` TextField 增加明确 accessibility label/value/hint。
4. VoiceOver value 应区分空值和当前输入值；hint 应说明必填/可选和保存影响。
5. 保持当前可见 UI、暗色科幻风格、保存/取消 toolbar、trim、disabled gate 和 SwiftData 保存行为不变。

## 非目标

- 不修改 PromptTemplate SwiftData schema。
- 不修改模板新增、编辑、加载、删除、搜索、分类分组或排序行为。
- 不修改 `renameCategory(_:to:)`、`clearCategory(_:)`、`matchesSearch(_:)` 或 `generation.load(template:)`。
- 不修改 Generate、Gallery、Models、Plan、Root 或共享 ParameterEditor 行为。
- 不引入 StoreKit、真实付费 gate、entitlement persistence、Mac Catalyst/Xcode 平台设置、native backend、workflow 或新的测试规范。
- 不新增视觉布局重排、sheet、confirmation dialog 或导航入口。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，本轮只改 SwiftUI 展示语义。
- Prompt Library 负责模板保存、分类、搜索、编辑、加载；SwiftData 模型仍由 `AppModels.swift` 提供。
- Prompt 模板编辑器当前在 `LocalDiffusion/Views/Prompts/PromptLibraryView.swift` 内实现，适合单文件小步改动。
- 测试策略按 `md/test/test.md`：Swift 源码变更需跑本地轻量检查和两条 Swift parse，并通过 `main` push 触发 GitHub Actions 结果包。

## 实现步骤

1. 阅读必读文件：`AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`README.md`、`md/prompt/README.md`、`.github/workflows/ci-results.yml`。
2. 确认当前分支是 `main`，同步 `origin/main`，确保工作区没有无关改动。
3. 在 `PromptTemplateEditor` 的 Template section 中，为 `TextField("Name", text: $name)` 增加：
   - `accessibilityLabel`：说明这是 template name。
   - `accessibilityValue`：空值为 `Template name empty`，非空为当前 trimmed 或 raw 输入值。
   - `accessibilityHint`：说明保存模板前必须输入名称。
4. 为 `TextField("Category", text: $category)` 增加：
   - `accessibilityLabel`：说明这是 template category。
   - `accessibilityValue`：空值为 `Uncategorized`，非空为当前输入值。
   - `accessibilityHint`：说明这是可选分类，留空会保存为 Uncategorized。
5. 在 `PromptCategoryNameEditor` 的 `TextField("Name", text: $name)` 增加：
   - `accessibilityLabel`：说明这是 category name。
   - `accessibilityValue`：空值为 `Category name empty`，非空为当前输入值。
   - `accessibilityHint`：说明保存 rename 前必须输入名称。
6. 可按现有局部 helper 风格添加小型 computed properties，避免在 body 中堆叠重复 trim 逻辑。
7. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录本轮 Prompt template metadata field VoiceOver 语义。
8. 创建本提示词文件并只提交本轮相关文件。
