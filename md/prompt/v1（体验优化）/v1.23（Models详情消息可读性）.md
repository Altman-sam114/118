# v1.23 / Models 详情消息可读性

## 版本号

v1.23

## 版本分配依据

- 当前体验优化阶段已完成到 v1.22。
- v1.22 将 Models 列表行的 `progress.message` 提取为可读、可访问的 `ModelMessageRow`。
- `ModelDetailView` 的 Status section 仍把同一类 `progress.message` 放进普通 `DetailTextRow(title: "Message", value: message)`，只额外套用 amber 前景色，缺少明确图标、44pt 最小阅读高度和专用 VoiceOver 语义。

## 背景

Models 页面详情 sheet 用来查看单个模型的状态、来源、native loading 和存储信息。当前状态消息代码为：

```swift
if let message = progress.message, !message.isEmpty {
    DetailTextRow(title: "Message", value: message)
        .foregroundStyle(SciFiTheme.amber)
}
```

这类消息可能来自下载失败、文件处理或存储状态。列表行已经有统一的 message row，但详情页还没有同等可读性。本轮只优化详情 sheet 的 message 展示，不改变消息来源、下载状态、模型保存、辅助模型选择或任何业务行为。

## 目标

- 新增私有 `ModelDetailMessageRow`，或将现有 `ModelMessageRow` 调整为可被详情页安全复用。
- 用该组件替换 `ModelDetailView` Status section 中的 `DetailTextRow(title: "Message", value: message)`。
- 保留原有 message 文本、Status section 位置和 amber 警示语义。
- 消息行应：
  - 使用真实 `Label` 和 `exclamationmark.triangle` SF Symbol。
  - 使用 `SciFiTheme.amber`。
  - 文本可自然换行，不使用缩放压缩。
  - 具有 44pt 最小高度。
  - 提供明确 VoiceOver label/value/hint，例如 label 为 `Model detail status message`，value 为实际 message，hint 说明这是该模型详情中的下载或存储状态说明。
- 保持 `ModelDetailView` 的 State、ProgressView、Downloaded、Model、Native Loading、Source、Storage、detail controls 和 toolbar 行为不变。
- 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`，记录 Models 详情消息可读性改善。

## 非目标

- 不修改 `HuggingFaceDownloadManager`、`ModelDownloadProgress`、`LocalModel` 或 SwiftData schema。
- 不修改下载、暂停、恢复、取消、删除、导入、reconcile 或 fileImporter 行为。
- 不修改 Add Model 表单、未跟踪文件导入编辑器、alert 文案或 confirmation dialog。
- 不修改 native loading picker 绑定、辅助模型文件名保存逻辑或 `modelContext.save()` 调用。
- 不新增 toast、banner、全局错误中心、日志系统或新状态字段。
- 不修改 native backend、生成流程、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。

## 当前架构依据

- `ModelLibraryView` 属于 UI 展示层，只负责展示模型状态、用户输入和操作反馈。
- `progress.message` 的来源仍由下载/存储状态层维护，UI 不应重写、清空或持久化该消息。
- 详情 sheet 中的 native loading 和辅助模型选择仍只通过现有 SwiftData 模型字段绑定保存。
- SwiftUI 可访问性要求：重要状态或错误信息应可读、可换行，并提供明确 label/value/hint；展示行应满足 44pt 阅读/触控基线。
