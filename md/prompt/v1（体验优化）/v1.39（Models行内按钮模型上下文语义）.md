# v1.39 / Models 行内按钮模型上下文语义

## 版本号

v1.39

## 版本分配依据

- 当前体验优化阶段已完成到 v1.38。
- v1.10 已让 Models row controls 具备可访问文本 label、44pt 命中区和 Dynamic Type 堆叠布局。
- v1.22、v1.23、v1.24、v1.25、v1.26、v1.27 已持续改善 Models 的行内状态、详情消息、错误、Storage Matrix、Add Model 提交和未跟踪文件上下文语义。
- Models 列表中每个 `ModelRow` 都可能出现重复的 icon-only 操作按钮，例如 `Model Details`、`Delete Model`、`Pause Download`、`Cancel Download`、`Resume Download`、`Download Model`。这些按钮当前通过 `Label(title, systemImage:)` 提供可访问文本，但没有把具体 `model.name` 编入 VoiceOver label，多个模型并列时命令上下文不够明确。
- 本轮选择一个小步 iPad/Mac readiness UI 目标：为 Models 行内操作按钮补充模型名上下文辅助功能语义，不改变下载、暂停、取消、删除、详情或文件状态行为。

## 背景

Local Diffusion 的 Models 页面是 GGUF 管理入口。用户可能同时拥有多个 queued、ready、failed、paused 或 downloading 模型，每个 row 都显示一组相似 icon-only controls。共享 `SciFiSecondaryButtonStyle` 已提供视觉样式、hit shape 和 pointer hover affordance，按钮本身也有 44pt 最小尺寸。

当前 `ModelRow.controlButton(...)` 只把通用动作名暴露给辅助技术，例如 `Delete Model` 或 `Download Model`。在多模型列表中，这些重复 label 对 VoiceOver 和 Voice Control 不够具体。本轮只给现有按钮追加模型名上下文和简短 hint，保持视觉、布局、状态机和 action 不变。

## 目标

- 在 `LocalDiffusion/Views/Models/ModelLibraryView.swift` 的 `ModelRow.controlButton(...)` helper 上补充模型上下文辅助功能语义：
  - label 示例：`"\(title): \(model.name)"`。
  - hint 示例：`"Applies this action to \(model.name)."`。
- 覆盖该 helper 生成的现有行内按钮：
  - `Model Details`
  - `Delete Model`
  - `Pause Download`
  - `Cancel Download`
  - `Resume Download`
  - `Download Model`
- 保持现有 `Button(role:action:)`、`Label(title, systemImage:)`、`.labelStyle(.iconOnly)`、`SciFiSecondaryButtonStyle`、`.frame(minWidth: 44, minHeight: 44)`、按钮顺序、状态分支和所有 closures 不变。
- 同步 README、flow、flowchart、update_log，说明 Models row inline controls 获得模型名上下文 VoiceOver semantics baseline。

## 非目标

- 不修改下载、暂停、恢复、取消、删除、详情、导入、未跟踪文件、Add Model、Storage Matrix 或 message row 业务行为。
- 不修改 `ModelDownloadProgress`、`HuggingFaceDownloadManager`、`AppFileStore`、SwiftData schema 或文件存储。
- 不修改 `SciFiSecondaryButtonStyle`、共享 button hover、整张 `ModelRow` 的点击行为、row hover 或卡片视觉。
- 不修改 native backend、Release asset、CI workflow、Generate、Gallery、Prompts 或 Plan 业务行为。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、Xcode project、签名、分发或 scheme 设置。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 visible keyboard shortcut 文案、Mac-only UI、自定义 hover 动画或额外手势。

## 当前架构依据

- Models 属于 `LocalDiffusion/Views/Models/ModelLibraryView.swift` 的 UI 展示层，只负责展示、输入、导航和用户反馈。
- `ModelRow` 是单个模型 row 的局部 view；下载状态和动作仍由外层 `ModelLibraryView`、`HuggingFaceDownloadManager` 和 `AppFileStore` 处理。
- `controlButton(...)` 已集中生成行内 icon-only controls，是补充模型名辅助语义的最小作用点。
- 共享 `SciFiSecondaryButtonStyle` 已提供 pointer hover 和 hit shape，本轮不修改共享样式，避免扩大到无关按钮。
