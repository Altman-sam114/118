# v1.110 / Plan Mac QA 边界语义

## 版本号

- 版本：`v1.110`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 上一轮 `v1.109 / Plan 概览购买边界语义` 已由 Agent C 验收通过，commit `231d34e17d4f46ca5d72940b98002e7c25b56bec`，run `28811659706`，attempt `1`，artifact `localdiffusion-ci-v1.109-main-231d34e-run28811659706-attempt1`。
- 人工总目标是持续优化 mac 版、iPad 版、UI、设计和付费功能。本轮继续选择一个小而可验收的 UI / 文案 / 辅助功能语义改进。
- 本轮必须避开 v1.109 的 Plan overview purchase boundary，不启用真实 StoreKit，不启用 Mac Catalyst，不需要付费账号或密钥，不修改 workflow、native backend、SwiftData 或推理链路。
- 当前 `PlanView` 已有 Mac Readiness checklist，其中 `Window and sidebar QA` 行说明未来 Mac UI 需要 QA。本轮递增为 `v1.110`，聚焦这一个 Mac/iPad 边界语义点。

## 背景

Plan 页面当前同时展示 Local plan、paid candidates planning-only、StoreKit 未配置、Mac/Catalyst 未启用和 Mac readiness blockers。`Window and sidebar QA` 行当前 detail 为：

```text
Mac window sizing, sidebar behavior, keyboard, and pointer states need QA validation.
```

该文案已经说明 Mac UI 仍需 QA，但项目已有 iPad regular 双栏、sidebar、pointer hover affordance。用户可能会误解为当前 iPad 布局 QA 可以代表未来 Mac/Catalyst app QA。Plan 页面需要更明确地区分：当前 iPhone/iPad app 可用，iPad layout/pointer support 是现有平台体验；未来 Mac/Catalyst app 仍需要专门的 window/sidebar/keyboard/pointer 验证。

本轮只补强 `Window and sidebar QA` 这一行的可见 detail 和 VoiceOver hint，不改变 Mac readiness 状态、布局、数据结构、平台配置或任何业务逻辑。

## 目标

1. 在 `PlanView.macReadinessItems` 中只调整 `Window and sidebar QA` 行的 `detail`，明确未来 Mac window/sidebar/keyboard/pointer QA 需要 dedicated Mac/Catalyst validation，并说明当前 iPad layout support 不能替代 Mac validation。
2. 同步调整同一行的 `accessibilityHint`，让 VoiceOver 用户听到 iPad regular layout / pointer affordance 并不等同于 Mac/Catalyst app validation。
3. 保持该行 title、status badge `Needs QA`、system image、排序、row layout、Mac Readiness footer、Platform Status 和 Availability facts 不变。
4. 保持 Plan 页面暗色科幻视觉、compact Form、iPad regular 双栏布局、Dynamic Type 回退和所有导航行为不变。
5. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前事实，记录 Mac QA 边界语义已补强。

## 非目标

- 不新增 native Mac/Catalyst backend slice，不修改 `NativeStableDiffusionBridge.h`、`StableDiffusionCppBridge.mm`、XCFramework、native backend asset metadata 或任何 native/backend 脚本。
- 不启用 StoreKit，不新增 product IDs、purchase UI、restore flow、receipt validation、subscriptions、entitlement persistence、paid gate、App Store Connect 请求或真实购买流程。
- 不修改 SwiftData schema、`AppFileStore`、`HuggingFaceDownloadManager`、`GenerationViewModel`、`ImageGenerationBackend`、模型下载、图片生成、Gallery、Prompt Library 或 Models 行为。
- 不修改 workflow、CI artifact 生成逻辑、测试规范、Agent 工作流或默认分支策略。

## 当前架构依据

- `PlanView` 位于 `LocalDiffusion/Views/RootContentView.swift`，属于 SwiftUI UI 展示层，只负责展示当前 Local plan、付费规划、平台状态和 Mac readiness 信息。
- `macReadinessItems` 是静态 UI 数据；`MacReadinessRow` 将 `MacReadinessItem.detail` 传入 `PlanStatusRow`，因此 detail 同时影响可见说明和 row-level accessibility label。
- `MacReadinessItem.accessibilityHint` 由 `MacReadinessRow` 追加到 `PlanStatusRow`，适合补充 iPad layout support 与未来 Mac/Catalyst validation 的区别。
- 本轮只修改 UI 文案和辅助功能语义，不触碰状态层、数据层、文件存储、native backend、StoreKit 或平台构建配置。
