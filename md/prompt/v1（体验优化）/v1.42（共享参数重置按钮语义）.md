# v1.42 / 共享参数重置按钮语义

## 版本号

v1.42

## 版本分配依据

- 当前体验优化阶段已完成到 v1.41。
- v1.13 已改善共享 `ParameterEditor` 的参数控件可读性、Dynamic Type 布局和基础辅助功能值。
- `ParameterEditor` 顶部的 `Reset Defaults` 按钮当前只有可见文本和图标，没有说明它会重置 steps、CFG、seed、canvas size 和 sampler，同时保留 positive / negative prompt。
- `ParameterEditor` 同时用于 Generate 和 Prompt Template Editor，因此本轮选择一个共享 UI 语义小步：补充 Reset Defaults 的作用范围和 prompt 保留语义，不修改现有重置行为。

## 背景

Local Diffusion 的共享参数编辑器负责生成参数与模板参数的统一编辑入口。Reset Defaults 按钮的实际实现会先保存当前 positive prompt 与 negative prompt，再用默认 `GenerationParameters` 覆盖参数值。视觉用户可以从上下文推断按钮用途，但 VoiceOver 用户聚焦按钮时只能听到通用的 Reset Defaults，无法确认 prompt 是否会被清空。

本轮只给该按钮增加明确的辅助功能 label/value/hint。按钮 action、参数默认值、参数范围、seed 随机逻辑、sampler 绑定和所有业务数据流保持不变。

## 目标

- 在 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的 Reset Defaults 按钮上增加清晰辅助功能语义：
  - label 表示这是参数默认值重置。
  - value 说明会重置 steps、CFG、seed、canvas size 和 sampler。
  - hint 说明 positive prompt 和 negative prompt 会保留。
- 语义文案必须通用，不能写死 Generate screen，因为该组件也用于 Prompt Template Editor。
- 保持按钮现有视觉、图标、`SciFiSecondaryButtonStyle()` 和 action 行为不变。
- 同步 README、flow、flowchart、update_log，记录共享参数重置按钮的 VoiceOver 语义基线。

## 非目标

- 不修改 `parameters = GenerationParameters(prompt: currentPrompt, negativePrompt: currentNegativePrompt)` 的行为。
- 不修改 `GenerationParameters` 默认值、范围、归一化、sampler raw value、seed 随机范围或 generation request 组装。
- 不修改 Generate、Prompt Library 的保存/加载模板行为。
- 不修改 SwiftData schema、文件存储、native backend、桥接头、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不修改平台、签名、分发或 scheme 设置。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线或真机 GGUF 生成验收。

## 当前架构依据

- `ParameterEditor` 属于 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的共享 UI 展示层，只负责展示和用户输入。
- Generate 和 Prompt Template Editor 通过 `@Binding var parameters: GenerationParameters` 复用同一个参数编辑器。
- Reset Defaults 的 action 当前只在 UI 层重建 `GenerationParameters`，并保留 prompt 字段，不触碰 SwiftData、文件存储或 native backend。
- 辅助功能语义属于 UI 表达，不应进入 ViewModel、数据层或 native 边界。
