# v1.128 / Gallery 预览下一步语义

## 版本号

- 版本：`v1.128`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.127 / Result Gallery 命中区`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.127 已补齐 Generate result 到 Gallery 的 handoff 命中区；本轮继续优化 Gallery detail 中生成结果预览的 VoiceOver 下一步提示。

## 背景

`GalleryDetailView` 的详情大图 `Image(uiImage:)` 已有 `Generated image preview` accessibility label 和 `imagePreviewAccessibilityValue`，能说明预览图片的尺寸和保存状态。但该预览没有 accessibility hint。用户进入详情页后，下一步通常是使用详情控件复用参数、重新生成、分享、整理 folder/tags 或删除结果。本轮只给详情大图预览补一个简短 hint，让 VoiceOver 用户听到可继续使用详情控件处理该结果。

## 目标

1. 只修改 `GalleryDetailView` 中详情大图预览的 accessibility 语义。
2. 在 `.accessibilityValue(imagePreviewAccessibilityValue)` 后增加：

```swift
.accessibilityHint("Use the detail controls to reuse, regenerate, share, organize, or delete this result.")
```

3. 保持 image preview 的显示、尺寸、clip、overlay、accessibility label/value 不变。
4. 保持 Reuse、Regenerate、Share、Folder picker、Tags、Delete、Gallery navigation、SwiftData 和文件存储逻辑不变。

## 非目标

- 不修改 Gallery grid tile、Generate result preview、Generate Gallery handoff、Gallery detail actions、folder/tags editor 或 delete confirmation。
- 不修改 `imagePreviewAccessibilityValue`。
- 不修改图片加载、文件 URL、SwiftData 模型、file storage、GenerationViewModel、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不引入固定宽度、固定高度、截图、动画、视觉重排或新组件。
- 不做 simulator / Stage Manager / Split View / VoiceOver 实机朗读检查；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GalleryView.swift` 位于 `LocalDiffusion/Views/Gallery/`，属于 SwiftUI UI 展示层。
- `GalleryDetailView` 从 `AppFileStore` 读取已保存 PNG 并展示详情。
- 详情操作已经通过 Button / Picker / TextField 提供各自语义；本轮只补预览图的下一步 hint，不改变任何 action 或 data flow。

## 实现步骤

1. 阅读 `GalleryDetailView.body` 中详情图片预览。
2. 在 `.accessibilityValue(imagePreviewAccessibilityValue)` 后追加 accessibility hint：

```swift
.accessibilityHint("Use the detail controls to reuse, regenerate, share, organize, or delete this result.")
```

3. 不修改 image preview 视觉 modifiers、操作按钮或组织控件。
4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Gallery detail preview 会提示可使用详情控件处理该结果。
5. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
