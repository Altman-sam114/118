# v1.28 / Prompt 空状态语义

## 版本号

v1.28

## 版本分配依据

- 当前体验优化阶段已完成到 v1.27。
- v1.11、v1.14 已优化 Prompt Library 的模板行控件和模板编辑器可读性。
- Prompt Library 的空模板状态和搜索无结果状态仍只依赖共享 `EmptyStateView` 的可视标题与说明，缺少页面特定的组合 VoiceOver 状态、当前搜索词和下一步操作提示。

## 背景

Prompt Library 是用户复用生成参数和 prompt 模板的入口。当前页面有两个空状态：

- 模板库没有任何模板时，显示 `No templates`。
- 搜索后无匹配模板时，显示 `No matching templates`。

本轮只改善这两个空状态的辅助功能语义，让 VoiceOver 用户明确知道当前状态和下一步可做什么，不改变模板保存、搜索、分类、编辑、删除或加载行为。

## 目标

- 为 Prompt Library 无模板空状态增加组合 accessibility label/value/hint：
  - label 表达这是 Prompt Library 空状态。
  - value 说明当前没有保存的 prompt templates。
  - hint 说明可用 Add 创建模板，或从 Generate 保存当前参数。
- 为搜索无结果空状态增加组合 accessibility label/value/hint：
  - label 表达搜索没有匹配模板。
  - value 包含当前搜索词。
  - hint 说明可调整搜索词或新增模板。
- 保持 `EmptyStateView` 视觉、暗色 Sci-Fi 面板、列表 row 背景、搜索行为和 toolbar Add 行为不变。
- 将空状态语义集中在 `PromptLibraryView` 的 Prompt 页面调用点，不修改共享 `EmptyStateView`，避免影响 Generate、Models 或 Gallery 的现有表现。

## 非目标

- 不修改 `PromptTemplate`、`GenerationParameters` 或 SwiftData schema。
- 不修改模板新增、编辑、删除、分类 rename/clear、搜索过滤、加载到 Generate 或保存流程。
- 不修改 `EmptyStateView` 共享组件、`ParameterEditor`、Root 导航、StoreKit、Mac Catalyst、Xcode project、native backend 或 CI workflow。
- 不新增 simulator screenshot、真机 VoiceOver 目检或真实 GGUF 生成验收。

## 当前架构依据

- `PromptLibraryView` 属于 UI 展示层，负责展示 SwiftData 查询出的 `PromptTemplate`，并将加载操作委托给 `GenerationViewModel.load(template:)`。
- 空状态只由 SwiftData 查询结果和 `searchText` 派生，不应写入数据层或状态层。
- SwiftUI 可访问性要求：图像和多段文本组成的状态卡片应合并成一个有意义的辅助功能元素，避免 VoiceOver 分散朗读图标、标题和说明而缺少上下文。
