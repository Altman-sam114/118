# v1.55 / Plan 能力矩阵提示语义

## 版本号

v1.55

## 版本分配依据

- v1.53 已为 Plan Availability rows 增加行级 VoiceOver hint。
- v1.54 已为 Plan Mac readiness rows 增加行级 VoiceOver hint。
- 本轮继续补齐 Plan 页付费能力规划区域，将 Capability Matrix rows 的状态含义通过行级 hint 明确给 VoiceOver 用户。

## 背景

Plan 页当前通过 Capability Matrix 区分当前 Local plan 已可用能力、planning-only paid candidates 和 StoreKit configuration gate。`PlanStatusRow` 已提供 label/value，Availability 与 Mac readiness 已在各自 wrapper 上追加 item-specific hint。Capability Matrix 仍只朗读 title/detail/status，缺少对“当前可用”“规划中但未售卖/未解锁”“需要 StoreKit 配置”的行级解释。

## 目标

- 为 Capability Matrix 的所有 row 增加行级 VoiceOver hint。
- 对三类能力分别说明：
  - Local generation workspace、Model and storage management、Gallery and prompt reuse 属于当前 Local plan 可用能力。
  - Batch queue controls、Curated prompt packs、Workflow export 是 planning-only paid candidates，当前未售卖、未解锁。
  - StoreKit purchases 需要 product IDs、entitlement rules 和 App Store Connect 后才能实现。
- 保持所有可视文案、状态、图标、顺序、布局和业务行为不变。

## 非目标

- 不接入 StoreKit。
- 不新增 product IDs、purchase/restore/receipt/subscription、entitlement persistence 或 paid gates。
- 不启用 Mac Catalyst。
- 不修改 Xcode project、native XCFramework、native bridge、SwiftData schema、文件存储、推理流程或 CI workflow。
- 不修改 shared `PlanStatusRow` 的通用行为，避免影响 Entitlement Rules、Availability 和 Mac readiness 之外的 row。
- 不新增 simulator VoiceOver 自动化、StoreKit 测试或真实购买测试。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，Plan 页面在 `LocalDiffusion/Views/RootContentView.swift`。
- `PlanCapabilityItem` 是 Capability Matrix 的数据源，`CapabilityRow` 是行级 wrapper。
- 现有 `AvailabilityRow` 和 `MacReadinessRow` 已使用 wrapper 追加 `.accessibilityHint(item.accessibilityHint)`，本轮应复用该局部模式。

## 实现步骤

1. 在 `PlanCapabilityItem` 增加 `accessibilityHint: String` 字段。
2. 为 `capabilityItems` 七个元素填入对应 hint。
3. 在 `CapabilityRow` 的 `PlanStatusRow(...)` 后追加 `.accessibilityHint(item.accessibilityHint)`。
4. 不改 title/detail/status/systemImage、row 顺序、layout、StoreKit facts 或 paid candidate facts。
5. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
