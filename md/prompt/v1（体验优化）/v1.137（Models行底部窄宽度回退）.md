# v1.137 / Models 行底部窄宽度回退

## 版本号

- 版本：`v1.137`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.136 / Models 行标题窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.136 已补齐 `ModelRow.header` 窄宽度回退；同一行的 `footer`（size label + controls）在普通 Dynamic Type 下仍固定横排，是下一个对称、可验收的 UI 韧性优化。

## 背景

`ModelRow.footer` 在 accessibility Dynamic Type 下已经使用纵向 size label + controls 布局，但普通 Dynamic Type 分支固定使用 `HStack { sizeLabel; Spacer(); controls }`。下载中控件较多（Details/Pause/Cancel 等），在 iPad split、Stage Manager 或窄 list 宽度下可能挤压。项目内已在 header、Prompt metric pills、Gallery metadata 使用 `ViewThatFits(in: .horizontal)`；本轮把同一模式补齐到 Models 行 footer，不改变下载/删除/详情业务。

## 目标

1. 只修改 `ModelRow.footer` 的普通 Dynamic Type 分支。
2. 使用 `ViewThatFits(in: .horizontal)` 优先显示既有横向 size + controls 布局。
3. 横向空间不足时回退到既有纵向 size + controls 布局。
4. 保持 accessibility Dynamic Type 现有纵向布局不变。
5. 保持 size 文案、controls 按钮、status 行为、header、progress、message row 不变。

## 非目标

- 不修改 `header`、`controls` 内部逻辑、`UntrackedModelFileRow`、Add Model、detail sheet、download manager。
- 不修改 SwiftData、AppFileStore、GenerationViewModel、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 禁止本地 Xcode / Simulator / 真机构建测试；完整构建交给 GitHub Actions。

## 当前架构依据

- `ModelLibraryView.swift` 属于 SwiftUI UI 展示层。
- `ModelRow.footer` 只布局 size 文本与行内控制按钮。

## 实现步骤

1. 阅读 `ModelRow.footer`、`sizeLabel`、`controls`。
2. 保留 accessibility Dynamic Type 纵向分支。
3. 将普通 Dynamic Type 分支改为：

```swift
ViewThatFits(in: .horizontal) {
    HStack {
        sizeLabel
        Spacer()
        controls
    }

    VStack(alignment: .leading, spacing: 8) {
        sizeLabel
        controls
    }
    .frame(maxWidth: .infinity, alignment: .leading)
}
```

4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`。
5. 检查 diff 范围。
