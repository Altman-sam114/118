# v1.123 / Stepper 标签窄宽度回退

## 版本号

- 版本：`v1.123`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.122 / CFG 标题窄宽度回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.121-v1.122 已收敛共享参数区 Seed、Size preset 和 CFG header 的窄宽度回退；本轮继续处理同一共享参数区内 Steps、Width、Height Stepper 的 label 压缩风险。

## 背景

`ParameterEditor` 同时服务 Generate 和 Prompt Library。当前 Steps、Width、Height 使用 `Stepper("Title: value", ...)` 的单行字符串标题。该写法简单稳定，但在 iPad Split View、Stage Manager、sheet 或未来本地化文案变长时，标题和值只能作为一个单行 label 被系统布局压缩。项目已在共享参数控件里使用 `ViewThatFits(in: .horizontal)` 做窄宽度回退，本轮将同一模式补齐到 Stepper label。

## 目标

1. 将 `stepsControl`、`widthControl`、`heightControl` 的 Stepper label 改为自定义 label。
2. 新增共享 `stepperLabel(title:value:)`，普通 Dynamic Type 优先横排 title/value，横向空间不足时回退纵向。
3. accessibility Dynamic Type 下直接使用纵向 title/value。
4. 保持 Stepper 的 binding、范围、步进、辅助功能 label/value/hint 和可见数值含义不变。
5. 让 Generate 与 Prompt Library 复用的参数编辑器同时受益。

## 非目标

- 不修改 `GenerationParameters`、steps/width/height 范围、默认值、归一化、native request mapping 或生成门禁。
- 不修改 Seed、Randomize Seed、Size preset、CFG slider、sampler、prompt、model picker、result、Gallery handoff、SwiftData、file storage、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `ParameterEditor` 位于 `LocalDiffusion/Views/Shared/ParameterEditor.swift`，属于共享 SwiftUI UI 展示层。
- `GenerationView` 和 `PromptLibraryView` 都复用 `ParameterEditor(parameters:)`。
- Steps 直接绑定 `parameters.steps`；Width/Height 通过 `widthBinding`、`heightBinding` 调用 `normalizedDimension(_:)`。
- 本轮只修改 SwiftUI label 展示，不触碰状态层、数据层、文件存储或 native backend。

## 实现步骤

1. 阅读 `stepsControl`、`widthControl`、`heightControl`、`widthBinding`、`heightBinding`。
2. 把三个 `Stepper("Title: value", ...)` 改为 `Stepper(value:in:step:) { stepperLabel(title:value:) }` 形态；Steps 不新增 step 参数。
3. 新增 `@ViewBuilder private func stepperLabel(title:value:) -> some View`。
4. `stepperLabel` 在 accessibility Dynamic Type 下使用纵向 `VStack`。
5. 普通 Dynamic Type 下使用 `ViewThatFits(in: .horizontal)`，先尝试 `HStack { Text(title); Spacer(); Text(value) }`，再回退纵向。
6. 不修改现有 `.accessibilityLabel`、`.accessibilityValue`、`.accessibilityHint`。
7. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`。
