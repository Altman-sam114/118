# v1.27 / 未跟踪模型文件行语义

## 版本号

v1.27

## 版本分配依据

- 当前体验优化阶段已完成到 v1.26。
- v1.25 优化了 Models 存储摘要 VoiceOver 语义，v1.26 优化了 Add Model 表单键盘提交和确认按钮语义。
- Models 页面 `UntrackedModelFileRow` 已有 Dynamic Type 纵向布局、icon-only 按钮隐藏文本 label 和 44pt 命中区，但 VoiceOver 仍缺少“这是哪个文件、文件多大、按钮操作哪个文件”的组合上下文。

## 背景

未跟踪模型文件来自 `fileStore.modelFilenames()` 扫描结果，排除 SwiftData 已跟踪的 `LocalModel.localFilename` 后显示在 Models 页 `Untracked Files` section。用户可以：

- 导入未跟踪 GGUF 文件到模型库。
- 删除未跟踪文件，并经过确认弹窗。

本轮只改善 `UntrackedModelFileRow` 的辅助功能语义，不改变文件扫描、排序、导入 sheet、删除确认、文件删除或 SwiftData 行为。

## 目标

- 为 `UntrackedModelFileRow.fileInfo` 增加组合 accessibility label/value/hint：
  - label 表达这是未跟踪模型文件。
  - value 包含文件名和格式化后的文件大小。
  - hint 说明可导入到模型库或删除文件。
- 为 Import 按钮增加包含具体文件名的 accessibility label/hint。
- 为 Delete 按钮增加包含具体文件名的 accessibility label/hint，并保留 destructive role。
- 复用 `ByteCountFormatter.fileSizeString(file.bytes)`，避免重复格式散落。
- 保留现有视觉布局、按钮样式、图标、隐藏 label、44pt 命中区、Dynamic Type 分支和 `.sciFiPanel()`。

## 非目标

- 不修改 `refreshUntrackedModelFiles()`、`importUntrackedModel()`、`performPendingUntrackedModelDeletion()` 或 confirmation dialog。
- 不修改未跟踪文件扫描、排序、大小计算、导入 sheet、默认名称推导或删除流程。
- 不修改 `AppFileStore`、`LocalModel`、SwiftData schema、`HuggingFaceDownloadManager`、native backend、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。
- 不修改 Add Model、Model row、Model detail、Storage Matrix 或其他页面。
- 不新增 simulator screenshot、真机 VoiceOver 目检或真实 GGUF 生成验收。

## 当前架构依据

- `UntrackedModelFileRow` 属于 UI 展示层，只接收 `UntrackedModelFile` 和两个 action closure。
- 文件发现、导入和删除由 `ModelLibraryView` 与 `AppFileStore` 处理，本轮不得改变这些数据流。
- SwiftUI 可访问性要求：图标按钮可保持 `.labelStyle(.iconOnly)`，但必须保留真实 `Label` 文本并为重复列表项提供具体上下文。
