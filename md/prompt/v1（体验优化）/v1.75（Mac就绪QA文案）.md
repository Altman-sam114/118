# v1.75 / Mac 就绪 QA 文案

## 版本号

v1.75

## 版本分配依据

- v1.74 已完成 Plan Availability footer 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续优化 Plan 的 Mac readiness copy，把面向开发测试的术语改成用户更容易理解的 QA validation 表述。

## 背景

Plan 的 Mac Readiness checklist 当前用于说明未来 Mac 支持仍需平台、native backend、UI QA 和分发签名等前置工作。其中 `Window and sidebar QA` 的 detail 使用 “smoke coverage”，这是测试流程术语，不如 “QA validation” 适合出现在用户可见 Plan 页面里。该项状态 badge 已经是 `Needs QA`，detail 应该与该状态一致。

## 目标

- 将 `Window and sidebar QA` 的 detail 从 “need smoke coverage” 改为 “need QA validation”。
- 保持该 row 的 title、status、icon、accessibility hint 和 Mac readiness 数据结构不变。
- 保持 Mac app not enabled、Mac Catalyst 未启用、native backend slice 未配置等事实不变。

## 非目标

- 不启用 Mac Catalyst，不修改 Xcode target/platform、signing、native backend slice、XCFramework、workflow、SwiftData、文件存储或推理链路。
- 不修改 StoreKit product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不修改 Mac readiness item 数组结构、status enum、row layout、Plan 导航或除该 detail 以外的可见文案。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- `MacReadinessItem.detail` 会进入 `PlanStatusRow` 的可见文本和 accessibility label。
- `Window and sidebar QA` 当前使用 `.planned` status，badge title 为 `Needs QA`。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `macReadinessItems`、`MacReadinessStatus`、`MacReadinessRow` 和 `PlanStatusRow`。
2. 只将 `Window and sidebar QA` 的 detail 文案从 `need smoke coverage` 改为 `need QA validation`。
3. 不改 row status、hint、icon、layout、Mac support facts 或任何业务逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
