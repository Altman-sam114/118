# v1.89 / Models 原生加载选择器语义

## 版本号

v1.89

## 版本分配依据

- v1.88 已完成 Models detail sheet 操作按钮的模型名 VoiceOver 上下文，并通过 `origin/main` 云端结果包验收。
- 本轮继续非 Plan 的 Models 体验优化，聚焦详情页 Native Loading 区域的 Picker 当前值和用途说明，让用户在 iPad 或键盘/辅助功能环境中能更清楚理解原生加载模式与辅助 GGUF 文件选择。

## 背景

`ModelDetailView` 的 `Native Loading` section 现在包含：

- `Picker("Mode", selection: nativeLoadModeBinding)`：选择 `Full Model` 或 `Standalone Diffusion`。
- `auxiliaryModelPicker(...)`：选择 CLIP-L、CLIP-G、T5XXL、VAE 等可选辅助 GGUF 文件。
- `DetailTextRow(title: "Mode", value: nativeLoadModeDescription)`：解释当前加载模式。

这些控件的可见标题存在，但 VoiceOver 读到的上下文偏通用，尤其 `Mode` 不能直接说明这是 native loading mode，辅助模型 Picker 也没有说明它们是用于 standalone diffusion native loading 的可选文件。本轮只补齐辅助功能 label/value/hint，不改变任何绑定、保存、可选文件列表或 native 后端行为。

## 目标

- 给 `ModelDetailView` 的 Native Loading mode Picker 增加明确的辅助功能语义：
  - label 指向 `Native loading mode`。
  - value 暴露 `model.nativeLoadMode.rawValue`。
  - hint 说明该控件用于选择完整模型加载或 standalone diffusion 加载。
- 给 `auxiliaryModelPicker(...)` 统一增加辅助功能语义：
  - label 使用 `<title> auxiliary model`。
  - value 暴露当前选择的文件名，未选择时为 `None selected`。
  - hint 说明该控件用于为当前模型选择可选 `<title>` GGUF 文件。
- 保留 Picker 可见标题、选项、tag、binding、`try? modelContext.save()`、availableModelFilenames 来源和表单布局不变。

## 非目标

- 不修改 `LocalModel`、SwiftData schema、`GenerationViewModel`、`ImageGenerationBackend`、native bridge、XCFramework 或 stable-diffusion.cpp 参数传递。
- 不修改辅助模型文件的过滤规则、可选文件来源、保存逻辑、默认值或校验逻辑。
- 不修改下载、导入、删除、暂停、恢复、取消、详情操作按钮、状态消息或 storage rows。
- 不修改 StoreKit、paid gates、Mac Catalyst、Xcode project、workflow、Plan 页面或真实购买能力。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Models 页面位于 `LocalDiffusion/Views/Models/ModelLibraryView.swift`，属于 UI 展示层。
- `ModelDetailView` 直接编辑 `LocalModel` 的 native load mode 与辅助文件名字段，并通过 SwiftData `modelContext.save()` 保存。
- `GenerationViewModel` 后续只读取 `LocalModel` 中已保存的 native mode 与辅助文件名；本轮不进入生成状态层或推理后端层。
- UI 只能通过既有模型字段和绑定表达选择语义，不得直接调用 C bridge 或变更 native 后端接口。

## 实现步骤

1. 阅读 `ModelDetailView` 中 `Native Loading` section、`nativeLoadModeBinding`、`auxiliaryModelPicker(...)` 和 `auxiliaryFilenameBinding(...)`。
2. 在 Mode Picker 上追加 `.accessibilityLabel(...)`、`.accessibilityValue(...)`、`.accessibilityHint(...)`，仅描述当前控件用途和当前值。
3. 在 `auxiliaryModelPicker(...)` helper 返回的 Picker 上追加统一 accessibility label/value/hint，使用 `selection.wrappedValue` 生成当前值。
4. 核对所有 Picker 的 visible title、选项、tag、binding setter 和保存逻辑不变。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
