# v1.48 / 共享宽高控件语义

## 版本号

v1.48

## 版本分配依据

- 当前体验优化阶段已完成到 v1.47。
- v1.42 至 v1.47 已连续补充共享 `ParameterEditor` 中 Reset Defaults、Randomize Seed、Canvas Size preset、Sampler picker、Steps stepper 和 CFG slider 的 VoiceOver 语义。
- `widthControl` 和 `heightControl` 当前有 `Width` / `Height` label 和当前值，但没有说明这两个 stepper 分别调整 canvas width / height pixels。
- `ParameterEditor` 同时用于 Generate 和 Prompt Template Editor，因此本轮继续选择共享参数编辑器的小步 VoiceOver 语义优化，不修改宽高范围、步进、归一化或生成行为。

## 背景

Local Diffusion 的 canvas width 和 height 既可以通过 size preset menu 设置，也可以通过独立 stepper 手动调整。v1.44 已说明 preset 会同时更新 width 和 height；当前独立 Width / Height stepper 仍只朗读数值，没有说明它们分别调整画布宽度和高度的像素尺寸。

本轮只给 Width 和 Height stepper 增加通用操作 hint。Stepper 标题、绑定、范围、step、尺寸归一化、size preset 行为、生成请求映射和模板保存/加载行为保持不变。

## 目标

- 在 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的 `widthControl` 上补充：
  - `.accessibilityHint("Adjust the canvas width in pixels for these generation parameters.")`
- 在 `heightControl` 上补充：
  - `.accessibilityHint("Adjust the canvas height in pixels for these generation parameters.")`
- 保持现有 `Stepper` 标题、binding、范围、step、foreground style、accessibility label 和 accessibility value 不变。
- 语义文案必须通用，不能写死 Generate screen，因为该组件也用于 Prompt Template Editor。
- 同步 README、flow、flowchart、update_log，记录共享 Width / Height stepper 获得 canvas pixel dimension 说明的 VoiceOver hint。

## 非目标

- 不修改 `GenerationParameters.minimumDimension`、`maximumDimension`、`dimensionStep`、默认尺寸或 normalization。
- 不修改 `widthBinding`、`heightBinding`、`normalizedDimension(_:)`、size preset 列表或 preset action。
- 不修改 CFG、Steps、Seed、Randomize Seed、Canvas Size preset、Sampler 或 Reset Defaults。
- 不修改 Generate、Prompt Library 的保存/加载模板行为。
- 不修改 native request mapping、stable-diffusion.cpp bridge、SwiftData schema、文件存储、ViewModel、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不新增 StoreKit import、product ID、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线或真机 GGUF 生成验收。

## 当前架构依据

- `ParameterEditor` 属于 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的共享 UI 展示层，只负责展示和用户输入。
- Generate 和 Prompt Template Editor 通过 `@Binding var parameters: GenerationParameters` 复用同一个参数编辑器。
- `widthControl` 和 `heightControl` 当前只在 UI 层通过 `widthBinding` / `heightBinding` 读写 `parameters.width` 和 `parameters.height`，不触碰 SwiftData、文件存储、ViewModel 或 native backend。
- 辅助功能 hint 属于 UI 表达，不应进入 ViewModel、数据层或 native 边界。
