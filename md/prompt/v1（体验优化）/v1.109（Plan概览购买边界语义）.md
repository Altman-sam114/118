# v1.109 / Plan 概览购买边界语义

## 版本号

- 版本：`v1.109`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.108 / Models 添加表单字段语义`。
- 人工总目标是优化 mac 版、iPad 版、UI、设计和付费功能，但本轮必须小步、可验收，不启用真实 StoreKit，不启用 Mac Catalyst，不需要付费账号或密钥。
- 本轮继续沿用 `v1（体验优化）` 的小步 UI / VoiceOver / paid-planning 透明度优化路线，递增为 `v1.109`。

## 背景

Plan 页面已经展示 Local plan、StoreKit 未配置、paid candidates planning-only、Purchase UI hidden、Mac/Catalyst not enabled 和 Mac readiness blockers。当前 Plan overview 的底部 note 写明没有 purchase state、entitlement 或 App Store product request，但 first-screen overview 还可以更直接地说明 paid candidates 不能在这里购买或解锁，让用户进入 Plan 后立即理解这是规划页面，不是真实购买入口。

本轮选择一个低风险、高价值的第一步：只补强 Plan overview 的购买边界可见文案和组合 VoiceOver value/hint，不改 StoreKit、Mac Catalyst、entitlement、purchase UI、Xcode project 或任何数据流。

## 目标

1. 在 `PlanView.planOverview` 的底部购买边界 note 中明确加入 paid candidates cannot be bought or unlocked here。
2. 同步更新 `planOverviewAccessibilityValue`，让 VoiceOver 组合朗读 Local plan、StoreKit not configured、no purchase state、no entitlement、no App Store product request，以及 paid candidates cannot be bought or unlocked here。
3. 视实现需要微调 overview 的 `accessibilityHint`，让它说明该 panel 汇总 current Local plan、planning-only paid candidates 和 purchase boundary。
4. 保持 Plan overview 的中性 checklist 图标、暗色科幻样式、布局结构、iPad regular 双栏/大字号单列回退行为不变。
5. 保持 Current Build、Platform Status、Mac Readiness、Capability Matrix、Entitlement Rules、Availability 的 row 数据、顺序和行为不变。

## 非目标

- 不启用 StoreKit，不新增 product IDs、purchase UI、restore flow、receipt validation、subscriptions、entitlement persistence、paid gate、App Store Connect 请求或真实购买流程。
- 不启用 Mac Catalyst，不修改 Xcode target/platform/scheme/signing/notarization/sandboxing，不新增 native Mac/Catalyst backend slice，不做真实 Mac 验证。
- 不修改 SwiftData schema、`AppFileStore`、`HuggingFaceDownloadManager`、`GenerationViewModel`、`ImageGenerationBackend`、native bridge、模型下载、生成、图库或 Prompt Library 行为。
- 不修改 `PlanStatusToken`、Plan status enum、Plan row 数据结构、导航结构、sidebar/tab 图标、workflow 或 native backend asset 元数据。
- 不新增截图验收、simulator smoke、本机完整 `xcodebuild`、真实 StoreKit 测试、真实 Mac 测试或真实 GGUF 推理作为默认本地步骤。

## 当前架构依据

- `PlanView` 位于 `LocalDiffusion/Views/RootContentView.swift`，属于 SwiftUI UI 展示层，只负责静态计划、平台状态和付费规划说明。
- Plan overview 当前由 `planOverview`、`planOverviewCopy` 和 `planOverviewAccessibilityValue` 组成；该区域已经使用 `.accessibilityElement(children: .ignore)` 暴露为一个组合 VoiceOver 元素。
- 本轮只修改 UI 文案和辅助功能语义，不触碰状态层、SwiftData、文件存储、native backend、StoreKit 或 Mac/Catalyst 平台配置。

## 实现步骤

1. 阅读 `LocalDiffusion/Views/RootContentView.swift` 中 `PlanView.planOverview`、`planOverviewCopy` 和 `planOverviewAccessibilityValue`。
2. 将 Plan overview 底部 note 从仅说明 no purchase state / no entitlement / no App Store product request，补强为同时说明 paid candidates cannot be bought or unlocked here。
3. 更新 `planOverviewAccessibilityValue`，确保组合 VoiceOver value 至少包含：
   - Local-first image generation app。
   - Paid capability planning is visible。
   - StoreKit is not configured。
   - No purchase state is stored。
   - No entitlement is granted。
   - No App Store product is requested from this screen。
   - Paid candidates cannot be bought or unlocked here。
4. 如需调整 overview `accessibilityHint`，只改 hint 文案，不改变 accessibility label、panel 合并方式或 visual hierarchy。
5. 不修改 Plan row arrays、status token、PlanNoteRow、PlanStatusRow、navigation、StoreKit、Mac Catalyst 或任何业务逻辑。
6. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前事实，说明 Plan overview 购买边界语义已补强。
