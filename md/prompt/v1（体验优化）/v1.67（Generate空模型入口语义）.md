# v1.67 / Generate 空模型入口语义

## 版本号

v1.67

## 版本分配依据

- v1.66 已完成 Models 未跟踪 GGUF 导入 editor toolbar accessibility 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段做小步、可验收的 Generate 空模型状态下一步入口增强。

## 背景

Generate Run section 中模型缺失时的 `Open Models` 次入口已经具备明确 VoiceOver value/hint。Generate 的 Model section 在 `readyModels.isEmpty` 时也展示 `Open Models` 按钮，但该按钮仍主要依赖可见标题，缺少当前阻塞状态和下载/导入 ready GGUF 的下一步语义；同时未显式保证 44pt 最小触控高度。

## 目标

- 为 Generate Model section 空模型状态下的 `Open Models` 按钮补充 VoiceOver value/hint。
- value 明确当前没有 ready local model。
- hint 说明该操作会打开 Models，以下载或导入可用于生成的 ready GGUF 模型。
- 为该按钮显式设置 44pt 最小触控高度。

## 非目标

- 不修改 `onShowModels()`、Root 导航、模型查询、模型选择绑定、生成门禁、生成/取消行为、Prompt/参数编辑、SwiftData、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project 或 GitHub Actions workflow。
- 不修改 iPad 双栏布局、宽屏 breakpoints、Run section 已有按钮语义或生成结果 handoff。
- 不新增购买、订阅、entitlement、价格、product ID 或 Mac 平台开关。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。
- Generate 页面只能通过 `GenerationViewModel` 和 `onShowModels` 与状态/导航协作，不直接触碰 SwiftData 文件存储或 native bridge。
- 当前核心流程文档要求 Generate 的运行门禁、Open Models/Edit Prompt 下一步入口、进度、取消按钮、结果预览和 Gallery handoff 提供明确可访问标签/值/提示。

## 实现步骤

1. 阅读 `GenerationView.swift` 中 `modelSection`、Run section 的 `Open Models` 次入口和 `GenerationGate`。
2. 只修改 `readyModels.isEmpty` 分支内的 Model section `Open Models` 按钮。
3. 保留 `Button { onShowModels() }` 行为不变。
4. 给该按钮增加 `.frame(minHeight: 44)`。
5. 给该按钮增加稳定 VoiceOver value/hint，说明 no ready local model 和打开 Models 下载/导入 ready GGUF 的下一步。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 语义事实。
