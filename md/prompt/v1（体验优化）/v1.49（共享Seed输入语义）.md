# v1.49 / 共享 Seed 输入语义

## 版本号

v1.49

## 版本分配依据

- 当前体验优化阶段已完成到 v1.48。
- v1.42 至 v1.48 已连续补充共享 `ParameterEditor` 中 Reset Defaults、Randomize Seed、Canvas Size preset、Sampler picker、Steps stepper、CFG slider、Width 和 Height stepper 的 VoiceOver 语义。
- `seedField` 当前有 `Seed` label 和当前值，但没有说明该文本框编辑当前 generation parameters 使用的 seed。
- `ParameterEditor` 同时用于 Generate 和 Prompt Template Editor，因此本轮继续选择共享参数编辑器的小步 VoiceOver 语义优化，不修改 seed 输入、随机范围、默认值或生成行为。

## 背景

Local Diffusion 的 seed 是生成参数的一部分，会随 Generate 使用，也会随 Prompt Template Editor 中的参数一起保存。当前 Seed 文本框会朗读当前数值，但 VoiceOver 用户聚焦时没有额外 hint 说明它是在编辑当前参数使用的 seed；旁边的 Randomize Seed 按钮已经说明会生成新随机 seed。

本轮只给 Seed 文本框增加通用操作 hint。TextField 标题、绑定、键盘类型、当前值、随机按钮、参数归一化、生成请求映射和模板保存/加载行为保持不变。

## 目标

- 在 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的 `seedField` 上补充：
  - `.accessibilityHint("Edits the seed value for these generation parameters.")`
- 保持现有 `TextField("Seed", value: $parameters.seed, format: .number)`、`.keyboardType(.numberPad)`、accessibility label 和 accessibility value 不变。
- 语义文案必须通用，不能写死 Generate screen，因为该组件也用于 Prompt Template Editor。
- 同步 README、flow、flowchart、update_log，记录共享 Seed 文本框获得 seed value 编辑说明的 VoiceOver hint。

## 非目标

- 不修改 `GenerationParameters.seed` 默认值、类型、范围、归一化或 native request mapping。
- 不修改 `randomizeSeed()`、Randomize Seed 按钮、seed field binding、键盘类型、TextField 格式或 Section 顺序。
- 不修改 Steps、CFG、Canvas Size preset、Width/Height、Sampler 或 Reset Defaults。
- 不修改 Generate、Prompt Library 的保存/加载模板行为。
- 不修改 native request mapping、stable-diffusion.cpp bridge、SwiftData schema、文件存储、ViewModel、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不新增 StoreKit import、product ID、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线或真机 GGUF 生成验收。

## 当前架构依据

- `ParameterEditor` 属于 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的共享 UI 展示层，只负责展示和用户输入。
- Generate 和 Prompt Template Editor 通过 `@Binding var parameters: GenerationParameters` 复用同一个参数编辑器。
- `seedField` 当前只在 UI 层读写 `parameters.seed`，不触碰 SwiftData、文件存储、ViewModel 或 native backend。
- 辅助功能 hint 属于 UI 表达，不应进入 ViewModel、数据层或 native 边界。
