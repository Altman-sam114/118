# v1.108 / Models 添加表单字段语义

## 版本号

- 版本：`v1.108`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.107 / Gallery 文件夹名称字段语义`。
- 本轮继续沿用小步 UI / VoiceOver 语义优化路线，递增为 `v1.108`。

## 背景

Models 页面已具备 Hugging Face GGUF 下载、URL 解析、键盘提交、错误提示、空状态、未跟踪文件导入、删除确认和模型详情操作语义。`AddModelView` 仍有多处输入字段主要依赖 placeholder 或 Picker 默认朗读，VoiceOver 用户在 iPad 表单中填写 Hugging Face 下载信息时缺少稳定的字段标签、当前值和用途说明。

## 目标

1. 为 `AddModelView` 的 Paste URL 输入框增加字段级 VoiceOver label、value 和 hint。
2. 为 Hugging Face 表单中的 `Display name`、`Repository`、`GGUF file path or direct URL`、`Revision` 输入框增加字段级 VoiceOver label、value 和 hint。
3. 为 `Family` Picker 增加字段级 VoiceOver label、当前 family value 和用途 hint。
4. 使用本地计算属性区分空值和当前 trimmed 值，避免空字符串在辅助功能中不可理解。
5. 保持现有可见 UI、键盘提交、URL 解析、Download disabled gate、错误提示和下载创建行为不变。

## 非目标

- 不修改 `ParsedHuggingFaceFileURL`、`HuggingFaceURLBuilder`、`applyHuggingFaceURL()`、`parseHuggingFaceURLIfReady()`、`submitModelIfReady()` 或 `addModel()` 的业务逻辑。
- 不修改 duplicate checks、`.gguf` validation、download state machine、untracked file scan/import/delete behavior。
- 不修改 `LocalModel`、SwiftData schema、`AppFileStore`、`HuggingFaceDownloadManager`、native backend、StoreKit、Mac Catalyst、Xcode project 或 workflow。
- 不新增真实付费功能、购买入口或 Mac/Catalyst 平台配置。

## 当前架构依据

- UI 层位于 `LocalDiffusion/Views/**`，本轮只修改 `LocalDiffusion/Views/Models/ModelLibraryView.swift` 中的 `AddModelView`。
- Models 页面只能收集输入、展示状态和调用既有回调；下载状态仍由 `HuggingFaceDownloadManager` 管理，文件路径仍由 `AppFileStore` 管理。
- 表单字段语义属于 SwiftUI 展示层增强，不应影响 SwiftData、文件存储或 native backend 边界。

## 实现步骤

1. 在 `AddModelView` 的 Hugging Face URL TextField 添加：
   - `accessibilityLabel("Hugging Face file URL")`
   - 当前 trimmed URL 或 `No Hugging Face URL` 的 value。
   - 提示用户粘贴 Hugging Face file URL 后使用 Parse action 填充模型字段的 hint。
2. 在 `Display name` TextField 添加：
   - `accessibilityLabel("Model display name")`
   - 当前 trimmed name 或 `No display name` 的 value。
   - 提示该字段为下载前必填，并作为 Models 列表显示名。
3. 在 `Repository` TextField 添加：
   - `accessibilityLabel("Hugging Face repository")`
   - 当前 trimmed repository 或 `No repository` 的 value。
   - 提示 direct URL 可让 repository 可选，否则使用 owner slash repository。
4. 在 `GGUF file path or direct URL` TextField 添加：
   - `accessibilityLabel("GGUF file path or direct URL")`
   - 当前 trimmed filename 或 `No GGUF file path or direct URL` 的 value。
   - 提示下载前必填，可输入 GGUF 文件路径或 Hugging Face direct URL。
5. 在 `Revision` TextField 添加：
   - `accessibilityLabel("Hugging Face revision")`
   - 当前 trimmed revision 或 `No revision` 的 value。
   - 提示默认 main，除非模型文件使用其他 branch、tag 或 commit。
6. 在 `Family` Picker 添加：
   - `accessibilityLabel("Model family")`
   - `family.rawValue` 的 value。
   - 提示该字段设置 Hugging Face 下载模型的 family metadata。
7. 将上述 value 抽为 `AddModelView` 私有计算属性，靠近现有 `parseURLAccessibilityValue` 和 `downloadAccessibilityValue`。
