# v1.62 / Generate 运行入口语义

## 版本号

v1.62

## 版本分配依据

- v1.61 已完成 Generate Save Template toolbar accessibility 语义并通过 `origin/main` 云端结果包验收。
- 本轮继续在 v1 体验优化阶段做小步、可验收的 SwiftUI 可访问性增强。

## 背景

Generate 页的 Run section 已有 `GenerationGatePanel` 暴露当前生成门禁状态，生成中进度、取消按钮和结果预览也有明确 VoiceOver 语义。未生成状态下的主 Generate 按钮、`Open Models` 次入口和 `Edit Prompt` 次入口仍主要依赖可见标题，VoiceOver 对 ready / blocked / 下一步动作的语义不够集中。

## 目标

- 为 Run section 的主生成按钮补充明确 VoiceOver label、value 和 hint。
- 为 Run section 中因模型缺失出现的 `Open Models` 次按钮补充当前阻塞上下文和下一步提示。
- 为 Run section 中因 positive prompt 缺失出现的 `Edit Prompt` 次按钮补充当前阻塞上下文和焦点行为提示。
- 保持 compact、iPad regular 双栏和 accessibility Dynamic Type 单栏布局现有行为。

## 非目标

- 不修改 `GenerationGate` 的判定条件。
- 不修改生成、取消、保存、Gallery handoff、模型选择或 prompt 编辑行为。
- 不修改 SwiftData schema、文件存储、native backend、C bridge、StoreKit、Mac Catalyst、Xcode project 或 GitHub Actions workflow。
- 不新增购买、订阅、entitlement、价格、product ID 或 Mac 平台开关。

## 当前架构依据

- UI 层只负责展示、输入、导航和用户反馈，位于 `LocalDiffusion/Views/**`。
- Generate 页只能通过 `GenerationViewModel` 和 `ImageGenerationBackend` 边界触发生成，不直接访问 native bridge。
- 当前核心流程文档要求 Generate 的运行门禁、进度、取消按钮、结果预览和 Gallery handoff 提供明确可访问标签、值和提示。

## 实现步骤

1. 阅读 `GenerationView.swift` 中 `runSection`、`generationGate`、`canGenerate` 和已有 accessibility helper。
2. 给未生成状态的主按钮增加：
   - 稳定 label，例如生成图片动作。
   - ready/blocked 的 `accessibilityValue`。
   - ready 时说明会用当前模型与 prompt 启动本地生成；blocked 时复用当前 gate 的阻塞 hint。
3. 给 `.openModels` 次按钮增加模型缺失 value 和下载/导入 ready GGUF 模型的 hint。
4. 给 `.focusPrompt` 次按钮增加 positive prompt 缺失 value 和移动焦点到 Positive Signal 的 hint。
5. 如需 helper，限制在 `GenerationView.swift` 内，避免扩大抽象。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 语义事实。

## 验收标准

- 主生成按钮在 ready 时向 VoiceOver 暴露 Ready 状态和会启动本地生成的提示。
- 主生成按钮在 backend/model/prompt 阻塞时向 VoiceOver 暴露 blocked 状态和当前 gate 的下一步提示。
- `Open Models` 和 `Edit Prompt` 次按钮暴露当前阻塞原因与下一步动作。
- 所有本地轻量检查和 Swift parse 通过。
- `origin/main` 最新 commit 的 CI artifact 经 Agent C 下载核对并通过。
