# v1.74 / Plan 可用性页脚语义

## 版本号

v1.74

## 版本分配依据

- v1.73 已完成 Mac readiness QA 状态表达，并通过 `origin/main` 云端结果包验收。
- 本轮继续优化 Plan 页面中付费候选和 Mac app 状态的说明语义，补齐 Availability 区块的页脚上下文。

## 背景

Plan 页面已经在 Mac Readiness、Capability Matrix 和 Entitlement Rules 区块提供 compact footer note 和 regular panel footer，帮助 VoiceOver 和视觉用户理解这些区块只是规划说明。Availability 区块同样承载 Local tools、planning-only paid candidates、Purchase UI prerequisite 和 Mac app not enabled 状态，但当前没有同等页脚上下文。

## 目标

- 为 Plan compact Availability section 增加 contextual footer note，并通过现有 `compactSectionFooter` 暴露 `Availability note` label/value。
- 为 iPad regular Availability `PlanPanel` 增加同一语义的 footer。
- 明确 Availability 区块只是在区分当前 Local tools、planning-only paid candidates、Purchase UI 前置条件和 Mac app 状态。
- 保持所有 Availability row 的 title、detail、status、icon、hint 和数据结构不变。

## 非目标

- 不启用 StoreKit product ID、购买、恢复购买、receipt、subscription、entitlement persistence 或 paid gate。
- 不启用 Mac Catalyst，不修改 Xcode target/platform、signing、native backend slice、XCFramework、workflow、SwiftData、文件存储或推理链路。
- 不修改 Plan row 数据、导航结构、状态枚举、布局结构或业务逻辑。
- 不新增截图验收、simulator smoke、真实 Mac 验证、真实 GGUF 推理或本机完整 build 作为默认本地步骤。

## 当前架构依据

- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责说明当前计划和状态。
- `compactSectionFooter` 已用于 compact Form 中的 Mac Readiness、Capability Matrix 和 Entitlement Rules。
- `PlanPanel` 已支持 optional footer，regular iPad layout 可复用该能力。

## 实现步骤

1. 阅读 `RootContentView.swift` 中 `regularDetailColumn`、`availabilitySection`、`availabilityContent`、`compactSectionFooter` 和 `PlanPanel`。
2. 为 `PlanPanel("Availability")` 增加 footer 文案。
3. 将 compact `availabilitySection` 从简单 `Section("Availability")` 调整为 header/footer 形式，并复用 `compactSectionFooter("Availability", ...)`。
4. 不修改 Availability rows、status token、accessibility hint、StoreKit/Mac facts 或任何数据流。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
