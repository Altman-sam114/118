# v1.90 / Prompt 模板行摘要语义

## 版本号

v1.90

## 版本分配依据

- v1.89 已完成 Models detail Native Loading mode 与 auxiliary model pickers 的当前值 VoiceOver 语义，并通过 `origin/main` 云端结果包验收。
- 本轮继续非 Plan 的体验优化，聚焦 Prompt Library 中重复模板行的快速扫描体验，让 VoiceOver 用户能先听到模板名称、prompt 和核心参数摘要，再按需进入现有 Edit/Load 按钮与 metric pill。

## 背景

`PromptTemplateRow` 当前以 sci-fi panel 形式展示模板名称、positive prompt、steps、sampler、canvas size 以及 Edit/Load 操作按钮。

现状：

- Edit/Load 按钮已经有 `Edit Template: <template name>` 和 `Load Template: <template name>` 的 accessibility label。
- steps/sampler/canvas size metric pills 已有独立 label/value/hint。
- 行卡片本身没有组合摘要，VoiceOver 用户需要逐项扫过标题、prompt、多个 metric pill 和按钮后才能理解整行内容。

本轮目标是为行卡片增加一个不隐藏子控件的可访问摘要，使列表扫描更高效，同时保留所有现有子控件、按钮和 metric pill 的可访问性。

## 目标

- 在 `PromptTemplateRow` 外层 panel 容器增加组合摘要：
  - 使用 `.accessibilityElement(children: .contain)` 保留子元素。
  - 增加行级 `accessibilityLabel`，例如 `Prompt template: <template name>`。
  - 增加行级 `accessibilityValue`，包含 positive prompt、steps、sampler 和 canvas size。
  - 增加行级 `accessibilityHint`，说明该行包含 Edit 与 Load controls。
- 可选新增私有 computed property，例如 `templateAccessibilityValue`，避免把长字符串直接堆在 `body`。
- 保留 `templateCopy`、`actions`、`metricPills`、Edit/Load button、metric pill、布局、颜色、spacing、Dynamic Type 分支和 `.sciFiPanel()` 不变。

## 非目标

- 不使用 `.accessibilityElement(children: .combine)` 或 `.ignore` 作用于整行，避免隐藏 Edit/Load 按钮或 metric pill。
- 不修改模板创建、保存、编辑、加载、删除、搜索、分类、排序或 `PromptTemplate.matchesSearch`。
- 不修改 `PromptTemplate`、`GenerationParameters`、SwiftData schema、`GenerationViewModel` 或任何数据持久化逻辑。
- 不修改 StoreKit、paid gates、Mac Catalyst、Xcode project、native backend、bridge、workflow 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Prompt Library 位于 `LocalDiffusion/Views/Prompts/PromptLibraryView.swift`，属于 UI 展示层。
- `PromptTemplateRow` 只负责展示单个 `PromptTemplate` 并通过外部闭包触发 load/edit；实际 load 由外层调用 `generation.load(template:)`。
- 本轮只改变 SwiftUI accessibility 修饰符和文档事实，不跨越 SwiftData、文件存储、GenerationViewModel 或 native backend 边界。

## 实现步骤

1. 阅读 `PromptTemplateRow.body`、`templateCopy`、`actions` 和 `metricPills`。
2. 在 `PromptTemplateRow.body` 外层 `VStack` 的 `.sciFiPanel()` 后追加行级 accessibility 修饰符。
3. 使用 `.accessibilityElement(children: .contain)`，确保 Edit/Load 按钮和 metric pill 仍保留独立辅助功能语义。
4. 新增 `templateAccessibilityValue` computed property，组合：
   - positive prompt 文本；
   - `template.steps`；
   - `template.samplerRawValue`；
   - `template.width` 和 `template.height`。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。

## 验收标准

- `PromptTemplateRow` 外层 panel 暴露行级 `Prompt template: <name>` label。
- 行级 value 包含 positive prompt、steps、sampler 和 canvas size。
- 行级 accessibility 使用 `.contain`，现有 Edit/Load buttons 和 metric pills 不被 `.combine` 或 `.ignore` 吃掉。
- 模板查询、分组、排序、搜索、编辑、加载、删除和保存行为不变。
- 所有本地轻量检查和 Swift parse 通过。
- `origin/main` 最新 commit 的 CI artifact 经 Agent C 下载核对并通过。

## 风险和禁止项

- 风险：错误使用 `.combine` 或 `.ignore` 会把 Edit/Load 按钮和 metric pill 合并或隐藏；实现后必须通过 diff 检查确认只使用 `.contain` 作用于外层 row。
- 禁止修改模板数据模型、SwiftData schema、GenerationViewModel、StoreKit、purchase UI、entitlement persistence、Mac Catalyst、native backend、workflow、Xcode project 或 Plan 文案。
- 禁止跳过 Agent C artifact 核对或用旧 run 冒充 v1.90。
