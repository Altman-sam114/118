# v1.9（Generate 动态字体宽屏回退）

## 版本号

v1.9

## 版本分配依据

v1.1 已建立 Generate 的 iPad regular 双栏创作台，v1.8 已为 Plan 补齐 accessibility Dynamic Type 下的单列回退和可换行状态展示。当前 Generate 仍只按 horizontal size class 切换双栏，未读取 `dynamicTypeSize`，在 accessibility 字号下双栏、控制台 header、状态 pill 和 metric grid 仍可能挤压。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：让 Generate 宽屏创作台在大字体下回退到更可读的单列，并调整控制台内部横排内容。

## 背景

Generate 是核心创作入口，iPad regular 下左侧放模型、prompt 和参数，右侧放控制台、运行门禁和结果预览。这个布局适合普通字号，但 accessibility Dynamic Type 下固定 `HStack` 双栏和控制台内部横排会压缩文本。Plan 已经采用 `dynamicTypeSize.isAccessibilitySize` 单列回退，这个模式可以小范围复用到 Generate。由于本轮只优化 SwiftUI 展示，不需要 StoreKit、Mac Catalyst、native backend 或 SwiftData 改动。

## 目标

- 在 `GenerationView.swift` 中读取 `@Environment(\.dynamicTypeSize)`。
- Generate 在 compact 下继续使用原有单列 `Form`。
- iPad regular 普通 Dynamic Type 下保持当前双栏创作台语义：左侧模型/prompt/参数，右侧控制台/run/result。
- iPad regular accessibility Dynamic Type 下回退到单列 `Form`，内容顺序应保持：console、model、prompts、parameters、run、result。
- 控制台 overview 在 accessibility Dynamic Type 下改为纵向图标和文案，避免固定图标横向挤压标题/说明。
- 控制台状态 pill 在 accessibility Dynamic Type 下改为纵向排列，避免两个 pill 横向争抢宽度。
- 控制台 metric grid 在 accessibility Dynamic Type 下改为单列，普通字号继续双列。
- 保持 Generate 行为不变：Save Template、模型选择、Open Models、Edit Prompt、Generate/Cancel、View in Gallery、alert、focus prompt、门禁判断和生成调用都不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不接入 StoreKit，不新增 product ID、价格、SKU、购买、恢复、receipt、subscription 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、native XCFramework、CI workflow 或 Xcode project。
- 不修改 SwiftData schema、文件存储、native bridge ABI、`GenerationViewModel`、`ImageGenerationBackend`、Models、Gallery、Prompts 或 Plan 状态行。
- 不改共享 `SciFiStatusPill` 或 `SciFiMetric` 的全局行为，避免扩大影响到其他页面。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `GenerationView` 属于 UI 层，只负责展示、输入、导航和用户反馈。
- 生成状态与业务动作由 `GenerationViewModel` 和 `ImageGenerationBackend` 边界负责，本轮不得越过这些边界。
- 当前 `GenerationView` 在 compact 下使用单列 `Form`，在 regular 下使用固定 `HStack` 双栏。
- `RootContentView.PlanView` 已有 `dynamicTypeSize.isAccessibilitySize` 单列回退，可作为本轮布局策略参考。
- SwiftUI 设计要求：避免固定屏幕宽度，不使用 `UIScreen.main.bounds`；支持 Dynamic Type；图标装饰应隐藏或搭配文本；状态不能只靠颜色表达。

## 实现步骤

1. 阅读必读文件、本提示词、`GenerationView.swift`、`ParameterEditor.swift` 和 v1.8 Plan 布局模式。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `GenerationView` 中加入 `@Environment(\.dynamicTypeSize)`。
4. 调整 `generationLayout`：
   - compact 继续 `compactGenerationLayout`。
   - regular 且 `dynamicTypeSize.isAccessibilitySize` 时使用单列布局。
   - regular 普通字号继续 `wideGenerationLayout`。
5. 可新增 `singleColumnGenerationLayout`，让 compact 和 accessibility regular 共享内容顺序，但不要改变 compact 的外观语义。
6. 将 `consoleSection` 内部拆成局部小 helper：
   - overview header：普通用横排，accessibility 用纵向。
   - status pill group：普通用横排，accessibility 用纵向。
   - metrics：普通双列，accessibility 单列。
7. 保持所有按钮、绑定、门禁和 `viewModel.generate(...)` 调用不变。
8. 同步文档：
   - `README.md`：说明 Generate iPad 创作台现在在 accessibility Dynamic Type 下回退单列。
   - `md/flow/flow.md`：更新 Generate UI 职责。
   - `md/flow/flowchart.md`：更新 Generate 节点。
   - `update_log.md`：追加 v1.9 记录。
