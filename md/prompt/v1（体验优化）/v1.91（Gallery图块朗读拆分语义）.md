# v1.91 / Gallery 图块朗读拆分语义

## 版本号

v1.91

## 版本分配依据

- v1.90 已完成 Prompt Library template row 的 row-level VoiceOver 摘要，并通过 `origin/main` 云端结果包验收。
- 本轮继续非 Plan 的体验优化，聚焦 Gallery image tile 的 VoiceOver 输出结构，让图块先读核心 prompt，再把 model/date/output size 放入 value，提升 iPad 和键盘/辅助功能扫图效率。

## 背景

`ImageTile` 当前通过：

```swift
.accessibilityElement(children: .ignore)
.accessibilityLabel(Text(accessibilitySummary))
.accessibilityHint("Opens image detail")
```

把 prompt、model name、created date、output size 全部组合进一个长 label。图块作为进入详情的单个交互入口，继续用 `.ignore` 合并子内容是合理的，但把所有信息塞进 label 会让 VoiceOver 的主名称过长，不利于快速扫描图片 prompt。

本轮只拆分 label/value：

- label：聚焦 generated image prompt。
- value：承载 model、created date、output size 等 metadata。
- hint：保留打开详情的行为说明。

## 目标

- 将 `ImageTile` 的 `accessibilityLabel` 从长 summary 改为 prompt-focused label，例如 `Generated image: <prompt>`。
- 新增 `accessibilityValue`，包含：
  - model name；
  - created date；
  - output width/height。
- 保留 `.accessibilityElement(children: .ignore)`、`.accessibilityHint("Opens image detail")`、`.hoverEffect(.highlight)`、`.contentShape(...)`、panel styling、visible prompt 和 metadata layout 不变。
- 可将现有 `accessibilitySummary` 重命名或拆分为 `accessibilityLabelText` / `accessibilityValueText`，但不得改变业务行为。

## 非目标

- 不修改 Gallery filtering、sorting、folder/tag selection、detail navigation、reuse/regenerate/share/delete、folder editor、tag editor 或 image deletion。
- 不修改 `GeneratedImage`、SwiftData schema、file storage、`AppFileStore`、`GenerationViewModel` 或 image loading behavior。
- 不修改 gallery grid layout、image tile visual design、thumbnail loading、date formatting outside accessibility metadata 或 Dynamic Type layout。
- 不修改 StoreKit、paid gates、Mac Catalyst、Xcode project、native backend、bridge、workflow 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Gallery 页面位于 `LocalDiffusion/Views/Gallery/GalleryView.swift`，属于 UI 展示层。
- `ImageTile` 只展示 `GeneratedImage` metadata 和 file-backed thumbnail；点击/导航由外层 `NavigationLink(value:)` 使用。
- 本轮只改变 SwiftUI accessibility 修饰符和文档事实，不进入 SwiftData、文件存储、GenerationViewModel 或 native backend 边界。

## 实现步骤

1. 阅读 `ImageTile.body`、`metadataView` 和现有 `accessibilitySummary`。
2. 将 `accessibilitySummary` 拆分为两个只读 computed properties：
   - prompt-focused label；
   - metadata-focused value。
3. 更新 body 中的 accessibility 修饰符：
   - `.accessibilityLabel(Text(accessibilityLabelText))`
   - `.accessibilityValue(Text(accessibilityValueText))`
   - 保留 `.accessibilityHint("Opens image detail")`
4. 核对没有改动 visible UI、NavigationLink、filter/sort logic、detail view 或 file loading。
5. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
