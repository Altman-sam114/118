# v1.6（Plan 宽屏布局）

## 版本号

v1.6

## 版本分配依据

v1.2 建立 Plan 入口，v1.4 增加能力矩阵，v1.5 增加 Mac readiness checklist。当前 Plan 信息量已经明显增加，但 iPad regular 下仍沿用单列 `Form`，用户需要纵向滚动才能比较 Local plan、Mac blockers 和付费能力候选。本轮继续推进人工总目标中的 iPad UI/design 优化：为 Plan 增加 iPad regular 宽屏双栏阅读布局。

## 背景

当前 Root 在 iPad regular size class 使用唯一顶层 `NavigationSplitView`，Plan 在 detail 内展示。Plan 的内容均为 UI 展示层信息，不涉及 SwiftData、文件存储、native backend、StoreKit 或 Xcode project。宽屏布局应该提高扫描和比较效率，但不能改变任何购买、平台或功能可用性事实。

## 目标

- `PlanView` 根据 horizontal size class 选择布局：
  - compact：保留现有单列 `Form` 结构和 Tab 场景体验。
  - regular：使用宽屏双栏阅读布局，减少长 Form 滚动。
- regular 左栏建议展示：
  - Local Plan overview。
  - Current Build。
  - Platform Status。
- regular 右栏建议展示：
  - Mac Readiness。
  - Capability Matrix。
  - Availability。
- 保持暗色科幻风格、44pt 以上可触区域、Dynamic Type 友好和状态文字+图标表达。
- 不新增购买、恢复、entitlement、StoreKit 或 Mac/Catalyst 行为。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不修改 Root 导航结构，不新增 tab/sidebar 入口。
- 不启用 Mac Catalyst，不修改 Xcode project 或 native XCFramework。
- 不接入 StoreKit，不新增 product ID、价格、SKU、订阅、恢复、receipt 或 entitlement。
- 不修改 Generate、Models、Gallery、Prompts、SwiftData schema、文件存储、native bridge ABI 或 CI workflow。
- 不做截图 smoke 或 simulator UI 验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `PlanView` 当前在 `RootContentView.swift` 内，由 `RootContentView.sectionContent(.plan)` 创建。
- `RootContentView` 已根据 horizontal size class 在 iPhone compact 和 iPad regular 间切换顶层导航。
- Plan 属于 UI 展示层，只能展示 plan、StoreKit 未配置、Mac readiness 和 capability matrix 状态。
- SwiftUI skill 要求布局支持 Dynamic Type，不只靠颜色区分状态，避免固定不可伸缩尺寸；本轮可使用 `ViewThatFits`、`ScrollView`、`VStack`、`LabeledContent` 等系统组件。
- 之前 SwiftUI `Section("...") { } footer:` 在完整 build 下暴露过兼容问题；新增带 footer 的 section 应使用显式 `Section { } header: { } footer: { }`。

## 实现步骤

1. 阅读必读文件、本提示词和 `RootContentView.swift`。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git switch main
git pull --ff-only origin main
git status --short
```

3. 在 `PlanView` 中读取 `@Environment(\.horizontalSizeClass)`。
4. 将原有 `Form` 提取为 compact 布局，保留现有内容顺序和语义。
5. 新增 regular 布局：
   - 使用 `ScrollView` 包裹内容。
   - 使用 `ViewThatFits(in: .horizontal)` 或等价方式，在空间不足时自动回退为单列堆叠，避免文字挤压。
   - 左右栏宽度用 flexible/maxWidth 约束，不使用 `UIScreen.main.bounds`。
   - 不把 UI 卡片嵌套在卡片内；overview panel 可独立放置，其他 section 可用轻量 panel 样式承载。
6. 抽取必要的小型私有 row/panel view 或 helper，避免复制同一状态行逻辑过多；但不要拆 Xcode project 文件或做大范围重构。
7. 同步文档：
   - `README.md`：说明 Plan 在 iPad regular 下有宽屏双栏布局。
   - `md/flow/flow.md`：更新 Plan UI 职责。
   - `md/flow/flowchart.md`：更新 Plan 节点。
   - `update_log.md`：追加 v1.6 记录。
