# v1.100 Generate 空模型状态摘要语义

## 版本号

v1.100

## 版本分配依据

v1.99 已完成 Generate 结果图库入口语义。当前继续 v1 体验优化阶段，选择 Generate Model section 中无 ready model 空状态卡片这一处可独立验收的 VoiceOver 汇总语义，因此递增为 v1.100。

## 背景

Generate 的无 ready model 状态会显示 `EmptyStateView`，随后提供已优化过的 `Open Models` 按钮。v1.67 已覆盖 `Open Models` 按钮的 value/hint 和 44pt 命中区，但空状态卡片本身仍依赖共享 `EmptyStateView` 的默认子元素朗读，可能把图标、标题和说明拆散读出。VoiceOver 用户需要先听到这个空状态的整体含义，再移动到下一步按钮。

## 目标

- 只在 `GenerationView.modelSection` 的无 ready model `EmptyStateView` 调用点添加组合 accessibility 语义。
- 空状态卡片作为一个元素朗读，label 为 `No local model`。
- value 明确说明当前没有 ready GGUF model 可用于 generation。
- hint 指向下一步：使用 `Open Models` 下载或导入 ready GGUF model。
- 保留可见标题、说明、图标、布局和 `Open Models` 按钮行为不变。

## 非目标

- 不修改共享 `EmptyStateView`，避免影响 Models、Gallery、Prompts 或其他调用点。
- 不修改 `Open Models` 按钮已有 value/hint、按钮样式、hit area 或 navigation action。
- 不修改 model query、ready model 判断、model selection binding、generation gate 或生成流程。
- 不修改 SwiftData schema、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或 Release asset。
- 不做 simulator VoiceOver 实机朗读或视觉重设计。

## 当前架构依据

- UI 层只在 `LocalDiffusion/Views/**` 展示状态、输入、导航和用户反馈。
- Generate 页面通过 `@Query` 获取 `LocalModel`，并用 `models.filter(\.isReady)` 得到 ready models。
- 无模型状态是 UI 展示与下一步入口，不应触碰下载状态机、文件存储或 SwiftData。
- SwiftUI 可访问性应使用 `accessibilityElement(children: .ignore)`、`accessibilityLabel`、`accessibilityValue` 和 `accessibilityHint` 提供可扫描语义。

## 实现步骤

1. 阅读必读文件、`GenerationView.swift`、共享 `EmptyStateView` 和最新 v1.99 提示词。
2. 在 `GenerationView.modelSection` 的无 ready model `EmptyStateView` 后追加 call-site 级 accessibility modifiers：
   - `.accessibilityElement(children: .ignore)`
   - `.accessibilityLabel(Text("No local model"))`
   - `.accessibilityValue(Text("No ready GGUF model is available for generation."))`
   - `.accessibilityHint(Text("Use Open Models to download or import a ready GGUF model before generating."))`
3. 不修改共享 `EmptyStateView` 类型本身。
4. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录本轮只改变 Generate 无 ready model 空状态辅助功能摘要。
