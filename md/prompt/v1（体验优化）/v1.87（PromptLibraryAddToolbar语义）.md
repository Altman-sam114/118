# v1.87 / Prompt Library Add toolbar 语义

## 版本号

v1.87

## 版本分配依据

- v1.86 已完成 Plan Current Build purchase note 文案优化，并通过 `origin/main` 云端结果包验收。
- 本轮避开连续多轮 Plan 文案，转向非 Plan 的 Prompt Library 可访问性小优化，聚焦 `Add` toolbar action 的 VoiceOver 语义。

## 背景

Prompt Library 的空状态会提示用户使用 Add 创建模板，新增模板 sheet 也会用当前 `generation.parameters` 作为初始参数。但 toolbar 中的 `Add` 按钮当前只暴露通用 `Label("Add", systemImage: "plus")`，没有明确说明它是新增 prompt template，也没有说明会打开使用当前 generation parameters 的新模板编辑器。对 VoiceOver、Voice Control 和外接键盘用户来说，通用 Add 在多页面环境中上下文不够具体。

## 目标

- 给 Prompt Library 顶部 `Add` toolbar button 增加 `accessibilityLabel(Text("Add prompt template"))`。
- 增加 `accessibilityValue(Text("Ready"))`。
- 增加 `accessibilityHint(Text("Opens a new prompt template editor using the current generation parameters."))`。
- 保持按钮可见文案、SF Symbol、toolbar placement、`showingAddTemplate = true` action 和 sheet 初始化逻辑不变。

## 非目标

- 不修改模板创建逻辑、`PromptTemplateEditor`、保存字段、`initialParameters: generation.parameters`、搜索、分类、编辑、加载或删除行为。
- 不修改 SwiftData schema、文件存储、native backend、bridge、XCFramework、Xcode project、workflow 或 CI。
- 不修改 Plan、StoreKit、paid candidates、purchase UI、entitlement persistence、Mac Catalyst 或 Mac readiness 语义。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 StoreKit 测试、Mac build 或真实 GGUF 生成作为默认本地步骤。

## 当前架构依据

- Prompt Library 位于 `LocalDiffusion/Views/Prompts/PromptLibraryView.swift`，属于 UI 展示层。
- `Add` toolbar button 只切换 `showingAddTemplate`，sheet 内部沿用 `PromptTemplateEditor` 和 `generation.parameters`。
- 本轮只增加辅助功能语义，不触碰状态层、SwiftData 模型、文件存储或 native backend。

## 实现步骤

1. 阅读 `PromptLibraryView.swift` 中 `ToolbarItem(placement: .primaryAction)` 和新增模板 sheet。
2. 只给现有 `Add` button 增加 accessibility label/value/hint。
3. 不修改按钮 action、sheet、editor、模板保存逻辑、搜索和分类逻辑。
4. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
