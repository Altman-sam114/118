# v1.126 / Run 操作命中区

## 版本号

- 版本：`v1.126`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.125 / 进度阶段文本可读性`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.120-v1.125 已持续收敛 Generate 的 iPad regular、窄宽度、控制台和运行状态可读性；本轮继续处理 Generate Run 区域中主要和次要操作的触控命中区一致性。

## 背景

`GenerationView.runSection` 中 Cancel、Generate、Open Models 和 Edit Prompt 都是生成流程的关键操作。共享 `SciFiPrimaryButtonStyle` 和 `SciFiSecondaryButtonStyle` 已提供视觉 padding 和 pointer hover affordance，但 Run 区按钮 call site 没有显式声明 44pt 最小高度。项目中同页 no-model 的 Open Models 入口已经单独使用 `.frame(minHeight: 44)`；本轮将同一可访问命中区约束补齐到 Run 区操作，减少 iPad 窄栏、较大字号或系统布局差异下的交互风险。

## 目标

1. 只修改 `GenerationView.runSection` 中的操作按钮布局。
2. 给正在生成状态的 Cancel 按钮增加 `.frame(minHeight: 44)`。
3. 给非生成状态的主 Generate 按钮增加 `.frame(minHeight: 44)`。
4. 给 secondary Open Models 和 Edit Prompt 按钮增加 `.frame(minHeight: 44)`。
5. 保持按钮文案、system image、button style、disabled 状态、辅助功能 label/value/hint、generation gate 和生成逻辑不变。

## 非目标

- 不修改 `SciFiPrimaryButtonStyle` 或 `SciFiSecondaryButtonStyle`。
- 不修改 `GenerationGatePanel`、`generationGate`、`canGenerate`、`viewModel.generate`、`viewModel.cancel`、progress、result section、model selection、prompt editor 或 ParameterEditor。
- 不修改 SwiftData、文件存储、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不引入固定宽度、固定总高度、缩小字体、单行限制或截断策略。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GenerationView` 位于 `LocalDiffusion/Views/Generation/GenerationView.swift`，属于 SwiftUI UI 展示层。
- `runSection` 只根据 `viewModel.isGenerating` 和 `generationGate` 展示当前进度、主操作和下一步入口。
- 本轮只改变按钮命中区约束，不触碰状态层、数据层、文件存储或 native backend。

## 实现步骤

1. 阅读 `runSection` 中 Cancel、Generate、Open Models 和 Edit Prompt 按钮。
2. 在每个按钮的 `.buttonStyle(...)` 后追加：

```swift
.frame(minHeight: 44)
```

3. 保持后续 `.disabled(...)` 和 accessibility modifiers 原样。
4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Generate Run 操作按钮显式保持 44pt 最小命中区。
5. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
