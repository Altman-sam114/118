# v1.24 / Add Model 错误提示可读性

## 版本号

v1.24

## 版本分配依据

- 当前体验优化阶段已完成到 v1.23。
- v1.22 和 v1.23 已分别优化 Models 列表行与详情页的状态消息可读性。
- `AddModelView` 中的 `errorMessage` 仍使用普通 danger 色 `Text` 展示，缺少图标、44pt 最小高度、可换行约束和明确 VoiceOver 语义。

## 背景

Add Model sheet 负责粘贴 Hugging Face URL、手动填写 repository/file/revision/family，并在用户输入不合法、来源重复或本地文件冲突时显示 `errorMessage`：

```swift
if let errorMessage {
    Section {
        Text(errorMessage)
            .foregroundStyle(SciFiTheme.danger)
    }
    .listRowBackground(SciFiTheme.panel)
}
```

这些错误直接决定用户下一步如何修正输入。本轮只优化错误消息的展示，不改变错误产生条件、解析规则、重复检测、文件存在检查或下载创建流程。

## 目标

- 新增私有 `AddModelErrorRow` 或等价小组件。
- 用该组件替换 `AddModelView` 中的裸 `Text(errorMessage)`。
- 保留原有 `errorMessage` 文案、显示位置和 `SciFiTheme.danger` 语义。
- 错误行应：
  - 使用真实 `Label` 和 `exclamationmark.triangle` SF Symbol。
  - 使用 `SciFiTheme.danger`。
  - 文本可自然换行，不使用缩放压缩。
  - 具有 44pt 最小高度。
  - 提供明确 VoiceOver label/value/hint，例如 label 为 `Add model error`，value 为实际错误文本，hint 说明用户需要修正模型来源或文件信息。
- 保持 Add Model 的 URL parse、text fields、Picker、Download disabled gate、`addModel()`、`applyHuggingFaceURL()` 和 `dismiss()` 行为不变。
- 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`，记录 Add Model 错误提示可读性改善。

## 非目标

- 不修改 `ParsedHuggingFaceFileURL`、`HuggingFaceURLBuilder` 或任何 URL 解析规则。
- 不修改 `errorMessage` 的具体文案、产生条件或清空时机。
- 不修改 `canSubmit`、`addModel()`、`applyHuggingFaceURL()`、`onAdd(model)`、`dismiss()` 或下载启动行为。
- 不修改 `LocalModel`、SwiftData schema、`AppFileStore`、文件存在检测、重复模型检测或本地文件命名规则。
- 不修改 Models 列表行、详情页、未跟踪文件导入编辑器、alert、confirmation dialog 或 fileImporter。
- 不修改 native backend、生成流程、StoreKit、Mac Catalyst、Xcode project 或 CI workflow。

## 当前架构依据

- `ModelLibraryView` 属于 UI 展示层，只负责展示模型状态、用户输入和操作反馈。
- Add Model sheet 的错误状态来自本地 `@State private var errorMessage: String?`，本轮只能改变展示，不应改变业务判断。
- 模型文件路径和存在性必须继续通过 `AppFileStore` 处理。
- SwiftUI 可访问性要求：重要错误信息应可读、可换行，并提供明确 label/value/hint；展示行应满足 44pt 阅读/触控基线。
