# v1.12（Gallery 图块可读性）

## 版本号

v1.12

## 版本分配依据

v1.8、v1.9、v1.10、v1.11 已分别优化 Plan、Generate、Models、Prompts 的 Dynamic Type 和控件可访问性。当前 Gallery 图块仍使用较小的 `caption2` 元数据，图片尺寸徽章和日期/模型行在大字号下容易变窄挤压。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升 Gallery 网格图块的可读性和辅助功能语义，不改图库业务行为。

## 背景

Gallery 是生成结果查看、筛选、复用和再生成的入口。图块承担图片缩略图、prompt、生成日期、模型名和输出尺寸的快速浏览任务。`swiftui-pro` accessibility 规则要求尊重 Dynamic Type，避免过小字体，并为不清晰的图片内容提供有用的 VoiceOver 读法；设计规则也要求避免固定窄布局导致文本在大字号下被挤压。

## 目标

- 在 `GalleryView.swift` 中只优化 Gallery 展示层。
- Gallery 网格在 accessibility Dynamic Type 下使用更宽的自适应最小列宽，减少图块文本横向挤压。
- `ImageTile` 读取 Dynamic Type size，在 accessibility 字号下将日期和模型名纵向排列。
- 将图块尺寸徽章和底部元数据从 `caption2` 提升到更可读的 Dynamic Type 字体。
- 图块 prompt 在 accessibility 字号下允许更多行数，减少过早截断。
- 为图块提供由 prompt、模型、日期和输出尺寸组成的辅助功能 label，让 VoiceOver 不只读到无语义图片或零散文本。
- 保持 Gallery 过滤、排序、文件 reconciliation、详情、Reuse、Regenerate、Delete、Share、folder/tag 编辑和跳转行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不接入 StoreKit，不新增 product ID、价格、SKU、购买、恢复、receipt、subscription 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、native XCFramework、CI workflow 或 Xcode project。
- 不修改 `GenerationViewModel`、SwiftData schema、`GeneratedImage` 模型、文件存储、native bridge、Generate、Models、Prompts 或 Plan。
- 不改变 Gallery 的过滤、排序、删除、文件清理、复用参数、再生成或分享业务路径。
- 不重构 Gallery 导航结构，不改 iPad embedded filter rail 的业务语义。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- Gallery 属于 UI 层，只负责展示、输入、导航和用户反馈。
- `GeneratedImage` 元数据由 SwiftData 保存，PNG 文件由 `AppFileStore` 管理；本轮不得改 schema、文件名、文件保存或删除语义。
- Gallery 详情中的 Reuse/Regenerate 通过 `GenerationViewModel.load(image:)` 和 `generate(using:modelContext:)` 完成，本轮不得改变跨页回填和生成调用。
- `GenerationView`、`PlanView`、`ModelLibraryView` 和 `PromptLibraryView` 已使用 `dynamicTypeSize.isAccessibilitySize` 做大字号回退，本轮可在 Gallery 网格和图块局部复用这个布局策略。
- SwiftUI 设计要求：支持 Dynamic Type；避免过小的 `caption2` 作为关键文字；图片若对 VoiceOver 不清晰，需要隐藏装饰性朗读或提供明确 label。

## 实现步骤

1. 阅读必读文件、本提示词、`GalleryView.swift` 和 v1.8-v1.11 的 Dynamic Type 布局模式。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `GalleryView` 读取 `@Environment(\.dynamicTypeSize)`，为 `imageGrid` 增加局部自适应最小列宽：普通字号保持当前紧凑网格，accessibility 字号使用更宽列宽。
4. 在 `ImageTile` 读取 `@Environment(\.dynamicTypeSize)`。
5. 将缩略图尺寸徽章字体从 `caption2` 改为更可读的 Dynamic Type 字体，并保持 monospaced digit 和 Sci-Fi 胶囊视觉。
6. 将 prompt 行数改为普通字号最多 2 行，accessibility 字号最多 3 行。
7. 将底部日期/模型元数据抽成局部布局：普通字号保留横向 `HStack`；accessibility 字号使用纵向 `VStack`，模型名允许换行。
8. 为 `ImageTile` 增加合成辅助功能 label，包含 prompt、model、date、output size；避免 VoiceOver 只读不清晰的图片内容。
9. 保持 `NavigationLink`、`ImageDetailView`、`reconcileImageFiles()`、`deleteImage(_:)`、`generation.load(image:)` 和 regenerate 调用路径不变。
10. 同步文档：
    - `README.md`：说明 Gallery 网格图块支持大字号可读布局和辅助功能 label。
    - `md/flow/flow.md`：更新 Gallery UI 职责。
    - `md/flow/flowchart.md`：更新 Gallery 节点。
    - `update_log.md`：追加 v1.12 记录。

## 验收标准

- Gallery 网格在 accessibility Dynamic Type 下不再强制使用 150pt 最小列宽。
- `ImageTile` 的日期和模型元数据在 accessibility Dynamic Type 下不再强制挤在同一横排。
- 图块关键元数据不再使用过小的 `caption2`。
- 图块 prompt 在 accessibility Dynamic Type 下允许最多 3 行。
- VoiceOver 能读到包含 prompt、模型、日期和输出尺寸的图块 label。
- 普通字号下 Gallery 仍保持紧凑暗色科幻网格风格。
- Gallery 过滤、排序、详情、Reuse、Regenerate、Delete、Share、folder/tag 编辑和跳转行为保持不变。
- 不新增 StoreKit、Mac Catalyst、Xcode project、native backend、SwiftData schema、文件存储或 CI workflow 改动。
- 本地轻量检查和 Swift parse 通过；建议的 iPhoneOS build 通过或明确记录沙箱限制。
- `origin/main` 最新 v1.12 run 成功，Agent C 下载并核对结果包通过。
