# v1.20 / Root 侧栏可访问性语义

## 版本号

v1.20

## 版本分配依据

- 当前体验优化阶段已完成到 v1.19。
- v1.19 补齐了 Plan Availability 状态行语义；下一处小范围 iPad / Mac-readiness 体验缺口在 Root iPad sidebar。
- 当前 iPad regular 使用唯一顶层 `NavigationSplitView`，sidebar row 仍是普通 `Label`，没有显式选择状态、用途 hint 或最小触控高度约束。

## 背景

Local Diffusion 的 Root 在 compact 下使用 `TabView`，在 regular size class 下使用 `NavigationSplitView`。iPad sidebar 是 Generate、Models、Gallery、Prompts、Plan 五个主入口的全局导航，也会影响未来 Mac/sidebar QA 的基础体验。

当前 sidebar row 代码为：

```swift
Label(section.title, systemImage: section.systemImage)
    .tag(section)
    .sciFiListRow()
```

SwiftUI `List(selection:)` 负责 selection，但 row 本身没有显式 VoiceOver value/hint，也没有明确 44pt 行高约束。本轮只补齐 row 语义和触控可读性，不改变导航行为。

## 目标

- 为 iPad regular sidebar 增加私有 `SidebarSectionRow`。
- 每个 sidebar row 保留可见标题和 SF Symbol。
- 每个 row 至少保持 44pt 可触控高度。
- 为 VoiceOver 提供明确：
  - label：当前 section title。
  - value：Selected 或 Not selected。
  - hint：说明此 row 会打开对应 workspace。
- 保持当前 `NavigationSplitView`、`List(selection:)`、`sidebarSelectionBinding`、`sectionContent` 和 Gallery embedded wide 逻辑不变。
- 更新 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`，记录 Root sidebar 语义改善。

## 非目标

- 不改变 iPhone compact `TabView`。
- 不改变 iPad `NavigationSplitView` 架构，不新增嵌套 split。
- 不改变 `selection` 默认值、tag、binding 或跨页跳转。
- 不改变 Gallery embedded wide / standalone 模式。
- 不新增页面、工具栏、菜单、快捷键或 command。
- 不接入 StoreKit，不新增购买、恢复、product ID、entitlement 或付费门禁。
- 不启用 Mac Catalyst，不修改 Xcode project、native backend、SwiftData schema、文件存储或 CI workflow。

## 当前架构依据

- `RootContentView` 是唯一全局导航层，负责 iPhone TabView 与 iPad NavigationSplitView 的切换。
- iPad regular 只能有一个顶层 split，Gallery 在 Root detail 中使用 embedded wide 布局，避免 split 嵌套。
- UI 层只能展示、输入、导航和用户反馈，不直接碰 SwiftData schema、文件存储或 native backend。
- SwiftUI 可访问性要求：有交互含义的 row 应提供清晰 label/value/hint；触控目标应满足 44pt。

## 实现步骤

1. 阅读必读文件和本提示词：
   - `AGENTS.md`
   - `update_log.md`
   - `md/flow/flow.md`
   - `md/flow/flowchart.md`
   - `md/test/test.md`
   - `README.md`
   - `md/prompt/README.md`
   - `.github/workflows/ci-results.yml`
   - `LocalDiffusion/Views/RootContentView.swift`
2. 在 `RootContentView.swift` 中新增私有 `SidebarSectionRow`。
   - 输入：`section: AppSection`、`isSelected: Bool`。
   - 使用 `Label(section.title, systemImage: section.systemImage)` 保持视觉和系统语义。
   - 加 `.frame(minHeight: 44, alignment: .leading)`。
   - 加 `.accessibilityElement(children: .ignore)`。
   - 加 `.accessibilityLabel(section.title)`。
   - 加 `.accessibilityValue(isSelected ? "Selected" : "Not selected")`。
   - 加 `.accessibilityHint(...)`，为 Generate / Models / Gallery / Prompts / Plan 分别说明打开的 workspace。
3. 将 sidebar `List` row 替换为 `SidebarSectionRow(section:isSelected:)`。
   - `.tag(section)` 必须保留在 row 上。
   - `.sciFiListRow()` 必须保留。
4. 不修改 compact tabLayout、不修改 `sectionContent`、不修改 `sidebarSelectionBinding`。
5. 更新文档：
   - README 当前实现和维护日志。
   - `md/flow/flow.md` Root 导航/UI 描述。
   - `md/flow/flowchart.md` Root 导航节点描述。
   - `update_log.md` 新增 v1.20 记录。

- iPad sidebar row 使用私有 helper，保留 `Label`、title、SF Symbol、`.tag(section)` 和 `.sciFiListRow()`。
- VoiceOver 可获得 section title、Selected / Not selected value 和具体 workspace hint。
- row 有 44pt 最小高度。
- `selection`、`sidebarSelectionBinding`、`sectionContent`、Gallery layout mode 和 compact TabView 行为不变。
- 没有 StoreKit、Mac Catalyst、SwiftData schema、native backend、CI workflow 或业务门禁变更。
- 本地要求的检查通过。
- `origin/main` 最新 commit 对应的 GitHub Actions 结果包通过 Agent C 核对。

## 风险和禁止项

- 禁止改变 Root 全局导航状态流。
- 禁止把 sidebar row 改成自定义按钮而破坏 `List(selection:)`。
- 禁止移除 `.tag(section)` 或改变 `sidebarSelectionBinding`。
- 禁止新增 nested `NavigationSplitView`。
- 禁止引入 StoreKit、Mac Catalyst、Xcode project、native backend 或数据层改动。
- 禁止跳过 Agent C artifact 验收。
- 禁止把旧 artifact、旧 run 或本地输出冒充本轮云端结果。
