# v1.132 / Plan 状态摘要行宽度回退

## 版本号

- 版本：`v1.132`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.131 / Gallery 宽屏筛选回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- 近期多轮已经改善 Generate 和 Gallery 的 iPad 窄宽度回退；本轮转向 Plan 页面，改善 Local plan、StoreKit 未配置状态和 Mac readiness 摘要行在宽屏与受限宽度下的阅读密度。

## 背景

`PlanStatusSummaryRow` 当前始终把标题和状态徽章纵向排列。这个布局在 compact Form 中稳定，但在 iPad regular 的 Plan 双栏面板中，Current Build 和 Platform Status 里的短标题/短状态会占用较多垂直空间；在未来 Mac/Catalyst 窗口验证前，这类摘要行也需要有明确的宽度回退语义。本轮只调整摘要行容器：普通 Dynamic Type 下优先横排标题与状态，空间不足时回退到既有纵向布局；accessibility Dynamic Type 继续直接使用纵向布局。

## 目标

1. 修改 `PlanStatusSummaryRow` 的内部布局。
2. 普通 Dynamic Type 下使用 `ViewThatFits(in: .horizontal)`：
   - 优先显示横向标题 + 状态徽章。
   - 横向空间不足时回退既有纵向标题 + 状态徽章。
3. accessibility Dynamic Type 下保持纵向布局，避免大字号横向挤压。
4. 让横向状态徽章不要强制占满整行；既有纵向状态徽章继续可撑满行宽。
5. 保留所有可见文案、状态 token、VoiceOver label/value/hint、Plan 数据、StoreKit 未配置边界、Mac/Catalyst 未启用说明不变。

## 非目标

- 不启用 StoreKit、购买按钮、restore flow、receipt validation、entitlement persistence、feature flag 或 paid unlock gate。
- 不修改 paid candidates 的业务含义、付费文案或可购买状态。
- 不修改 Mac Catalyst、Xcode project platform settings、native backend slice、signing、sandboxing、notarization 或 workflow。
- 不修改 Generate、Models、Gallery、Prompts、SwiftData、AppFileStore、GenerationViewModel、native backend 或测试规范。
- 不做 simulator / Mac Catalyst / Stage Manager 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `RootContentView.swift` 中的 `PlanView` 属于 SwiftUI UI 展示层，只负责展示 Local plan、付费候选规划状态、StoreKit 边界和平台 readiness。
- 当前 Root iPad 使用单层 `NavigationSplitView`，Plan regular 布局使用双栏阅读面板，accessibility Dynamic Type 下回退单列。
- Plan 页面当前明确不请求 App Store product、不存储 entitlement、不展示购买 UI；本轮不得改变该边界。

## 实现步骤

1. 阅读 `PlanStatusSummaryRow`、`PlanStatusBadge`、`currentBuildContent` 和 `platformStatusContent`。
2. 给 `PlanStatusSummaryRow` 读取 `dynamicTypeSize` 环境。
3. 抽出两个私有子视图：
   - `horizontalRow`：`HStack` 内显示标题 label、spacer 和非全宽状态徽章。
   - `verticalRow`：保留既有 `VStack` 标题 label + 全宽状态徽章。
4. 普通 Dynamic Type 下使用 `ViewThatFits(in: .horizontal)` 在 `horizontalRow` 和 `verticalRow` 之间选择。
5. accessibility Dynamic Type 下直接使用 `verticalRow`。
6. 给 `PlanStatusBadge` 增加可选参数，例如 `fillsWidth: Bool = true`，默认保持现有行为；横向摘要行传 `false`。
7. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Plan summary rows 在普通字号下可横排并在窄宽度回退。
8. 检查 diff，确认只包含本轮相关源码、文档和本提示词。
