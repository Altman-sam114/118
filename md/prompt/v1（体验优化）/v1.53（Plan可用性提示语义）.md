# v1.53 / Plan 可用性提示语义

## 版本号

v1.53

## 版本分配依据

- 当前体验优化阶段已完成到 v1.52。
- Plan 页面已具备 Local plan、StoreKit 未配置、能力矩阵、entitlement rules、availability rows、Mac readiness 和 iPad regular 双栏布局。
- Availability rows 当前通过 label/value 暴露标题、详情和状态，但 VoiceOver hint 没有进一步强调“Paid candidates 仅规划且未售卖/未解锁”“Purchase UI 需要 StoreKit 与 entitlement mapping”“Mac app 当前未启用”这些关键付费/Mac 边界。
- 本轮选择 Plan Availability rows 的小步语义优化，不修改任何 StoreKit、购买、entitlement、Mac Catalyst、Xcode 或业务门禁行为。

## 背景

Plan 页面用于展示当前 Local plan、未来付费能力规划和 Mac 可行性状态。Availability 区是用户判断哪些能力当前可用、哪些只是规划或未启用的关键位置。可视文案已经明确，但 VoiceOver 用户在逐行浏览时更适合听到带上下文的 hint，避免把 paid candidates 误解为已售卖/已解锁，或把 Mac readiness 误解为当前 Mac app 已发布。

本轮只为 Availability rows 补充行级 hint。可视文案、状态 badge、Plan facts、compact/regular 布局、StoreKit 未配置事实和 Mac Catalyst 未启用事实保持不变。

## 目标

- 在 `LocalDiffusion/Views/RootContentView.swift` 的 Plan 相关代码中：
  - 为 `PlanAvailabilityItem` 增加 `accessibilityHint` 字段。
  - 为四个 availability item 分别写明确 hint：
    - Core local tools：确认当前 Local plan 可用。
    - Paid candidates：明确 batch queue、curated prompt packs、workflow export 仅规划，当前未售卖或解锁。
    - Purchase UI：明确必须等 StoreKit products 和 entitlement mapping 存在后才可添加。
    - Mac app：明确当前 iOS target 不发货 Mac 或 Catalyst app。
  - 让 `AvailabilityRow` 把 item hint 传给 `PlanStatusRow`。
  - 让 `PlanStatusRow` 支持可选 hint，同时保持现有 label/value 语义。
- 同步 README、flow、flowchart、update_log，记录 Plan Availability rows 提供付费候选、购买 UI 和 Mac app 的 VoiceOver hint。

## 非目标

- 不修改 `PlanCapabilityItem`、`PlanEntitlementRuleItem`、`MacReadinessItem` 的事实、状态、顺序或可视文案。
- 不修改 Plan compact Form、iPad regular 双栏/单列回退、panel footer、badge 颜色、图标、row layout 或 note row。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、Xcode project、native XCFramework、签名或 workflow。
- 不修改 SwiftData schema、文件存储、`GenerationViewModel`、native backend 或 Release asset。
- 不新增 simulator VoiceOver 自动化、截图基线、Mac build、StoreKit 测试或真机 GGUF 生成验收。

## 当前架构依据

- `PlanView` 位于 `LocalDiffusion/Views/RootContentView.swift`，属于 UI 展示层，只负责 Plan 信息架构、平台状态和付费规划事实展示。
- `PlanAvailabilityItem` 已是 Availability rows 的本地 UI 数据模型；新增 hint 属于展示语义，不影响持久化、业务状态或 entitlement。
- `PlanStatusRow` 是 Plan 内部共享 row 组件；给它增加可选 hint 可以只让 AvailabilityRow 提供额外说明，保持其它 rows 现有行为不变。
- StoreKit 和 Mac Catalyst 的真实启用仍需要人工提供产品/平台/native slice/签名等外部决策，本轮不得伪装完成。

## 验收标准

- Plan Availability rows 的可视文案、状态、顺序和布局不变。
- VoiceOver 读取 Core local tools row 时能听到当前 Local plan 可用的 hint。
- VoiceOver 读取 Paid candidates row 时能听到这些候选仅规划、当前未售卖或未解锁。
- VoiceOver 读取 Purchase UI row 时能听到需要 StoreKit products 和 entitlement mapping 后才能添加购买 UI。
- VoiceOver 读取 Mac app row 时能听到当前 iOS target 不发货 Mac 或 Catalyst app。
- 本轮不修改 StoreKit、entitlement、Mac Catalyst、Xcode project、native backend、SwiftData schema、文件存储或 CI workflow。
- 本地要求的检查通过。
- `origin/main` 最新 commit 对应的 GitHub Actions 结果包通过 Agent C 核对。
