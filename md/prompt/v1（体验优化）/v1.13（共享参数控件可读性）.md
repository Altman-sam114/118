# v1.13（共享参数控件可读性）

## 版本号

v1.13

## 版本分配依据

v1.9 优化了 Generate 宽屏创作台在 accessibility Dynamic Type 下的回退，v1.10、v1.11、v1.12 分别优化 Models、Prompts、Gallery 的控件和图块可读性。当前共享 `ParameterEditor` 仍有 Seed 随机按钮只用 `Image`，Seed/Size/CFG 行在大字号下强制横排；共享 `SciFiStatusPill` 仍通过 `lineLimit(1)` 与 `minimumScaleFactor` 压缩状态文字，`SciFiMetric` 也固定横排。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升共享参数控件、状态 pill 和 metric 卡片在大字号与辅助功能下的可读性，不改任何生成参数语义。

## 背景

`ParameterEditor` 同时服务 Generate 和 Prompt Template Editor，是 steps、CFG、seed、size、sampler 等核心生成参数的共享入口。`SciFiStatusPill` 被 Generate、Models、Prompts 共用，`SciFiMetric` 被 Generate 控制台使用。共享组件一旦在 accessibility Dynamic Type 下挤压，会影响多个高频页面。`swiftui-pro` accessibility 规则要求图标按钮必须包含文本 label，即使视觉上继续 icon-only；设计规则要求避免过小字体、固定横向挤压和低于 44x44 的触控目标。

## 目标

- 在 `ParameterEditor.swift` 中只优化共享 UI 展示层和控件语义。
- `ParameterEditor` 读取 Dynamic Type size，在 accessibility 字号下让 CFG、Seed、Size 等横向参数行改为纵向或更可换行的布局。
- Steps、CFG、Seed、Size、Width、Height、Sampler 等共享参数控件补足明确的 accessibility label/value，避免只依赖横向文案或 placeholder。
- Seed 随机按钮改为带文本的 `Label("Randomize Seed", systemImage: "dice")`，普通字号可继续 icon-only，且满足至少 44x44 命中区。
- Size preset `Menu` 保留当前 preset 行为，在 accessibility Dynamic Type 下不再与 “Size” 标签强制挤在同一横排。
- `SciFiStatusPill` 在 accessibility Dynamic Type 下不再通过 `minimumScaleFactor` 压缩文字，允许长状态自然换行，并保持当前 Sci-Fi 状态视觉。
- `SciFiMetric` 在 accessibility Dynamic Type 下不再强制图标和文字横排，避免值文本被截断。
- 保持 Generate、Prompt Template Editor、Models、Prompts 调用共享组件的业务行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不修改 `GenerationParameters` 默认值、范围、归一化、sampler raw value、seed 随机范围或任何生成参数语义。
- 不修改 `GenerationViewModel`、SwiftData schema、文件存储、native bridge、`ImageGenerationBackend`、Hugging Face 下载或 Gallery 复用逻辑。
- 不接入 StoreKit，不新增 product ID、价格、SKU、购买、恢复、receipt、subscription 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、native XCFramework、CI workflow 或 Xcode project。
- 不重构 Generate、Models、Prompts、Gallery 或 Plan 页面结构。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- `ParameterEditor`、`SciFiStatusPill`、`SciFiMetric` 属于 UI 展示层，只负责展示、输入控件和用户反馈。
- `GenerationParameters` 负责参数值、范围和归一化，本轮不得改变参数业务规则。
- Generate 和 Prompt Template Editor 通过 binding 使用 `ParameterEditor`；本轮不得改变 binding 所指向的状态或保存/加载路径。
- `SciFiStatusPill` 被 Generate、Models、Prompts 复用；本轮应只改善可读性，不改变状态来源、颜色含义或调用者业务语义。
- 既有 v1.9-v1.12 已用 `dynamicTypeSize.isAccessibilitySize` 做局部大字号布局回退，本轮可在共享控件中复用同一策略。

## 实现步骤

1. 阅读必读文件、本提示词、`ParameterEditor.swift`、`GenerationView.swift`、`PromptLibraryView.swift` 中调用共享组件的位置。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `ParameterEditor` 中加入 `@Environment(\.dynamicTypeSize)`。
4. 将 CFG header 抽成可复用的小布局：普通字号保持横向 label/value；accessibility Dynamic Type 下使用纵向 label/value，value 不靠压缩适配。
5. 为 Steps、CFG slider、Seed TextField、Size Menu、Width/Height stepper 和 Sampler picker 补足明确的 accessibility label/value；不得改变现有 binding、范围、step 或 picker 值。
6. 将 Seed 控件抽成局部 helper：
   - TextField 绑定、numberPad、seed value 行为保持不变。
   - 随机按钮使用 `Label("Randomize Seed", systemImage: "dice")`，普通字号可 `.labelStyle(.iconOnly)`。
   - 补足至少 44x44 命中区。
   - accessibility Dynamic Type 下让 TextField 与按钮纵向排列，避免横向挤压。
7. 将 Size preset 行抽成局部 helper：
   - preset 列表和对 `parameters.width/height` 的赋值保持不变。
   - 普通字号保持横向结构。
   - accessibility Dynamic Type 下让标题和 Menu 纵向排列。
8. 调整 `SciFiStatusPill`：
   - 读取 Dynamic Type size。
   - 移除 `minimumScaleFactor`。
   - 普通字号仍可单行；accessibility Dynamic Type 下允许 title 换行。
   - 保持 Label、系统图标、颜色、背景和描边语义。
9. 调整 `SciFiMetric`：
   - 读取 Dynamic Type size。
   - 普通字号保持当前横向卡片。
   - accessibility Dynamic Type 下改为纵向图标和文本布局，value 允许换行。
   - 图标作为装饰时避免 VoiceOver 重复朗读无意义系统图标名。
10. 保持所有 bindings、按钮 action、`parameters = GenerationParameters(...)` reset、slider、stepper、picker、menu preset 行为不变。
11. 同步文档：
    - `README.md`：说明共享参数控件、状态 pill 和 metric 支持 accessibility Dynamic Type 可读布局。
    - `md/flow/flow.md`：更新 Shared / Generate / Prompts UI 职责。
    - `md/flow/flowchart.md`：更新 Generate/Prompts 节点或共享 UI 说明。
    - `update_log.md`：追加 v1.13 记录。

## 验收标准

- Seed 随机按钮不再只是 `Image(systemName:)` 加 `accessibilityLabel`，而是有真实文本 label。
- Seed 随机按钮满足至少 44x44 命中区。
- Steps、CFG slider、Seed TextField、Size Menu、Width/Height stepper 和 Sampler picker 有明确 accessibility label/value。
- accessibility Dynamic Type 下，Seed TextField 和随机按钮不再强制挤在同一横排。
- accessibility Dynamic Type 下，Size label 和 preset Menu 不再强制挤在同一横排。
- `SciFiStatusPill` 不再使用 `minimumScaleFactor` 压缩状态文字；accessibility Dynamic Type 下长状态可以换行。
- `SciFiMetric` 在 accessibility Dynamic Type 下不再强制图标和文字横排，value 不再只能单行截断。
- 普通字号下 Generate、Prompts、Models 的共享 Sci-Fi 视觉保持紧凑、暗色、清晰。
- Reset defaults、steps、CFG slider、seed field/randomize、size presets、width/height steppers、sampler picker 行为保持不变。
- 不新增 StoreKit、Mac Catalyst、Xcode project、native backend、SwiftData schema、文件存储或 CI workflow 改动。
- 本地轻量检查和 Swift parse 通过；建议的 iPhoneOS build 通过或明确记录沙箱限制。
- `origin/main` 最新 v1.13 run 成功，Agent C 下载并核对结果包通过。
