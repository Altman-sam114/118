# v1.4（Plan 能力矩阵）

## 版本号

v1.4

## 版本分配依据

v1.2 已建立 Plan/付费能力入口，v1.3 已把 Mac 平台状态纳入 Plan。当前 Plan 已能说明事实，但付费能力候选和当前免费能力仍是散列文本，不够利于用户快速判断哪些能力已可用、哪些只是规划、哪些需要外部配置。本轮继续优化付费功能信息架构，建立 Plan 能力矩阵。

## 背景

Local Diffusion 当前没有 StoreKit 产品、订阅、entitlement、账号系统或 App Store Connect 配置。Plan 页面必须继续 truthful：当前是 Local plan，现有本地能力不被锁定，未来付费能力只是规划候选。任何 UI 都不得让用户误以为可以购买、恢复或已经获得付费权益。

## 目标

- 在 `PlanView` 中新增一个清晰的能力矩阵 / readiness 列表，展示：
  - 当前 Local plan 已包含的本地能力。
  - 未来 Pro/paid 候选能力。
  - 每项能力当前状态：Available、Planned 或 Requires configuration。
- 保持暗色科幻风格、Form 结构和 Dynamic Type 友好。
- 不接入 StoreKit，不新增购买按钮，不新增 entitlement 状态。
- 不修改 Generate、Models、Gallery、Prompts、SwiftData、文件存储、native backend 或 Xcode project。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不实现真实购买、订阅恢复、收据校验、App Store Server API、账号系统或 entitlement 持久化。
- 不把任何现有功能锁到付费层。
- 不启用 Mac Catalyst。
- 不新增 StoreKit import、product ID、价格、SKU 或假升级成功态。
- 不拆分 Swift 文件或修改 Xcode project 文件；本轮仍把小型 Plan 支撑类型留在 `RootContentView.swift`，避免扩大工程维护范围。

## 当前架构依据

- `RootContentView` 管理顶层入口，`PlanView` 当前位于同文件内。
- Plan 属于 UI 展示层，只能展示产品/平台状态，不能承担购买业务逻辑。
- 现有共享样式在 `ParameterEditor.swift` 中提供 `SciFiTheme`、`.sciFiScreen()`、`.bottomTabBarClearance()` 和 `.sciFiPanel()`。
- Swift 源码变更至少运行两条 Swift parse；为避免 SwiftUI 类型检查只在云端暴露，建议本轮继续跑沙箱外 iPhoneOS build。

## 实现步骤

1. 阅读必读文件、本提示词和 `RootContentView.swift`。
2. 确认当前分支和远端：
   - `git fetch origin`
   - `git switch main`
   - `git pull --ff-only origin main`
   - `git status --short`
3. 在 `PlanView` 中新增小型能力状态数据结构，建议字段：
   - title
   - detail
   - status
   - systemImage
   - color
4. 新增 `Capability Matrix` section：
   - 使用 `ForEach` 渲染每项能力。
   - 每行使用 `Label`/`LabeledContent`，状态文本和图标都要可读，不只靠颜色区分。
   - Local plan 已有能力建议包括：Local generation workspace、Model and storage management、Gallery and prompt reuse。
   - 未来 paid 候选建议包括：Batch queue controls、Pro prompt packs、Workflow export。
   - StoreKit/configuration 相关项必须写为 `Requires configuration`，不能写成可购买。
5. 保留现有 Current Build、Platform Status、Future Paid Capabilities 和 Availability 语义，允许删除重复或弱化的未来能力散列列表，只要 Plan 更清晰且不误导。
6. 同步文档：
   - `README.md` Current implementation 增加 Plan 能力矩阵。
   - `md/flow/flow.md` 更新 Plan UI 职责。
   - `md/flow/flowchart.md` 更新 Plan 节点。
   - `update_log.md` 追加 v1.4 记录。

- 禁止修改 native backend、Xcode 平台或 SwiftData schema。
- 禁止跳过 Agent C artifact 核对。
