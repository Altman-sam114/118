# v1.102 Prompt 模板编辑字符计数语义

## 版本号

v1.102

## 版本分配依据

v1.101 已完成 Generate prompt header 字段化字符计数语义。当前继续 v1 体验优化阶段，将同类可访问性补齐到 Prompt Library 的模板编辑器 prompt fields，范围独立、可验收，因此递增为 v1.102。

## 背景

Prompt Library 的 `PromptTemplateEditor` 中，Positive Prompt 和 Negative Prompt header 都显示 `N chars`。当前这两个字符计数没有字段上下文，VoiceOver 单独聚焦时只能听到类似 `42 chars`，无法判断属于 Positive Prompt 还是 Negative Prompt。v1.101 已为 Generate prompt editor 解决同类问题，本轮将同样的字段化语义补齐到模板编辑器。

## 目标

- 为 `PromptTemplateEditor.promptEditor` 的字符计数补充字段上下文。
- Positive Prompt 字符计数暴露 `Positive Prompt character count` label。
- Negative Prompt 字符计数暴露 `Negative Prompt character count` label。
- accessibility value 使用 `0 characters` / `1 character` / `N characters`。
- 保留可见 `N chars` 文案、PromptTemplateEditor 布局、TextEditor binding、保存行为和参数编辑行为不变。

## 非目标

- 不修改模板新增、编辑、保存、加载、搜索、分组、分类重命名或分类清除行为。
- 不修改 PromptTemplate SwiftData schema、GenerationParameters、ParameterEditor、Generate 页面或 Gallery 页面。
- 不修改 StoreKit、付费功能、Mac Catalyst、native backend、Xcode project、workflow 或 Release asset。
- 不做 simulator VoiceOver 实机朗读或视觉重设计。

## 当前架构依据

- UI 层只在 `LocalDiffusion/Views/**` 展示状态、输入、导航和用户反馈。
- `PromptTemplateEditor.promptEditor` 已经拿到 prompt field 的 `title`，可在本地 helper 中生成字段化 accessibility label。
- 这是纯 SwiftUI accessibility metadata，不应改变模板数据、保存流程或参数编辑。
- SwiftUI 可访问性应使用 label/value 提供可扫描语义，同时保持当前简洁暗色科幻风格。

## 实现步骤

1. 阅读必读文件、`PromptLibraryView.swift`、最新 v1.101 提示词和相关源码。
2. 在 `PromptTemplateEditor` 中新增私有 `promptCharacterCount(_:title:)` helper，保留可见 `Text("\(count) chars")`。
3. 将 accessibility Dynamic Type 和普通 header 两处字符计数替换为 helper 调用。
4. 为 helper 添加：
   - `.accessibilityLabel(Text("\(title) character count"))`
   - `.accessibilityValue(Text(promptCharacterCountAccessibilityValue(count)))`
5. 新增私有 `promptCharacterCountAccessibilityValue(_:)`，处理 `1 character` / `N characters`。
6. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录本轮只改变 Prompt template editor 字符计数辅助功能语义。
