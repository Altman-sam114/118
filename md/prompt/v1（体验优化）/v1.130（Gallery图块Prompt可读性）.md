# v1.130 / Gallery 图块 Prompt 可读性

## 版本号

- 版本：`v1.130`
- 阶段：`v1（体验优化）`

## 版本分配依据

- 当前最新正式版本为 `v1.129 / Gallery 图块模型名回退`。
- 人工总目标要求持续优化 Mac/iPad/UI/设计/付费功能。
- v1.129 已改善 Gallery 图块 date/model metadata 在窄 tile 宽度下的回退；本轮继续优化同一图块里 prompt 文本的垂直可读性。

## 背景

`ImageTile` 的 prompt 文本已经按 Dynamic Type 限制为普通字号 2 行、accessibility Dynamic Type 3 行，但该 `Text` 没有显式允许垂直展开。Gallery grid 在 iPad split、Stage Manager、较窄 adaptive tile 或大字号下，prompt 文本可能被父布局压缩，导致已有行数预算没有充分展示。本轮只给 prompt 文本增加 `.fixedSize(horizontal: false, vertical: true)`，让它在既有 line limit 内保留可读高度。

## 目标

1. 只修改 `ImageTile` 的 prompt `Text(image.prompt)`。
2. 在既有 `.lineLimit(dynamicTypeSize.isAccessibilitySize ? 3 : 2)` 后追加：

```swift
.fixedSize(horizontal: false, vertical: true)
```

3. 保持普通 Dynamic Type 仍最多 2 行，accessibility Dynamic Type 仍最多 3 行。
4. 保持 tile thumbnail、metadata fallback、accessibility label/value/hint、NavigationLink、filter/sort、detail navigation、SwiftData 和文件存储逻辑不变。

## 非目标

- 不修改 Gallery detail、folder/tags、删除确认、Refresh Gallery、Sort menu、filter rail 或 image grid column minimum。
- 不修改 tile panel 样式、thumbnail frame、prompt 字体、prompt line limit、metadata 字体或颜色。
- 不修改 `ImageTile` 的 accessibility label/value/hint。
- 不修改 SwiftData schema、AppFileStore、GenerationViewModel、native backend、StoreKit、Mac Catalyst、Xcode project、workflow 或测试规范。
- 不做 simulator / Stage Manager / Split View 截图目检；本轮通过 Swift parse 和云端 build 覆盖语法与构建风险。

## 当前架构依据

- `GalleryView.swift` 位于 `LocalDiffusion/Views/Gallery/`，属于 SwiftUI UI 展示层。
- `ImageTile` 只展示 `GeneratedImage` 元数据并通过 `NavigationLink` 打开详情。
- Gallery 图块已经提供 pointer hover、prompt-focused VoiceOver label、model/date/output metadata value 和 date/model metadata 窄宽度回退；本轮只改善可见 prompt 文本在既有行数内的垂直可读性，不改变数据或交互。

4. 同步更新 README、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，说明 Gallery 图块 prompt 文本会在既有行数内垂直展开以保留可读性。
