# v1.44 / 共享尺寸预设菜单语义

## 版本号

v1.44

## 版本分配依据

- 当前体验优化阶段已完成到 v1.43。
- v1.13 已为共享 `ParameterEditor` 的 Size、Width、Height 控件提供基础可读布局和辅助功能值。
- v1.42 与 v1.43 已连续补充共享参数编辑器中 Reset Defaults 与 Randomize Seed 的操作范围语义。
- `ParameterEditor` 的 `sizePresetMenu` 当前有 `Canvas size preset` label 和当前尺寸 value，但没有说明选择 preset 会同时更新 width 和 height，也没有说明自定义 width / height 仍可继续编辑。
- `ParameterEditor` 同时用于 Generate 和 Prompt Template Editor，因此本轮继续选择共享参数编辑器里的小步 VoiceOver 语义优化，不修改尺寸预设或归一化行为。

## 背景

Local Diffusion 的 canvas size 既可以通过 preset menu 选择，也可以通过 Width / Height stepper 手动调整。当前菜单会朗读当前尺寸，例如 `512 x 512`，但 VoiceOver 用户聚焦菜单时无法从菜单本身听到它会成对更新 width 和 height，也无法确认后续仍可用独立 stepper 做自定义尺寸。

本轮只给 size preset menu 增加操作 hint。菜单标题、preset 列表、当前尺寸 value、Width / Height stepper、尺寸范围、归一化逻辑和所有生成/模板行为保持不变。

## 目标

- 在 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的 `sizePresetMenu` 上补充：
  - `.accessibilityHint("Choose a preset that updates width and height. Custom width and height remain editable.")`
- 保持现有 `Menu(currentSizeText)`、`ForEach(sizePresets, id: \.0)`、preset 文案、赋值行为、tint、44pt frame、label 和 value 不变。
- 语义文案必须通用，不能写死 Generate screen，因为该组件也用于 Prompt Template Editor。
- 同步 README、flow、flowchart、update_log，记录共享尺寸预设菜单获得 width/height 更新范围和自定义尺寸仍可编辑的 VoiceOver hint。

## 非目标

- 不修改 `sizePresets` 列表、preset 名称、preset 宽高值或按钮顺序。
- 不修改 `parameters.width` / `parameters.height` 的 preset 赋值行为。
- 不修改 `widthBinding`、`heightBinding`、`normalizedDimension(_:)`、尺寸最小/最大值或 step。
- 不修改 Reset Defaults、Randomize Seed、steps、CFG、sampler 或其他参数控件。
- 不修改 Generate、Prompt Library 的保存/加载模板行为。
- 不修改 SwiftData schema、文件存储、native backend、桥接头、Xcode project、CI workflow 或 Release asset。
- 不启用 Mac Catalyst，不修改平台、签名、分发或 scheme 设置。
- 不新增 StoreKit import、product ID、价格、购买、恢复、receipt、subscription、entitlement persistence 或付费门禁。
- 不新增 simulator VoiceOver 自动化、截图基线或真机 GGUF 生成验收。

## 当前架构依据

- `ParameterEditor` 属于 `LocalDiffusion/Views/Shared/ParameterEditor.swift` 的共享 UI 展示层，只负责展示和用户输入。
- Generate 和 Prompt Template Editor 通过 `@Binding var parameters: GenerationParameters` 复用同一个参数编辑器。
- `sizePresetMenu` 当前只在 UI 层写入 `parameters.width` 和 `parameters.height`，不触碰 SwiftData、文件存储、ViewModel 或 native backend。
- 辅助功能 hint 属于 UI 表达，不应进入 ViewModel、数据层或 native 边界。
