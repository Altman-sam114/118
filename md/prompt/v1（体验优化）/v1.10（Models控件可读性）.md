# v1.10（Models 控件可读性）

## 版本号

v1.10

## 版本分配依据

v1.8 已优化 Plan 状态行的 Dynamic Type 可读性，v1.9 已让 Generate 宽屏创作台在 accessibility Dynamic Type 下回退单列。当前 Models 页面仍有多个 icon-only 按钮只提供 `Image`，VoiceOver 和 Voice Control 缺少稳定文本；模型行和未跟踪文件行也在大字号下过度依赖横向 `HStack`。本轮继续人工总目标中的 iPad/UI/design 优化，选择一个可独立验收的小步：提升 Models 控件语义和大字号布局，不改业务行为。

## 背景

Models 页面负责下载、导入、暂停、恢复、取消、删除和检查未跟踪 GGUF 文件。这里的操作风险高，按钮必须有清晰可读的语义，同时在 iPad regular 或 accessibility Dynamic Type 下不能因为状态 pill、文件名、大小文本和控制按钮同排而互相挤压。`swiftui-pro` accessibility 规则要求图标按钮必须包含文本 label，即使视觉上继续使用 icon-only 样式；Apple 最小点击区域应至少 44x44。

## 目标

- 在 `ModelLibraryView.swift` 中只优化 Models 页展示层。
- `UntrackedModelFileRow` 的导入和删除按钮改为带文本的 `Label`，视觉上可继续 icon-only。
- `ModelRow.controls` 中 Details、Delete、Pause、Cancel、Resume、Download 等按钮全部提供文本 label，不能只用 `Image`。
- 行内小按钮保留 Sci-Fi 风格，并补足至少 44x44 的可点击区域。
- `StorageSummaryRow` 在 accessibility Dynamic Type 下将 ready status pill 放到标题下方，普通字号保留横向标题结构。
- `UntrackedModelFileRow` 在 accessibility Dynamic Type 下改为文件信息和操作按钮纵向堆叠，文件名允许换行；普通字号保留紧凑横向布局。
- `ModelRow` 在 accessibility Dynamic Type 下将标题区、状态 pill、大小文本和控制按钮纵向排列，减少横向挤压；普通字号保持当前紧凑结构。
- 保持下载、暂停、恢复、取消、删除、导入、详情、确认弹窗和 SwiftData 保存行为不变。
- 同步 README、flow、flowchart、update_log。

## 非目标

- 不接入 StoreKit，不新增 product ID、价格、SKU、购买、恢复、receipt、subscription 或 entitlement 持久化。
- 不启用 Mac Catalyst，不修改 `SUPPORTED_PLATFORMS`、`SUPPORTS_MACCATALYST`、native XCFramework、CI workflow 或 Xcode project。
- 不修改 `HuggingFaceDownloadManager`、`AppFileStore`、SwiftData schema、native bridge、`GenerationViewModel`、`ImageGenerationBackend`、Generate、Gallery、Prompts 或 Plan。
- 不改共享 `SciFiStatusPill`、`SciFiSecondaryButtonStyle` 的全局行为，避免扩大影响到其他页面。
- 不绕过删除确认弹窗，不直接导入未跟踪文件。
- 不做 simulator smoke 或截图验收，除非人工另外要求或本地验证暴露必须复现的问题。

## 当前架构依据

- Models 属于 UI 层，只负责展示、输入、导航和用户反馈。
- 下载状态由 `HuggingFaceDownloadManager` 管理；文件导入/删除由 `AppFileStore` 和现有确认流程负责。
- UI 不得直接调用 native C bridge，也不得修改生成后端边界。
- `GenerationView` 和 `PlanView` 已使用 `dynamicTypeSize.isAccessibilitySize` 做大字号回退，本轮可在 Models 行内局部复用这个布局策略。
- SwiftUI 设计要求：支持 Dynamic Type；图标按钮必须包含文本 label；装饰图标应隐藏或搭配文本；点击区域不小于 44x44；状态不能只靠颜色表达。

## 实现步骤

1. 阅读必读文件、本提示词、`ModelLibraryView.swift`、共享 Sci-Fi 样式和 v1.8/v1.9 的 Dynamic Type 布局模式。
2. 同步最新 `origin/main`，确认当前分支是 `main` 且无无关改动：

```bash
git fetch origin
git status --short --branch
git rev-parse HEAD origin/main
```

3. 在 `StorageSummaryRow`、`UntrackedModelFileRow`、`ModelRow` 需要处读取 `@Environment(\.dynamicTypeSize)`。
4. 为 Models 局部新增小型 helper/modifier 时必须保持文件内私有、范围小，不影响其他页面。
5. 将 `UntrackedModelFileRow` 的两个按钮改为 `Label("Import Untracked Model", systemImage: "plus.circle")` 和 `Label("Delete Untracked Model File", systemImage: "trash")`，继续使用对应 button style 和 destructive role。
6. 将 `ModelRow.controls` 中所有 action button 改成带文本的 `Label`：
   - Details: `Model Details`
   - Delete: `Delete Model`
   - Pause: `Pause Download`
   - Cancel: `Cancel Download`
   - Resume: `Resume Download`
   - Download: `Download Model`
7. 对这些行内控制按钮设置 `.labelStyle(.iconOnly)` 和最小 44x44 可点击区域；不要改变按钮闭包调用。
8. 调整 Dynamic Type 布局：
   - `StorageSummaryRow` 普通字号标题和 status pill 横排；accessibility 字号纵向。
   - `UntrackedModelFileRow` 普通字号横排；accessibility 字号纵向，按钮组靠左并可换行。
   - `ModelRow` 普通字号保持现有 header + progress + size/control row；accessibility 字号 header status 与 size/control row 改纵向。
9. 保持 `ProgressView`、错误 message、status text/icon/color、size text 和所有业务动作不变。
10. 同步文档：
    - `README.md`：说明 Models 控件有可访问文本 label，并在 accessibility Dynamic Type 下堆叠。
    - `md/flow/flow.md`：更新 Models UI 职责。
    - `md/flow/flowchart.md`：更新 Models 节点。
    - `update_log.md`：追加 v1.10 记录。
