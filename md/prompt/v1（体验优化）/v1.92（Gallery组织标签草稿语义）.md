# v1.92 / Gallery 组织标签草稿语义

## 版本号

v1.92

## 版本分配依据

- v1.91 已完成 Gallery image tile 的 prompt-focused label 和 metadata value 拆分，并通过 `origin/main` 云端结果包验收。
- 本轮继续 Gallery 细节页体验优化，聚焦 Organization 区域的 Tags / Save Tags VoiceOver 状态，让用户在保存前能听到草稿标签、已保存标签和未保存状态。

## 背景

`ImageDetailView` 的 Organization 区域当前已经包含：

- Folder picker 的 `Image folder` label、当前 folder value 和分配 hint。
- Tags text field 的 `Image tags` label、当前文本 value 和保存 hint。
- Save Tags button 的 `Unsaved changes` / `No changes` value 和保存 hint。

不足是 Save Tags button 只读出是否有未保存变化，没有读出当前草稿标签与已保存标签的差异。VoiceOver 用户在编辑逗号分隔 tags 后，可能需要回到 text field 才能确认将要保存的标签集合。

本轮只补强 Save Tags 的可访问 value，并保持保存行为、tag parsing、folder assignment 和视觉布局不变。

## 目标

- 为 `ImageDetailView` 增加简洁的 tags 可访问描述 helper，例如：
  - 空数组：`No tags`
  - 非空数组：逗号分隔标签文本
- 将 Save Tags button 的 `accessibilityValue` 从单一 `Unsaved changes` / `No changes` 扩展为组合状态，包含：
  - `Unsaved changes` 或 `No changes`
  - `Draft tags: ...`
  - `Saved tags: ...`
- 保留 Tags text field 的当前文本可访问 value，并可复用同一个 helper 保持空状态文案一致。
- 保留 `.buttonStyle(SciFiSecondaryButtonStyle())`、44pt 最小高度、button action、`image.tags = tagText.tagsFromCSV()`、`try? modelContext.save()` 和 Organization section layout 不变。

## 非目标

- 不修改 tag CSV parsing、tag normalization、保存时机、SwiftData schema、`GeneratedImage.tags` 类型或 folder assignment binding。
- 不修改 Gallery filtering、sorting、folder/tag filters、detail navigation、reuse/regenerate/share/delete、folder editor、image deletion 或 file reconciliation。
- 不修改 visible UI copy、按钮标题、颜色、spacing、Dynamic Type layout、Form section 结构或 Sci-Fi 风格。
- 不修改 StoreKit、paid gates、Mac Catalyst、Xcode project、native backend、bridge、workflow 或 Plan 页面。
- 不新增 simulator smoke、真实 VoiceOver 朗读、真实 GGUF 生成、本机完整 build 或真实 Mac 验证作为默认本地步骤。

## 当前架构依据

- Gallery 页面位于 `LocalDiffusion/Views/Gallery/GalleryView.swift`，属于 UI 展示层。
- `ImageDetailView` 直接展示并编辑当前 `GeneratedImage` 的 folder 和 tags；本轮只改变 SwiftUI accessibility 字符串，不跨越 SwiftData 数据模型、文件存储、GenerationViewModel 或 native backend 边界。
- Tags 的草稿来源仍是 `@State private var tagText`，保存值仍是 `image.tags`。

## 实现步骤

1. 阅读 `ImageDetailView` 的 Organization section、`tagTextAccessibilityValue`、`savedTags`、`draftTags`、`hasUnsavedTagChanges` 和 `saveTagsAccessibilityValue`。
2. 新增或调整一个私有 helper，将 `[String]` 转成 VoiceOver 友好的 tag 列表描述。
3. 让 `tagTextAccessibilityValue` 基于 `draftTags` 或现有 trimmed text 返回一致的 `No tags` / 标签列表描述。
4. 让 `saveTagsAccessibilityValue` 返回组合状态：未保存状态、draft tags、saved tags。
5. 核对 `Button` action、`TextField` binding、`folderBinding`、layout 和视觉样式未变。
6. 同步 README、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 的当前 UI 事实。
