# v1.54 / Plan Mac 就绪提示语义

## 版本号

v1.54

## 版本分配依据

- v1.53 已为 Plan Availability rows 增加行级 VoiceOver hint。
- 本轮继续在 Plan 页做窄范围辅助功能语义优化，补齐 Mac readiness checklist 的行级 hint。
- 该轮属于 v1 体验优化阶段，不改变平台支持、付费功能或 native backend。

## 背景

Plan 页已经展示当前 Local plan、StoreKit 未配置事实、平台状态、Mac readiness checklist、能力矩阵、entitlement rules 和 availability rows。当前 `PlanStatusRow` 已提供 label/value，Availability row 通过局部 `accessibilityHint` 解释当前可用性语义；Mac readiness row 仍只朗读标题、详情和状态，缺少对每个 Mac 阻塞项的下一步含义说明。

## 目标

- 为 Mac readiness checklist 的四个 row 增加行级 VoiceOver hint。
- 保持现有可视文案、状态、图标、顺序、布局和业务行为不变。
- 用 hint 分别说明：
  - Xcode target platform 需要启用 Mac 或 Catalyst 平台配置。
  - Native backend slice 需要 Mac 或 Catalyst native backend slice 后才能构建 Mac app。
  - Window and sidebar QA 需要未来专门验证窗口、sidebar、keyboard 和 pointer 状态。
  - Distribution and signing 需要先做签名和分发路径的产品决策。

## 非目标

- 不启用 Mac Catalyst。
- 不修改 `SUPPORTED_PLATFORMS`、Xcode project、scheme、signing 或 distribution 配置。
- 不新增 StoreKit、product IDs、purchase/restore/receipt/subscription、entitlement persistence 或 paid gate。
- 不修改 native XCFramework、native bridge、SwiftData schema、文件存储或推理流程。
- 不修改 shared `PlanStatusRow` 的通用行为，除非实现无法局部完成。
- 不新增 simulator VoiceOver 自动化或 Mac build。

## 当前架构依据

- UI 层只在 `LocalDiffusion/Views/**` 负责展示、输入、导航和用户反馈。
- Plan 页面位于 `LocalDiffusion/Views/RootContentView.swift`，其中 `MacReadinessItem`、`MacReadinessRow` 和 `PlanStatusRow` 已隔离在 UI 层。
- `PlanStatusRow` 已统一提供 label/value；行级 hint 可在具体 row wrapper 上补充，避免影响 Capability Matrix、Entitlement Rules 和 Availability 之外的行。

## 实现步骤

1. 在 `MacReadinessItem` 增加 `accessibilityHint: String` 字段。
2. 为 `macReadinessItems` 四个元素填入对应 hint。
3. 在 `MacReadinessRow` 的 `PlanStatusRow(...)` 后追加 `.accessibilityHint(item.accessibilityHint)`。
4. 不改现有 title/detail/status/systemImage、row 顺序和布局。
5. 更新 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md` 和 `update_log.md`，记录 Plan Mac readiness row hints。
