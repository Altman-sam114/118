# v0.5（引入 Agent X 循环迭代）

## 版本号

v0.5

## 版本依据

人工指定本轮版本为 v0.5，版本标题为“引入 Agent X 循环迭代文档基线”。仓库中已存在 `v0.5（CI恢复Native资产）`，本文件放在 `v0（协作自动化）` 目录，用标题和目录区分，不覆盖既有 CI 资产恢复记录。

## 背景

项目已有 Agent A、Agent B、Agent C 三角色流程：

- Agent A 分析目标并写版本化提示词。
- Agent B 实现、跑本地轻量检查、提交并 push `origin/main`。
- Agent C 下载 GitHub Actions 未加密 artifact，核对 manifest、JUnit/摘要、日志和结果包。

本轮为未来 Agent X 主控循环做文档准备。Agent X 后续用于围绕人工总目标 X 调度多轮 `Agent A -> Agent B -> Agent C`，但本轮不启动真实循环。

## 目标

- 在 `AGENTS.md` 增加 Agent X 召唤规则、职责、循环边界、停止条件和最终回复格式。
- 在 `md/flow/flow.md` 和 `md/flow/flowchart.md` 增加 Agent X 主控循环说明和 Mermaid 图。
- 在 `md/test/test.md` 增加 Agent X 循环验证规则和小数据量 artifact 下载约束。
- 在 `md/prompt/README.md` 增加 Agent X 对每轮版本化提示词的管理规则。
- 在 README 简短补充 `agentx:` 入口和 Agent X 不替代 A/B/C 的边界。
- 在 `update_log.md` 记录本轮文档基线。

## 非目标

- 不启动真实 Agent X 循环。
- 不修改 Swift 源码、Xcode project、native backend 或 app 行为。
- 不修改 `.github/workflows/ci-results.yml`。
- 不新增测试 target。
- 不创建 PR、不切换分支、不 push。

## 验证要求

本轮是文档-only 修改，至少运行：

```bash
git diff --check
```

不需要运行 Swift parse、Xcode build、XCTest、native preflight 或 simulator smoke，除非实际误改 Swift、Xcode、workflow 或 native 文件。

## 后续 Agent X 使用方式

人工后续可用 `agentx:`、`x:` 或 `X:` 提供总目标 X。Agent X 必须将总目标拆成小轮次，并在每轮要求 Agent A 生成版本化提示词；每轮提示词必须包含目标、非目标、验证命令、CI 触发方式、artifact 内容和 Agent C 验收要求。

Agent X 不得跳过 Agent C artifact 验收，不得把旧 run、旧 artifact、本地输出或 Agent B 文字汇报冒充最新云端结果。
